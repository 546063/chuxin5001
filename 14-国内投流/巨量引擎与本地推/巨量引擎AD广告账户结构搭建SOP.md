# 巨量引擎AD广告账户结构搭建SOP

> 适用范围：巨量引擎/巨量本地推/千川
> 最后更新：2026-05-22

---

## 一、账户层级

```
广告主账户
├── 广告计划（Campaign）— 控制投放目标+预算
│   ├── 广告组/单元（Ad Group）— 控制定向+出价
│   │   ├── 创意A（Creative）
│   │   ├── 创意B
│   │   └── 创意C
│   ├── 广告组2
│   └── 广告组3
└── 广告计划2
```

### 核心原则
- **一个计划只对应一个转化目标**（表单留资 or 私信咨询 or 团购成交 or 电话拨打）
- **一个广告组只对应一个定向人群**（地域/年龄/兴趣/搜索词）
- **一个广告组至少3条创意**（素材轮播，自动选优）
- **计划之间互相独立**，方便关停/放量/对比

---

## 二、计划命名规则

格式：`[日期]-[行业/产品]-[投放目标]-[测试/放量/稳定]-[序号]`

| 示例 | 含义 |
|---|---|
| 20260522-美业-表单留资-测试-01 | 5月22日美业行业，表单留资目标，测试期第1条计划 |
| 20260522-教培-私信咨询-放量-01 | 5月22日教培行业，私信咨询目标，放量期第1条计划 |
| 20260522-餐饮-团购成交-稳定-01 | 5月22日餐饮行业，团购成交目标，稳定期第1条计划 |

### 命名禁忌
- 不要用纯数字命名
- 不要在名称中使用特殊符号（如 #, *, $）
- 不要省略日期，方便后续按时间查找

## 三、单元拆分规则

### 按地域拆分
| 拆分维度 | 适用场景 |
|---|---|
| 城市级 | 多城市投放的品牌 |
| 区域级 | 单城市多区域（如北京朝阳/海淀/丰台） |
| 半径级 | 门店为中心X公里半径 |

### 按人群拆分
| 拆分维度 | 适用场景 |
|---|---|
| 年龄 | 不同年龄段的素材/利益点不同 |
| 性别 | 男女需求差异大时（如医美vs植发） |
| 兴趣标签 | 不同兴趣人群的痛点不同 |
| 搜索词 | 高意向搜索词 vs 泛兴趣人群 |

### 按素材类型拆分
| 拆分维度 | 适用场景 |
|---|---|
| 实拍 vs 图文 | 验证素材形式效果 |
| 口播 vs 展示 | 验证内容形式 |
| 痛点型 vs 结果型 | 验证创意方向 |

**注意**：同一广告组内不要混拆素材类型，让算法自己去选优。

---

## 四、创意分组规则

每个广告组下至少 3 条创意，建议 5-8 条：

| 创意编号 | 素材类型 | 用途 |
|---|---|---|
| 创意1 | 痛点型短视频 | 测试痛点共鸣度 |
| 创意2 | 前后对比型 | 测试效果展示力 |
| 创意3 | 口播/知识型 | 测试专业信任度 |
| 创意4 | 场景/探店型 | 测试真实感 |
| 创意5 | 优惠/活动型 | 测试价格驱动力 |
| 创意6 | 客户证言/UGC | 测试社会证明 |

### 创意命名
格式：`[素材类型]-[核心钩子]-[时长]-[序号]`
例：`痛点型-头发掉光怎么办-15s-01`

## 五、转化目标设置

| 转化目标 | 适用场景 | 优化方式 |
|---|---|---|
| 表单提交 | B端获客、高客单服务 | 优化目标：表单提交 |
| 私信咨询 | 需要沟通确认需求的行业 | 优化目标：私信打开 |
| 电话拨打 | 紧急需求行业 | 优化目标：电话拨打 |
| 团购购买 | 本地生活低客单 | 优化目标：团购成交 |
| 门店到店 | 到店核销行业 | 优化目标：到店核销 |
| 主页访问 | 品牌曝光阶段 | 优化目标：主页访问 |

### 深转 vs 浅转
- **浅转**：点击/访问/互动 — 用于素材测试、品牌曝光
- **深转**：表单/私信/成交 — 用于实际获客
- **建议**：冷启动期先用深转目标打标签，不要用浅转

## 六、三种标准账户结构

### 结构1：冷启动测试结构（D1-D7）

```
计划A：核心获客测试
├── 组1：高意向人群（搜索词+兴趣） → 5条创意
├── 组2：泛人群（地域+年龄） → 5条创意
└── 组3：再营销（浏览未留资） → 3条创意

计划B：素材形式测试
├── 组1：实拍视频 → 5条创意
├── 组2：图文/信息流 → 3条创意
└── 组3：口播视频 → 5条创意

日预算：200-400元
出价：目标成本+20%（加速学习）
```

### 结构2：放量结构（D8-D30）

```
计划A：主力获客（已验证有效）
├── 组1：最佳定向 → 最佳素材5-8条
├── 组2：扩量定向 → 最佳素材3-5条
└── 组3：新素材测试 → 新素材5条

计划B：补充获客
├── 组1：新人群包 → 测试素材
└── 组2：新素材方向 → 测试定向

日预算：400-1000元
出价：目标成本（恢复正常）
```

### 结构3：稳定期结构（D30+）

```
计划A：主力稳定产出
├── 组1：核心人群 → 已验证素材
├── 组2：扩量人群 → 已验证素材
└── 组3：每周新增2条素材

计划B：探索测试
├── 组1：新定向测试
├── 组2：新素材测试
└── 组3：新转化目标测试

日预算：按ROI动态调整
出价：目标成本-10%（优化效率）
```

---

## 七、冷启动注意事项

1. **前3天不要频繁改计划** — 算法需要 20-50 个转化才能出学习期
2. **素材至少5条** — 避免单素材疲劳
3. **预算给够** — 日预算 ≥ 目标成本 × 20
4. **出价不要过低** — 冷启动期可以比目标价高 20-30%
5. **每天观察但不干预** — 只在数据明显异常时调整
6. **第4-5天做第一次优化** — 关停成本超标 150% 的计划

## 八、账户健康检查清单

| 检查项 | 标准 |
|---|---|
| 活跃计划数 | 2-5 个 |
| 每个计划广告组数 | 2-4 个 |
| 每个广告组创意数 | ≥ 3 条 |
| 素材更新频率 | 每 3-5 天新增 2-3 条 |
| 学习期通过率 | ≥ 60% |
| 消耗集中度 | 前3条计划消耗占比 ≤ 70% |
