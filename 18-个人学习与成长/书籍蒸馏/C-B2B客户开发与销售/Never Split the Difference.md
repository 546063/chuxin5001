# 《Never Split the Difference》蒸馏

## 这本书最值得带走的 5 个思想

### 思想 1：同理心不是让步，是武器

- **原始含义**：谈判中最强力的工具不是压价或让步，而是让对方感觉你真正理解了他
- **底层逻辑**：人在感到被理解后，会放下防御，开始告诉你真实情况。信息才是谈判的最大筹码
- **适用场景**：WhatsApp 私聊、客户嫌贵、谈判僵局
- **不适用场景**：标准化产品的自动成交
- **对我业务的应用**：客户嫌贵时，不要急着降价或解释，先说"听起来你觉得这个价格超出了你的预算，是这样吗？"——用"是这样吗？"结尾的标注句，让客户说出真实顾虑

### 思想 2：标注（Labeling）

- **原始含义**：说出对方的情绪和顾虑，用"听起来你..."、"好像你觉得..."这样的句式
- **底层逻辑**：当你准确描述了对方的顾虑时，对方会觉得被理解，防御会降低
- **适用场景**：客户犹豫、客户有顾虑但不说
- **不适用场景**：客户已经明确表达诉求
- **对我业务的应用**：
  - "听起来你担心的是效果不达标怎么办"
  - "好像你对之前的代运营经历不太满意"
  - 标注后停顿，让客户自己说真话

### 思想 3：校准问题（Calibrated Questions）

- **原始含义**：用"什么"和"怎么"开头的问题，把问题抛回给对方，让他帮你解决问题
- **底层逻辑**：不直接说"不行"，而是让对方自己意识到他的要求不合理
- **适用场景**：客户砍价、客户提不合理要求
- **不适用场景**：紧急情况下需要快速决定
- **对我业务的应用**：
  - 客户要大幅降价 → "如果价格降低 50%，我能给你什么样的服务呢？你觉得我应该怎么调整才能保证效果？"
  - 客户要加服务但不加预算 → "如果我同时做这些内容，你觉得质量会怎么变化？"
  - 永远不要用"为什么"，会让人产生防御

### 思想 4：不要妥协——折中是最差的选择

- **原始含义**：折中（split the difference）不是解决问题，是双方都没得到想要的
- **底层逻辑**：折中意味着你从一开始就没真正谈判——只是各退一步，谁都没赢
- **适用场景**：价格谈判、合作条件谈判
- **不适用场景**：长期关系中的一次性让步
- **对我业务的应用**：不要"各退一步"。要么找到客户真正的需求（可能不是价格问题），要么不成交。一个 50% 折扣的烂客户不如一个原价的好客户

### 思想 5：让对方觉得是你的主意

- **原始含义**：最高明的谈判是让对方觉得最终的方案是他自己想出来的
- **底层逻辑**：人对自己想出来的主意执行力最强
- **适用场景**：推进客户做决定
- **不适用场景**：客户有明确主见且方向正确
- **对我业务的应用**：不要直接推方案，而是用校准问题引导客户自己说出方案："你觉得什么样的方式能帮你最快看到效果？"客户说出后，顺势确认"那我按照你说的做"

## 可以沉淀成的判断问题
1. 我有没有用标注句让客户说出真实顾虑？
2. 客户嫌贵时，我是在解释还是在问校准问题？
3. 我是在妥协折中，还是在找真正的解决方案？
4. 客户觉得这个方案是他的还是我的？

## 一句话总结
这本书最适合在"客户嫌贵、谈判僵持、不知道怎么推进客户做决定"时被调用。
