## 二、执行顺序

**Step 1**：把提示词1发给 Hermes → 建立老板画像+情报中心基础架构 → 老板确认画像 → Hermes 跑完 Week 0 基准快照（3-5小时）

**Step 2**：把提示词2发给 Hermes/OpenClaw → 建立周二02:00定时任务 → 测试确认跑通

**Step 3**：把提示词3发给 Hermes/OpenClaw → 建立每月1号03:00定时任务 → 测试并启动

**Step 4**：定时任务跑顺后（约2-4周）→ 让 Hermes 反向优化 subject-learning Skill，新增"情报中心维护"子技能

## 三、三类问题 vs 三个系统

| 系统 | 解决什么问题 | 更新频率 |
|------|-------------|----------|
| 静态知识库（UPS/AI娃娃SOP） | 我业务该怎么干的方法论 | 偶尔（平台大改版时复核） |
| 周报（政策合规） | 我会不会被封号/被罚款 | 每周 |
| 月报（竞品市场） | 我的选品/内容策略对不对 | 每月 |

---

## 四、分层过滤原则

老板每周真正要读的只有：🟢重大告警（通常0-2条）+ 月初的TL;DR（3句话）。其余都是备查，Hermes在问答时自动调用。

- 🟢级：立即推送飞书
- 🟡级：写入本周周报
- ⚪级：仅日志一行

---

## 五、老板现在要做的事

**今天**：
1. 把提示词1（初始化）发给 Hermes
2. 等 Hermes 产出 `_老板画像.md` 后仔细过一遍（画像是整个系统的根基，花15分钟认真审）

**画像确认后**：
3. 让 Hermes 继续跑完 Week 0 快照
4. 把提示词2和3也发给 Hermes/OpenClaw，让它们建定时任务

**4周之内**：
5. 每次跑完抽查一次分级质量
6. 调参

**3个月后**：
7. 回看情报中心是否真的被用，没用就改；被用就让 Hermes 反向优化 subject-learning Skill

---

## 六、与知识库的联动

- 周报发现🔴政策变化 → 自动标记相关SOP需复核
- 月报发现竞品重大创新 → 自动提示相关SOP章节要新增
- 季度回看 → 整体SOP做季度级更新

老板不用手工维护知识库，靠情报中心驱动更新。
