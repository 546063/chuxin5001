## 1. 一句话总结

客户是故事的英雄，品牌是引导英雄成功的导师——用SB7框架让客户一眼看懂"你帮我解决什么问题"。

## 2. 这本书适合解决我的什么问题

- **选品**：用故事框架描述产品价值，而非参数堆砌
- **定位**：建立"客户英雄+我是导师"的品牌叙事
- **内容**：所有内容围绕同一个品牌脚本
- **曝光**：用故事结构提升内容的记忆度和传播度
- **客户开发**：让客户感到被理解，而非被推销
- **询盘转化**：客户清晰知道"你能帮我解决什么问题"
- **报价成交**：用故事框架组织报价和方案
- **长期经营**：建立一致的品牌叙事资产
- **一人公司效率**：一个品牌脚本指导所有内容生产
- **风险控制**：避免自说自话、客户听不懂的品牌传播

## 3. 核心思想

### 3.1 客户是英雄，不是品牌
- **原文思想**：大多数品牌错误地把自己当作故事的英雄。实际上，客户才是英雄，品牌只是导师（Yoda），帮助英雄成功。
- **我的理解**：我的社媒内容、网站、开发信，主角应该是客户（他们的挑战、他们的成功），不是我（我的公司、我的历史）。
- **对跨境社媒外贸的启发**：不要发"我们公司成立于XX年"，要发"我们的客户如何用这台设备提升了产能"。
- **对食品加工机械业务的启发**：视频开头不是"这是我们最新的面条机"，而是"你的面条厂是否面临产能瓶颈？"

### 3.2 SB7框架
- **原文思想**：StoryBrand 7部分框架：
  1. 一个主角（客户）
  2. 遇到一个问题（外部/内部/哲学问题）
  3. 遇到一个导师（品牌）
  4. 给主角一个计划（解决方案）
  5. 召唤主角行动（CTA）
  6. 帮助主角避免失败（风险）
  7. 帮助主角成功（愿景）
- **我的理解**：所有品牌传播都可以套用这个框架，确保客户能立刻理解价值。
- **对跨境社媒外贸的启发**：网站首页、社媒简介、视频脚本、开发信，都可以用SB7组织。
- **对食品加工机械业务的启发**：
  - 主角：中东面条厂老板
  - 问题：产能不足、人工成本高、产品质量不稳定
  - 导师：我（面条机专家）
  - 计划：三步升级方案（评估→选型→安装培训）
  - 行动：预约免费产能评估
  - 避免失败：继续用旧设备的风险（订单流失、成本上升）
  - 成功：产能翻倍、成本降低、质量稳定

### 3.3 问题三层结构
- **原文思想**：客户面临的问题有三个层次：外部问题（表面）、内部问题（情感）、哲学问题（意义）。
- **我的理解**：只解决外部问题（如"需要一台面条机"）不够，要触及内部问题（如"担心投资失败"）和哲学问题（如"好的设备是对客户的责任"）。
- **对跨境社媒外贸的启发**：内容要同时回应三层问题。"这台面条机产能500kg/h"（外部）+"让您不再担心交不出订单"（内部）+"好的设备是对每一位食客的尊重"（哲学）。
- **对食品加工机械业务的启发**：客户买设备不只是买机器，是买"安心""竞争力""对品质的追求"。

### 3.4 导师的两种工具：共情与权威
- **原文思想**：导师必须同时展示共情（我理解你的挑战）和权威（我有能力帮你）。
- **我的理解**：客户需要感到"你懂我"，同时"你行"。缺一不可。
- **对跨境社媒外贸的启发**：社媒内容既要展示专业深度（权威），也要展示对客户痛点的理解（共情）。
- **对食品加工机械业务的启发**："我理解作为面条厂老板，每天4点起床和面的辛苦"（共情）+"这台设备可以让您睡到6点"（权威）。

### 3.5 清晰的计划
- **原文思想**：客户需要清晰、简单的步骤，知道与你合作会经历什么。模糊的计划导致犹豫。
- **我的理解**：不要只说"联系我们"，要说"三步获得适合您的设备方案"。
- **对跨境社媒外贸的启发**：网站和社媒要明确展示合作流程。开发信要说明下一步具体是什么。
- **对食品加工机械业务的启发**："第一步：告诉我们您的产能需求 → 第二步：我们推荐2-3个方案 → 第三步：视频验厂确认 → 第四步：签约生产 → 第五步：安装培训"

### 3.6 两种召唤：直接式与转化式
- **原文思想**：直接式CTA（立即购买）用于已准备好的客户；转化式CTA（免费资源、试用）用于还在考虑的客户。
- **我的理解**：社媒和网站要同时提供两种CTA，覆盖不同阶段的客户。
- **对跨境社媒外贸的启发**：网站同时有"立即询盘"（直接）和"下载选型指南"（转化）。社媒同时有"私信咨询"和"加入群组获取资料"。
- **对食品加工机械业务的启发**：对热客户直接邀请验厂，对冷客户提供免费行业报告。

### 3.7 失败的风险
- **原文思想**：客户需要知道不行动的代价。但不要用恐惧营销，要温和地展示风险。
- **我的理解**：让客户理解"不改变的代价"，而非恐吓。
- **对跨境社媒外贸的启发**："继续使用旧设备，每年多支出的人工成本是XX"比"不买我们的设备你会倒闭"更有效。
- **对食品加工机械业务的启发**：计算并展示"不升级设备"的隐性成本（人工、损耗、机会成本）。

### 3.8 成功的愿景
- **原文思想**：客户需要看到行动后的美好画面。这个画面要具体、可感知。
- **我的理解**：不要只说"提高效率"，要说"每天少工作2小时，产能还提高了30%"。
- **对跨境社媒外贸的启发**：用客户案例展示成功后的具体场景。视频展示设备安装后工厂的运转。
- **对食品加工机械业务的启发**："王厂长安装设备3个月后，每天可以多睡2小时，月产能从10吨提升到25吨，还招了2个新工人做包装"

### 3.9 一句话品牌介绍
- **原文思想**：品牌需要一句话介绍，让客户在3秒内理解"你帮谁解决什么问题"。
- **我的理解**：我的社媒简介、网站标题、名片、开发信签名，都要包含这句话。
- **对跨境社媒外贸的启发**："我们帮助中小型面条厂用自动化设备降低人工成本、提升产能"——一句话说清楚。
- **对食品加工机械业务的启发**：这句话要出现在所有触点。

### 3.10 品牌脚本的统一性
- **原文思想**：所有部门、所有渠道、所有内容都要讲同一个故事。不一致会混淆客户。
- **我的理解**：我的Facebook、TikTok、LinkedIn、YouTube、网站、开发信、报价单，都要基于同一个品牌脚本。
- **对跨境社媒外贸的启发**：建立品牌脚本文档，所有内容生产前对照检查。
- **对食品加工机械业务的启发**：即使是我个人回复客户，也要符合品牌脚本的语调和信息。

## 4. 可执行原则

### 原则1：客户永远是内容的主角
- **来源观点**：客户是英雄
- **核心解释**：所有内容的主角是客户，不是我
- **适用场景**：内容创作、网站文案、开发信、视频脚本
- **我的业务应用**：发布内容前检查：这条内容的主角是谁？
- **具体动作**：
  - 建立"主角检查"习惯：每条内容问自己"客户看到这条内容，会觉得是关于他的还是关于我的？"
  - 内容公式：客户痛点 → 我的理解 → 解决方案 → 客户成功案例
  - 删除所有"我们成立于XX年""我们有XX员工"的自说自话
- **反例**："我们公司成立于2010年，有200名员工，通过了ISO认证"——客户不关心
- **相关双链**：[[客户问题即内容]]、[[11-AI技能与智囊团/50_我的个人操作系统/我的内容原则]]、[[StoryBrand产品介绍提示词]]

### 原则2：用SB7框架组织所有核心文案
- **来源观点**：SB7框架
- **核心解释**：用7步故事框架确保品牌传播清晰有力
- **适用场景**：网站首页、社媒简介、宣传册、视频脚本、开发信
- **我的业务应用**：为我的业务写一个完整的SB7品牌脚本
- **具体动作**：
  - 写出我的SB7脚本（见第3.2节示例）
  - 将脚本分解为：一句话介绍、问题描述、计划步骤、CTA、风险、成功愿景
  - 所有核心文案基于这个脚本
- **反例**：网站信息杂乱、社媒内容无主题、开发信只说产品参数
- **相关双链**：[[StoryBrand产品介绍提示词]]、[[生成高信任感文案提示词]]

### 原则3：同时回应三层问题
- **来源观点**：问题三层结构
- **核心解释**：内容要同时触及外部、内部、哲学三个层次
- **适用场景**：产品介绍、客户沟通、内容创作
- **我的业务应用**：每个产品描述都包含三层问题回应
- **具体动作**：
  - 外部：设备参数、功能、价格
  - 内部：省心、安心、有面子、不被同事质疑
  - 哲学：对品质的追求、对客户的责任、对行业的贡献
  - 检查清单：每个核心文案是否包含三层？
- **反例**：只讲参数（外部）、只讲情感（内部）、只讲大道理（哲学）
- **相关双链**：[[客户痛点诊断]]、[[顾问式销售]]

### 原则4：展示共情+权威
- **来源观点**：导师的两种工具
- **核心解释**：让客户感到"你懂我"同时"你行"
- **适用场景**：个人简介、内容创作、客户初次沟通
- **我的业务应用**：所有触点同时传递共情和权威
- **具体动作**：
  - 共情："我理解作为面条厂老板，您最担心的是..."
  - 权威："根据我服务32个国家客户的经验..."
  - 社媒内容交替发布：共情类（理解痛点）+ 权威类（展示专业）
- **反例**：只展示专业（显得冷漠）或只展示共情（显得不专业）
- **相关双链**：[[顾问式销售]]、[[能力圈]]、[[简单具体可信]]

### 原则5：提供清晰的合作步骤
- **来源观点**：清晰的计划
- **核心解释**：客户需要知道"下一步是什么"，模糊导致犹豫
- **适用场景**：网站、开发信、报价单、客户沟通
- **我的业务应用**：为不同阶段的客户提供清晰的下一步
- **具体动作**：
  - 网站展示：3步获得方案
  - 开发信结尾：明确的下一步（"回复您的产能需求，我24小时内提供方案"）
  - 报价单：附上下一步流程图
  - 客户沟通：每次结束都确认下一步
- **反例**："如有需要请联系我们"——模糊、无行动引导
- **相关双链**：[[SPIN客户跟进提示词]]、[[报价前确认SOP]]

## 6. 可变成提示词的内容

### 提示词1：SB7品牌脚本生成器
- **使用场景**：建立或优化品牌叙事
- **输入信息**：我的产品、目标客户、客户痛点、我的优势
- **输出格式**：完整的SB7脚本（主角、问题、导师、计划、CTA、风险、成功）
- **判断标准**：是否让客户3秒内理解"你帮我解决什么问题"

### 提示词2：客户英雄内容转换器
- **使用场景**：将产品中心的内容转换为客户中心的内容
- **输入信息**：原始产品描述、目标客户画像
- **输出格式**：客户英雄版本（主角视角、痛点驱动、成功导向）
- **判断标准**：主角是否从"我"变成了"客户"

### 提示词3：三层问题挖掘器
- **使用场景**：深度理解客户问题，生产有穿透力的内容
- **输入信息**：客户表面需求、行业背景
- **输出格式**：外部问题、内部问题、哲学问题的三层拆解
- **判断标准**：是否触及客户不愿说出口的深层顾虑

### 提示词4：品牌一致性检查器
- **使用场景**：发布内容前检查是否符合品牌脚本
- **输入信息**：待发布内容、我的SB7脚本
- **输出格式**：一致性评分、偏离点、修改建议
- **判断标准**：是否符合品牌脚本的核心信息

### 提示词5：CTA优化器
- **使用场景**：优化网站的行动召唤
- **输入信息**：当前CTA、目标客户阶段、页面目的
- **输出格式**：直接式CTA、转化式CTA、A/B测试建议
- **判断标准**：是否覆盖不同阶段的客户

## 7. 我的业务应用

### Facebook
- 个人简介用SB7一句话介绍
- 内容主角是客户："您的面条厂是否面临..."
- 提供两种CTA：直接（私信咨询）+ 转化（加入群组）

### TikTok
- 视频开头用问题吸引（"产能不够？"）
- 展示客户成功案例（英雄的成功）
- 结尾清晰的CTA（"评论您的产能，我帮您算需要多大设备"）

### LinkedIn
- 文章用SB7结构：客户挑战 → 我的洞察 → 解决方案 → 案例
- 展示共情+权威
- 提供转化式CTA（免费报告、咨询）

### YouTube
- 频道介绍用SB7框架
- 视频分类对应客户的不同阶段（认知→考虑→决策）
- 描述中包含清晰的下一步

### 外贸客户开发
- 开发信用SB7结构："很多面条厂老板面临...（问题）→ 我们帮助...（计划）→ 张厂长使用我们的设备后...（成功）"
- 提供转化式CTA（"回复您的产能，我发您选型指南"）

### 报价成交
- 报价单用故事框架组织：客户现状 → 问题 → 方案 → 成功愿景 → 投资
- 展示不行动的风险（温和地）
- 清晰的下一步（"确认方案后，3天内提供PI"）

8. 收集3个客户成功案例，用SB7结构重写
9. 写一封新的开发信，完全基于SB7框架
10. 建立"品牌脚本一致性检查"习惯

## 9. 不适合我的地方

- **美式营销风格**：SB7框架源于美国市场，部分表达（如过度积极的愿景）需要适配更保守的B2B文化
- **服务型企业导向**：原书更多面向服务企业，实体产品需要增加技术参数和认证信息
- **一对一服务假设**：一人公司资源有限，无法为每个客户提供高度个性化的"导师"体验，需要SOP化
- **文化差异**："英雄叙事"在西方文化中强，在部分亚洲文化中客户可能更期待"专家指导"而非"自我实现"
