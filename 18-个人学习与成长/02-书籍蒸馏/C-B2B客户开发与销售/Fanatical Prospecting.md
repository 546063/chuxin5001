# 《Fanatical Prospecting》蒸馏

## 这本书最值得带走的 4 个思想

### 思想 1：管道法则（30天规则）

- **原始含义**：你今天的开发行为，决定 30 天后的业绩。如果你今天停止开发，30 天后你的管道就会枯竭
- **底层逻辑**：销售有滞后效应。今天的开发 → 明天的对话 → 下个月的成交。不能等到没客户了才开始开发
- **适用场景**：规划日常获客节奏
- **不适用场景**：已有充足线索等待处理时
- **对我业务的应用**：即使手头有 3 个客户在做，每天也要花 30-60 分钟开发新线索。不能因为忙就停止开发

### 思想 2：空管是最贵的

- **原始含义**：管道里没有线索 = 公司最大的成本。不是广告费、不是人力费，而是没有客户找你
- **底层逻辑**：当你没有足够的线索时，你会做很多蠢事：降价、接烂客户、焦虑决策
- **适用场景**：评估业务健康度
- **不适用场景**：线索已经充足
- **对我业务的应用**：每周检查一次"管道健康度"——有多少潜在客户在哪个阶段（刚接触/在沟通/等报价/在跟进）。如果总数低于 20 个，说明开发不够

### 思想 3：多通道平衡开发

- **原始含义**：不要只依赖一个渠道。最好的 prospectors 同时使用：电话、邮件、社交销售、短信、面对面、推荐
- **底层逻辑**：单一渠道不稳定。多通道不仅增加覆盖面，还能互相强化（客户在 LinkedIn 看到你，又收到你的邮件，信任度翻倍）
- **适用场景**：B2B 客户开发策略
- **不适用场景**：刚开始测试一个渠道（先专注测试一个，验证后再扩展）
- **对我业务的应用**：LinkedIn + Facebook + TikTok + 邮件 + WhatsApp，多通道并行。但每个渠道都要有明确的每日/每周目标量

### 思想 4：社交时间块（Time Blocking）

- **原始含义**：把开发时间固定在每天同一个时间段，当成不可动摇的会议
- **底层逻辑**：如果不固定时间，开发这件事永远会被"更紧急"的事挤掉
- **适用场景**：日常时间管理
- **不适用场景**：项目交付高峰期
- **对我业务的应用**：每天上午 10:00-10:30 固定为"开发时间"——LinkedIn 加好友、Facebook 群组互动、发开发信。雷打不动

## 可以沉淀成的判断问题
1. 我的管道里现在有多少线索？够不够支撑下个月的收入目标？
2. 我今天开发了吗？还是只是在等客户找我？
3. 我在用几个渠道开发？还是只依赖一个？
4. 每天有没有固定的开发时间块？

## 一句话总结
这本书最适合在"客户来源不稳定、不知道每天该花多少时间开发、线索太少"时被调用。
