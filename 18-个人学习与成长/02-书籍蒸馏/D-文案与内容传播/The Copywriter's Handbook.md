# 《The Copywriter's Handbook》蒸馏

## 这本书最值得带走的 5 个思想

### 思想 1：好的文案不是写得"聪明"，而是让读者行动

- **原始含义**：文案的目的不是展示文采，而是让读者做具体的事——打电话、点链接、回复消息、下单
- **底层逻辑**：客户不关心你写得多好，只关心你能不能帮他赚钱或省钱。文案要站在客户利益角度写
- **适用场景**：广告文案、落地页、开发信、WhatsApp 首条消息、短视频脚本
- **不适用场景**：品牌宣传文案（目的是认知而非行动）
- **对我业务的应用**：每一段文案结尾都要有明确的 CTA（行动召唤）——"回复'FB'，我帮你免费诊断账号"，而不是"了解更多"。CTA 要具体、低门槛、有利益

### 思想 2：标题决定 80% 的成败

- **原始含义**：读标题的人是读正文人数的 5 倍。标题不好，后面写得再好也没人看
- **底层逻辑**：在信息过载的环境中，标题是唯一决定用户是否继续看的因素
- **适用场景**：短视频前 3 秒、广告标题、邮件主题行、WhatsApp 首条消息
- **不适用场景**：长文正文
- **对我业务的应用**：
  - 短视频标题模板："你的 Facebook 账号不涨询盘，是因为这 3 个错误"
  - 开发信标题："帮你每月增加 15 个 B 端询盘的方法"
  - WhatsApp 首条消息："你好，我刚看了你们的 Facebook 页面，有个地方我觉得可以优化"
  - 每次写文案，先花 50% 时间打磨标题

### 思想 3：4U 公式——好的标题必须满足至少 2 个 U

- **原始含义**：Urgent（紧迫感）、Unique（独特性）、Ultra-specific（超具体）、Useful（有用性）。好标题至少占 2 个，最好 3-4 个
- **底层逻辑**：标题要同时满足多个心理触发条件，才能让用户停下来
- **适用场景**：标题写作、Hook 设计
- **不适用场景**：内部沟通
- **对我业务的应用**：
  - 差标题："Facebook 代运营服务"——0 个 U
  - 好标题："7 天内，帮跨境工厂在 Facebook 上获取第一个 B 端询盘，否则免费"——Unique（7 天首询盘）+ Ultra-specific（跨境工厂、B 端询盘）+ Useful（免费试）
  - 每次写标题用 4U 打分，低于 2 个 U 重写

### 思想 4：销售文案的万能结构——PAS + FAB

- **原始含义**：PAS（Problem 问题 → Agitate 放大 → Solve 解决）是销售文案基本骨架。FAB（Feature 特点 → Advantage 优势 → Benefit 利益）是产品描述的框架
- **底层逻辑**：人不是因为"你的产品多好"而购买，而是因为"你的产品能解决我的问题"而购买
- **适用场景**：落地页、产品介绍、开发信、短视频脚本
- **不适用场景**：品牌故事
- **对我业务的应用**：
  - PAS 示例——问题："你的 Facebook 账号发了很多内容，但没有询盘？" → 放大："这意味着你每月在浪费至少 $500 的内容制作费和 20 小时时间" → 解决："我用 3 步法帮跨境企业在 30 天内获得第一批精准询盘"
  - FAB 示例——特点："每周 5 条定制内容" → 优势："你的账号持续活跃" → 利益："每月稳定获得 15-30 个 B 端询盘"
  - 所有文案都先用 PAS 写骨架，再用 FAB 填充产品细节

### 思想 5：信任元素——好文案必须包含信任证据

- **原始含义**：客户看到你的文案后，本能反应是"凭什么相信你"。必须在文案中加入信任证据：数据、案例、第三方背书、用户评价
- **底层逻辑**：文案再漂亮，没有信任证据就是吹牛。信任证据是把"你说的"变成"事实"
- **适用场景**：落地页、提案、开发信、WhatsApp 跟进
- **不适用场景**：内部沟通
- **对我业务的应用**：
  - 每个文案/提案至少包含 2 个信任证据：数字（"帮客户获得 47 个询盘"）、截图（后台数据截图）、案例（"XX 公司，30 天从 0 到月均 20 个询盘"）
  - 不要只说"我们很专业"，而是"这是我们上周帮一个做食品机械的跨境客户做的 Facebook 页面，30 天获得 12 个询盘"

## 可以沉淀成的判断问题
1. 我的文案标题有没有至少 2 个 U？
2. 我的文案结构是 PAS 还是只是在介绍自己？
3. 我有没有在文案里加入信任证据？
4. 我的 CTA 够不够具体和低门槛？
