# 100个思维模型库｜跨境社媒与AI工作流版

> 一句话定位：这不是“收藏100个概念”，而是把模型变成你每天做外贸获客、食品机械内容、投流、客户成交、AI自动化和Obsidian治理时的**决策工具箱**。


## 0. 使用结论

你不需要平均使用100个模型。  
你的工作更需要一个 **25个核心模型 + 45个备选模型 + 30个暂存模型** 的结构。

### 模型分级

| 等级 | 定义 | 处理方式 |
|---|---|---|
| S 核心模型 | 每周都会用，直接影响获客、成交、投流、知识库和AI执行 | 放入主模型库，做模板和复盘表 |
| A 备选模型 | 特定场景有用，但不需要每天调用 | 放入备选库，按场景调用 |
| B 暂存模型 | 目前与你的业务连接弱，先保留名称和一句话 | 不展开，避免知识库膨胀 |

## 1. 你的工作应该怎么用模型

### 1.1 五个主要工作场景

| 工作场景 | 核心问题 | 优先调用模型 |
|---|---|---|
| 跨境社媒获客 | 什么内容能带来高质量询盘？ | 黄金圈、长尾理论、飞轮效应、传播模型、损失规避 |
| 食品机械产品研究 | 哪些设备、行业、国家值得做？ | 第一性原理、机会成本、幂律分布、长尾理论、波特五力 |
| Facebook/TikTok/LinkedIn投流 | 哪些素材、国家、人群要加预算？ | 多臂老虎机、非线性、PDCA、随机游走、崎岖景观 |
| 客户成交与WhatsApp沟通 | 客户为什么不下单？怎么推进？ | 损失规避、决策树、信号模型、博弈论、阈值模型 |
| AI工作流与Obsidian治理 | 怎么让AI持续产出可复用资产？ | 金字塔、MECE、系统动力学、冗余备份、路径依赖 |


## 2. 模型调用总流程

每次遇到一个问题，不要直接问“怎么办”，而是按下面流程：

```text
问题 → 场景归类 → 选择3个模型 → 得出假设 → 找证据 → 做小实验 → 复盘入库
```

### [[机会成本模型]]

- **适用场景**：销售报价、广告预算、工厂选择
- **一句话用途**：报价/投流预算/选品决策：每做一个客户或内容方向，都要问放弃了什么更高价值机会。
- **你的工作连接**：
  - 跨境社媒：把模型用于选题、脚本、素材、客户痛点拆解。
  - 食品机械：把模型用于设备卖点、行业场景、证书与售后风险说明。
  - 客户成交：把模型用于报价、异议处理、信任建立和下一步推进。
  - AI工作流：把模型转成提示词、检查表、复盘表。
- **调用问题**：
  1. 这个问题的底层变量是什么？
  2. 哪个变量最影响结果？
  3. 我下一步最小动作是什么？
- **输出格式**：判断结论 + 证据 + 下一步动作。

# Part D：《模型思维》补充模型库

《模型思维》强调：复杂问题不要只用单一模型，而要用多模型组合来推理、解释、设计、沟通、行动、预测和探索。你可以把它作为“模型使用方法论”，而不是单纯模型清单。

## D1. 模型的7大用途：REDCAPE

| 用途 | 中文解释 | 你的工作用法 |
|---|---|---|
| Reason | 推理：看清假设和逻辑后果 | 判断客户为什么不买、广告为什么不转化 |
| Explain | 解释：解释已经发生的现象 | 复盘投流、成交失败、工厂配合问题 |
| Design | 设计：设计机制、规则、流程 | 设计报价流程、线索归属、AI工作流 |
| Communicate | 沟通：把复杂问题讲清楚 | 给客户讲机器、风险、方案 |
| Act | 行动：指导下一步动作 | 决定加预算、换素材、追客户、换工厂 |
| Predict | 预测：预测未来结果 | 判断市场机会、客户成交概率、投流趋势 |
| Explore | 探索：探索新可能 | 新国家、新设备、新内容形式测试 |

## D2. 适合你补充进入模型库的20个《模型思维》模型

| 模型 | 你的业务用法 |
|---|---|
| [[正态分布]] | 看平均表现：平均询盘成本、平均转化率、平均成交周期。不要用个别极端客户代表整体。 |
| [[幂律分布]] | 识别少数爆款素材、少数大客户、少数高价值工厂；重大收益通常不是均匀分布。 |
| [[线性模型]] | 做基础相关性：素材数量、跟进次数、回复率、报价次数与成交率。 |
| [[非线性模型]] | 承认广告和内容有阈值：预算、频次、素材疲劳后结果不线性。 |
| [[网络模型]] | 拆行业关系链：工厂、经销商、终端客户、展会、协会、KOL、平台群组。 |
| [[广播/扩散/传染模型]] | 判断内容、案例、口碑如何扩散。 |
| [[熵：不确定性建模]] | 评估信息混乱度：客户需求不清、产品参数不全、数据来源不稳定。 |
| [[随机游走]] | 识别短期波动，不因一天广告好坏就推翻策略。 |
| [[路径依赖模型]] | 处理旧流程、旧内容、旧知识库路径造成的惯性。 |
| [[局部互动模型]] | 分析客户和小圈层互动，尤其LinkedIn/WhatsApp/经销商网络。 |
| [[马尔可夫模型]] | 把客户状态建模：陌生→已回复→有需求→报价→样品/视频→成交/流失。 |
| [[系统动力学模型]] | 建立增长反馈：内容资产、信任、询盘、成交、案例、复购。 |
| [[阈值模型]] | 客户是否行动常有阈值：预算、风险、交期、证书、售后承诺。 |
| [[博弈论模型]] | 报价、代理权、付款条件、独家合作中的策略。 |
| [[合作模型]] | 设计你和工厂、客户、渠道伙伴的长期合作机制。 |
| [[机制设计模型]] | 用规则降低扯皮：线索归属、报价审批、售后责任、返佣规则。 |
| [[信号模型]] | 证书、案例、视频验厂、客户评价都是信任信号。 |
| [[学习模型]] | 建立投流/内容/销售的持续学习机制。 |

# Part E：按工作场景调用模型

## E1. 食品机械外贸获客

### 推荐模型组合

```text
第一性原理 + 长尾理论 + 损失规避 + 信号模型 + 决策树
```

### 使用方式

1. 用第一性原理拆产品：客户买的不是“切菜机”，而是“人工替代、产能稳定、卫生标准、售后确定性”。
2. 用长尾理论找机会：不要只做 vegetable cutting machine，也要做 cabbage cutting line、root vegetable washing line、meat dicing machine 等长尾场景。
3. 用损失规避写内容：客户怕停产、维修难、切割不稳定、人工贵、食品安全不达标。
4. 用信号模型补信任：视频验厂、CE证书、客户案例、出货国家、售后承诺。
5. 用决策树推进成交：客户需求→设备匹配→产能确认→报价→视频验证→付款条件→成交。

# Part J：执行规则

1. 不要追求把100个模型都写成独立长文。
2. 先把25个核心模型变成可调用模板。
3. 备选模型只在真实业务问题中被调用后再展开。
4. 暂存模型只保留名称和一句话，不主动扩写。
5. 每次复盘必须至少调用3个模型，避免单一视角。
6. 每个模型都必须连接到具体工作：内容、投流、客户、工厂、AI、知识库。
7. 没有决策价值的模型，不进入主工作流。
