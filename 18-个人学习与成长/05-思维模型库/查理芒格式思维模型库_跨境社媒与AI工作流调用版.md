## 0. 重要说明：什么叫“查理芒格提到的所有模型”？

严格说，查理·芒格没有发布过一个官方、封闭、不可增删的“全部模型清单”。他强调的是：

1. 要从多个学科抽取**大思想 / big ideas（大关键模型）**；
2. 这些模型必须互相连接成 **latticework（格栅/网状结构）**；
3. 不要只靠一个模型解释世界，否则会把现实硬塞进自己的锤子里；
4. 重点不是背模型，而是在具体问题中调用、检验、修正。

所以本文采用“**芒格式全量工作库**”标准：  
- A级：与你的工作高度相关，建议高频调用；  
- B级：业务中常见，但按场景调用；  
- C级：备选模型，做复杂复盘或模型组合时调用；  
- 每个模型都提供：关键词、工作连接、使用方法、可复制提示词。

2. 它提醒我避免什么误判

# 4. 100个芒格式模型调用卡

## 001-多模型格栅

- **等级**：A
- **英文/解释**：Latticework of Mental Models｜多学科模型网格
- **关键词**：`跨学科、决策、反单一视角`
- **模型含义**：所有复杂问题先不要只用一个解释。把心理学、经济学、概率、系统、商业模型一起挂到同一张网格上。
- **跟我的工作连接**：你的外贸获客、食品机械、投流、客户成交、AI工作流都不是单因果问题。每次重大决策先拉出3-5个模型交叉判断。
- **调用提示词**：

```text
面对【问题】，请用多模型格栅拆解：心理学、经济学、概率、系统、商业竞争各给1个判断，并输出最稳妥行动。
```


## 002-逆向思维

- **等级**：A
- **英文/解释**：Inversion｜反过来想
- **关键词**：`避坑、风控、失败清单`
- **模型含义**：先问“怎样一定会失败”，再反向设计成功路径。
- **跟我的工作连接**：做广告、选工厂、谈客户前，先列失败条件：无差异内容、低信任工厂、无报价标准、无复盘。
- **调用提示词**：

```text
请用逆向思维分析【任务】：列出10个失败原因，再给出对应规避动作。
```


## 003-激励机制

- **等级**：A
- **英文/解释**：Incentives｜利益驱动
- **关键词**：`销售、团队、代理、工厂、客户`
- **模型含义**：人的行为经常不是由口头承诺决定，而是由利益结构决定。
- **跟我的工作连接**：判断工厂是否配合、客户是否回复、广告团队是否认真，先看激励是否对齐。
- **调用提示词**：

```text
分析【合作对象/流程】的激励机制：谁因什么受益，谁可能偷懒，如何重设激励。
```


## 004-能力圈

- **等级**：A
- **英文/解释**：Circle of Competence｜能力边界
- **关键词**：`聚焦、选品、业务边界`
- **模型含义**：只在自己真正理解的领域做高杠杆决策，不懂的先降级为学习或小测试。
- **跟我的工作连接**：你应优先围绕跨境社媒+食品机械+AI工作流建能力圈，不要被所有行业资料牵着走。
- **调用提示词**：

```text
判断【机会】是否在我的能力圈内：已有经验、可验证数据、可调用资源、风险盲区分别是什么。
```


## 005-安全边际

- **等级**：A
- **英文/解释**：Margin of Safety｜安全缓冲
- **关键词**：`预算、交付、承诺、投流`
- **模型含义**：承认判断会错，提前留出时间、成本、质量和现金流缓冲。
- **跟我的工作连接**：投流预算不要一次打满；客户承诺不要过度；工厂交期要留余量；AI批量改库要先备份。
- **调用提示词**：

```text
为【计划】设计安全边际：预算、时间、质量、权限、备份分别留多少缓冲。
```


## 006-机会成本

- **等级**：A
- **英文/解释**：Opportunity Cost｜被放弃的最佳选择
- **关键词**：`选择、优先级、时间管理`
- **模型含义**：每做一件事，都在放弃另一个可能收益更高的选择。
- **跟我的工作连接**：整理知识库、找工厂、做内容、投流测试不能同时满负荷；优先做能带来客户和成交的动作。
- **调用提示词**：

```text
分析【选择A】的机会成本：我放弃了什么？是否比备选B/C更值得？
```


## 007-复利效应

- **等级**：A
- **英文/解释**：Compounding｜长期积累
- **关键词**：`内容资产、客户资产、知识库`
- **模型含义**：小的、持续的正确动作会在时间中放大。
- **跟我的工作连接**：每天沉淀行业案例、客户复盘、广告素材和标准话术，长期会形成护城河。
- **调用提示词**：

```text
把【任务】改造成复利动作：今天做什么，如何可复用，如何沉淀到Obsidian。
```


## 008-幂律/二八法则

- **等级**：A
- **英文/解释**：Power Law / Pareto｜少数决定多数
- **关键词**：`重点客户、爆款素材、核心工厂`
- **模型含义**：结果通常不是平均分布，少数因素贡献大部分结果。
- **跟我的工作连接**：重点找20%高潜力工厂、20%高价值客户、20%有效内容素材，不要平均整理500篇低价值资料。
- **调用提示词**：

```text
对【资源池】做二八筛选：找出最可能贡献80%结果的20%，并说明理由。
```


## 009-二阶思维

- **等级**：A
- **英文/解释**：Second-Order Thinking｜后果的后果
- **关键词**：`投流、报价、客户策略`
- **模型含义**：不仅看直接结果，还看后续连锁反应。
- **跟我的工作连接**：低价获客可能带来低质量客户；过度承诺可能损害长期信任；删库压缩可能破坏链接。
- **调用提示词**：

```text
分析【动作】的一阶、二阶、三阶后果，并给出是否执行。
```


## 010-概率思维

- **等级**：A
- **英文/解释**：Probabilistic Thinking｜概率判断
- **关键词**：`投流测试、客户成交、风险`
- **模型含义**：用概率和区间判断，而不是非黑即白。
- **跟我的工作连接**：客户不回复不是失败，是概率漏斗的一环；投流素材要按胜率测试。
- **调用提示词**：

```text
把【判断】改成概率表达：成功概率、失败概率、关键变量、如何提高胜率。
```


## 011-贝叶斯更新

- **等级**：A
- **英文/解释**：Bayesian Updating｜根据新证据修正判断
- **关键词**：`客户反馈、广告数据、工厂验证`
- **模型含义**：先有初始判断，看到新证据后及时更新。
- **跟我的工作连接**：客户询盘、报价反馈、广告点击率、工厂资料都应该更新你的判断，而不是固守第一印象。
- **调用提示词**：

```text
基于新证据【数据/反馈】，更新我对【对象】的判断和下一步动作。
```


## 012-基准率

- **等级**：A
- **英文/解释**：Base Rate｜先看大盘概率
- **关键词**：`选品、投流、成交预估`
- **模型含义**：判断个案前先看同类事情的一般成功率。
- **跟我的工作连接**：不要只听工厂说某设备好卖，要看行业需求、竞品、搜索量、成交周期。
- **调用提示词**：

```text
为【项目】找基准率：同类业务通常转化率/周期/风险是多少，再判断我这个案例。
```


## 013-清单思维

- **等级**：A
- **英文/解释**：Checklist｜防漏项
- **关键词**：`客户开发、探厂、广告上线、知识库治理`
- **模型含义**：把重复高风险动作标准化，减少靠记忆执行。
- **跟我的工作连接**：探厂、客户沟通、投流上线、Obsidian批量整理都应该有检查清单。
- **调用提示词**：

```text
为【流程】生成一份执行前/执行中/执行后检查清单。
```


## 014-确认偏误

- **等级**：A
- **英文/解释**：Confirmation Bias｜只看支持自己的证据
- **关键词**：`选品、广告复盘、工厂判断`
- **模型含义**：人会倾向寻找证明自己正确的信息。
- **跟我的工作连接**：不要只找支持食品机械跨境好做的资料，也要找失败案例、低转化原因。
- **调用提示词**：

```text
请反驳我对【观点】的判断，找出相反证据和我可能忽略的风险。
```


## 015-可得性偏误

- **等级**：A
- **英文/解释**：Availability Bias｜最近/显眼信息过度影响
- **关键词**：`短视频热点、客户判断`
- **模型含义**：最近看到的案例容易被误认为普遍规律。
- **跟我的工作连接**：看到一个爆款视频或一个客户成交，不能直接当成全行业规律。
- **调用提示词**：

```text
检查【判断】是否受最近案例影响，给出更全面的数据需求。
```


## 016-社会认同

- **等级**：A
- **英文/解释**：Social Proof｜从众证明
- **关键词**：`广告素材、客户信任、案例包装`
- **模型含义**：人们会用他人的选择判断是否可信。
- **跟我的工作连接**：B2B获客必须展示案例、客户国家、工厂实拍、发货记录、证书、成交截图。
- **调用提示词**：

```text
为【产品/工厂】设计社会认同证据链：案例、数据、资质、第三方背书。
```


## 017-权威误导

- **等级**：A
- **英文/解释**：Authority-Misinfluence｜盲信权威
- **关键词**：`专家建议、平台课程、AI输出`
- **模型含义**：权威会影响判断，但权威不等于正确。
- **跟我的工作连接**：AI、课程、工厂老板、投手建议都要回到证据和结果验证。
- **调用提示词**：

```text
审查【建议】是否存在权威误导：证据是什么？可验证结果是什么？
```


## 018-损失厌恶

- **等级**：A
- **英文/解释**：Loss Aversion｜害怕损失大于追求收益
- **关键词**：`客户成交、报价、投流止损`
- **模型含义**：人对损失更敏感，容易拖延或过度保守。
- **跟我的工作连接**：客户怕买错设备，你要提供风险兜底、验收标准、售后说明，而不是只报价。
- **调用提示词**：

```text
用损失厌恶模型优化【销售话术】：客户怕失去什么？如何降低感知风险。
```


## 019-禀赋/剥夺超级反应

- **等级**：A
- **英文/解释**：Deprival-Superreaction｜被拿走时反应过度
- **关键词**：`促销、客户谈判、账号权限`
- **模型含义**：人对已经拥有或即将失去的东西反应更强。
- **跟我的工作连接**：限时方案可以用，但不能滥用；客户已有供应商时，替换阻力很大。
- **调用提示词**：

```text
分析【客户/合作】中的既得利益与替换阻力，设计温和切换方案。
```


## 020-对比误判

- **等级**：A
- **英文/解释**：Contrast Misreaction｜被参照物误导
- **关键词**：`报价、方案包装、竞品对比`
- **模型含义**：判断常受前后对比影响，而非绝对价值。
- **跟我的工作连接**：报价时要设置基础版/标准版/增长版，让客户看到差异，不要单报一个价。
- **调用提示词**：

```text
为【服务/产品】设计对比框架，避免客户只按低价判断。
```


## 021-关联误导

- **等级**：A
- **英文/解释**：Mere Association｜因联想产生好恶
- **关键词**：`品牌、视频视觉、工厂形象`
- **模型含义**：人会把场景、人物、情绪和产品绑定。
- **跟我的工作连接**：食品机械视频要用干净工厂、真实产线、海外场景、专业参数建立正向联想。
- **调用提示词**：

```text
优化【内容/页面】的联想资产：视觉、语言、案例、人物、场景。
```


## 022-互惠倾向

- **等级**：A
- **英文/解释**：Reciprocation｜先给价值再求回应
- **关键词**：`私信、WhatsApp、LinkedIn`
- **模型含义**：人收到有价值帮助后更愿意回应。

- **等级**：C
- **英文/解释**：First Principles｜回到底层事实
- **关键词**：`战略、产品`
- **模型含义**：不要照搬同行，从不可再拆的事实推导。
- **跟我的工作连接**：客户真正买的是稳定产能、卫生标准、降低人工、售后保障，不是机器本身。
- **调用提示词**：

```text
用第一性原理重构【业务/产品】。
```


## 099-地图不是疆域

- **等级**：C
- **英文/解释**：Map is Not Territory｜模型不是现实
- **关键词**：`AI、报告、数据`
- **模型含义**：模型只是简化，不等于真实世界。
- **跟我的工作连接**：AI报告再漂亮，也要用客户反馈和市场数据验证。
- **调用提示词**：

```text
检查【模型/报告】与现实之间的差距。
```


## 100-现实检验

- **等级**：C
- **英文/解释**：Skin in the Game / Reality Test｜让结果说话
- **关键词**：`执行、验收`
- **模型含义**：判断最终要接受现实反馈。
- **跟我的工作连接**：不要只在知识库里推演，必须联系客户、发素材、跑广告、看数据。
- **调用提示词**：

```text
为【结论】设计现实检验：什么结果证明它对/错。
```
