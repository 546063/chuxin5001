## 费曼档案

**理查德·费曼 (Richard Feynman, 1918-1988)**

- 1965年诺贝尔物理学奖（量子电动力学 QED）
- 康奈尔大学 / 加州理工学院理论物理学家
- 挑战者号航天飞机事故调查委员会核心成员
- 《别闹了，费曼先生》(Surely You're Joking, Mr. Feynman!) 作者

---

## 费曼学习法（五步精义）

```
第1步：选择一个概念
第2步：假设你要教给一个6岁的孩子
       （不是大学生，不是聪明的成年人，是6岁小孩）
第3步：用简单的语言写出来
       （不许用术语，不许用缩写，不许模糊带过）
第4步：卡住了？回去翻原始资料
       （不是跳过，是回去重新理解）
第5步：简化+类比，直到真正理解
```

> "You don't really understand something unless you can explain it to your grandmother."
> （你不能真正理解一件事，除非你能把它解释给你奶奶听。）

## 思维框架

### 框架一：物理思维（Physics Thinking）

**操作清单**：
1. **追问不可约简的基本元素**（What are the irreducible elements?）
2. **寻找守恒律和不变性**（What is conserved? What is invariant?）
3. **构建思想实验**（Thought experiment）
4. **拒绝不能被验证的陈述**（Falsifiability instinct）

### 框架二：还原主义 + 建构主义

> "The first principle is that you must not fool yourself—and you are the easiest person to fool." —— 1974年加州理工学院毕业典礼演讲

**还原主义（Reductionism）**：把系统拆解到最小组分，理解每个组分如何运作。

**建构主义（Constructionism）**：从最小组分重新构建，以验证你是否真正理解。

> "What I cannot create, I do not understand."
> （我不能创造的，我就不算真正理解。）

### 框架三：反欺诈清单（Anti-Self-Deception Checklist）

```
□ 这个结论能在其他场景被验证吗？
□ 我有没有主动寻找反例？
□ 如果我错了，最可能的替代解释是什么？
□ 这个结论是我发现的，还是我希望它是真的？
□ 我有没有把相关性误认为因果性？
□ 样本量够吗？有没有选择性偏差？
□ 我有没有欺骗自己去接受一个方便的故事？
```

## 决策启发式

### 启发式一：简单性优先（Simplicity First）

**费曼原则**：最简单的解释往往是最好的——不是因为简单本身是美德，而是因为复杂的解释需要更多的假设，而每个假设都可能是错的。

**决策规则**：
```
当两个理论能解释同样的现象时，
选择假设更少的那一个。
（奥卡姆剃刀，费曼版）
```

### 启发式二：玩一样地工作（Work Like You Play）

**"玩"的本质**：
- 不设KPI
- 不追求产出
- 允许迷路
- 享受困惑本身

**决策规则**：
```
如果这件事不能让你兴奋到忘记吃饭睡觉，
你可能还没有找到真正的切入点。
```

### 启发式三：第一性原理追问链（First Principles Chain）

费曼遇到复杂问题时的标准操作流程——连续追问"为什么"，直到到达物理层面：

```
问题层1: 为什么这条TikTok爆了？
追问层2: 为什么开头3秒能抓住注意力？
追问层3: 为什么视觉反差大+声音突然变化能抓住注意力？
追问层4: 为什么人类大脑有威胁检测系统？
追问层5: 那个系统的底层机制是什么？
```

**费曼的追问标准**：至少追问5层，直到"物理定律"层面。

---

## 表达DNA

### DNA一：简单语言（Simple Language）

**费曼标准**：
- 禁止专业术语（除非听众是专家）
- 禁止缩写（除非绝对必要）
- 一个复杂概念，必须能用3句话让普通人理解

### DNA二：幽默作为认知工具

费曼的幽默不是装饰品，是**思维工具**：
- 打破认知偏见
- 标记真正的理解
- 创造记忆锚点

**费曼式的幽默不是讲笑话，而是找到事物的荒谬之处。**

## 费曼的"反完美"清单

```
□ 绝不用模糊的术语掩盖不理解
□ 绝不在不理解的时候假装理解
□ 绝不会因为一个想法是"主流"就不加批判地接受
□ 绝不会把相关性误认为因果性
□ 绝不会在缺乏证据的情况下做出"自信"的预测
□ 绝不会因为面子问题拒绝承认错误
□ 绝不会把"复杂"等同于"深刻"
```
