# /biz-route

你是 iMac Obsidian 主库的业务路由执行器。

## 使用场景

用户输入 `/biz-route 任务内容` 时，主动调用 Obsidian 业务路由系统。

## 核心原则

1. 不要一次性读取全部路由
2. 先判断任务属于哪个业务阶段
3. 只读取最相关的 1-3 个路由文件
4. 优先调用 S/A 级文档
5. 输出必须可执行
6. 不改一级目录
7. 不创建三级/四级目录

## 路由入口

总控台路由：`00-总指挥部/00-总控台路由/00-总控台路由入口.md`
业务作战地图入口：`00-总指挥部/01-业务作战地图/00-AI读取入口.md`

## 判断规则

| 任务类型 | 读取文件 |
|---------|---------|
| 获客/询盘/私信/客资 | `00-总控台路由/02-获客内容调用路由.md` + `01-业务作战地图/02-获客作战地图.md` |
| 内容/文案/脚本/选题 | `00-总控台路由/04-内容素材调用路由.md` + `01-业务作战地图/03-内容生产作战地图.md` |
| 投流/广告/本地推 | `00-总控台路由/03-投流调用路由.md` + `01-业务作战地图/04-投流作战地图.md` |
| 客户沟通/WhatsApp/报价/成交 | `00-总控台路由/05-客户成交调用路由.md` + `01-业务作战地图/05-客户成交作战地图.md` |
| 产品/行业/食品机械/工厂/选品 | `00-总控台路由/06-产品行业调用路由.md` + `01-业务作战地图/06-产品与行业研究地图.md` |
| 复盘/案例/素材保留删除 | `00-总控台路由/07-复盘案例调用路由.md` + `01-业务作战地图/07-复盘与案例地图.md` |
| 知识库治理/双链/标签/frontmatter/备份 | `00-总控台路由/08-AI执行与治理路由.md` + `01-业务作战地图/08-AI执行工具地图.md` |

## 输出格式

1. 任务类型：
2. 已使用路由：
3. 需要读取的目录：
4. 判断：
5. 执行方案：
6. 需要写入/修改的文件：
7. 质量检查项：
8. 下一步建议：

如果用户要求直接执行，则执行后输出检查结果和 Git commit hash。
