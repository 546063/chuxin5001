## 线索分级标准

| 等级 | 标准 | 响应时效 | 跟进频率 |
|------|------|---------|---------|
| A | 明确需求 + 预算 + 时间线 | 5 分钟内 | 每日跟进 |
| B | 明确需求，预算/时间线不确定 | 30 分钟内 | 每 2-3 天 |
| C | 有兴趣但未明确需求 | 2 小时内 | 每周 |
| D | 无效线索/竞品调研/广告性质 | 24 小时内 | 不主动跟进 |

## 跟进记录模板

### 记录格式

每条记录代表一条线索及其完整跟进历史。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| lead_id | string | ✅ | 格式：LEAD-YYYYMMDD-001 |
| customer_id | string | ✅ | 格式：Customer-A, Customer-B... |
| source_platform | string | ✅ | 来源平台 |
| source_campaign | string | ⬜ | 来源广告系列 |
| source_country | string | ✅ | 来源国家 |
| source_product | string | ⬜ | 咨询产品 |
| inquiry_date | date | ✅ | 询盘日期 |
| inquiry_content | string | ⬜ | 询盘内容摘要 |
| lead_grade | string | ✅ | A / B / C / D |
| contact_method | string | ✅ | WhatsApp / Email / Phone / LinkedIn DM |
| contact_person | string | ✅ | 负责跟进的人 |
| first_response_time | string | ⬜ | 首次响应时间（如 "5min"） |
| follow_up_log | string | ⬜ | 跟进记录（日期 + 内容，JSON 数组） |
| quote_sent | boolean | ⬜ | 是否已发报价 |
| conversion_id | string | ⬜ | 关联转化编号：CONV-YYYYMMDD-001 |
| lead_status | string | ✅ | new / contacted / qualified / quoted / won / lost / dormant |
| loss_reason | string | ⬜ | 丢单原因（价格/竞品/不需要/无预算/其他） |
| last_contact_date | date | ⬜ | 最后联系日期 |
| next_action | string | ⬜ | 下一步行动 |
| data_source | string | ✅ | manual_entry / crm_export / form_export |
| verified_status | string | ✅ | pending / verified |
| notes | string | ⬜ | 备注 |

---

## 跟进日志格式

`follow_up_log` 字段建议采用 JSON 数组格式：

```json
[
  {"date": "2026-05-22", "action": "首响", "detail": "回复询盘，发送产品目录"},
  {"date": "2026-05-23", "action": "报价", "detail": "发送 Dumpling Machine DM-500 报价单"},
  {"date": "2026-05-25", "action": "跟进", "detail": "询问是否收到报价，解答技术参数问题"}
]
```
