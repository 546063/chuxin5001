# 执行SOP

## 一句话总结

从开户到投放：基础设施准备 → 素材制作 → 结构搭建 → 发布观察 → 首次优化 → 扩量，7步完整流程。

## 核心结论

- 学习期内绝对不要大改，除非预算完全花不出去
- 每次扩量预算不超过 20%，间隔 3-4 天
- 每日只看数据不调整，每周三诊断、周五执行替换
- ASC 前 7-10 天完全不动，只换素材不改受众/出价

## 适用场景

- 新账户从零到投放
- 新产品冷启动
- 测品流程
- 扩量操作
- 日常运营节奏管理

## 具体内容

### 1. 从零到投放的完整 SOP（7 步）

**Step 1：账户与基础设施准备（Day 1-2）**
- 创建 Business Manager → 添加 Ad Account
- 添加支付方式
- 创建并安装 Pixel
- 部署 CAPI
- 验证域名
- 配置 Aggregated Event Measurement（8 事件优先级）
- 安装 Meta Pixel Helper 浏览器插件验证
- 创建至少 1 个自定义受众（网站访客）

**Step 2：素材与创意准备（Day 2-4）**
- 制作 3-5 个广告图片（不同视觉角度）
- 制作 2-3 个视频广告（15-30 秒，含 Hook）
- 准备 5 个文案变体（不同 Hook 开头）
- 制作适配不同版位的素材变体（1:1, 4:5, 9:16, 16:9）
- 准备落地页（加载速度 < 3 秒，移动端优先）

**Step 3：Campaign 结构搭建（Day 4）**
- Campaign 1：TOF 拉新（Sales 目标，ABO，3 个 Ad Set）
- Campaign 2：BOF 再营销（Sales 目标，CBO，1 个 Ad Set）

**Step 4：广告创建（Day 4-5）**
- 每个 Ad Set 放 2-3 个广告变体
- 确保文案含明确 CTA
- 落地页 URL 加 UTM 参数追踪
- 检查广告预览在不同版位的效果

**Step 5：发布与学习期观察（Day 5-12）**
- 发布后 24 小时：不碰！让系统开始学习
- Day 2-3：检查预算消耗率是否正常（> 70%）
- Day 4-5：看 CTR（是否 > 1%）、CPC、CPM
- Day 7：首次评估是否有转化数据
- Day 7-14：绝对不要大改

**Step 6：首次优化决策（Day 14）**

| 情况 | 动作 |
|------|------|
| Ad Set 转化 < 15 次 | 继续观察或关停 |
| Ad Set 转化 15-50 次 | 学习期中，等 |
| Ad Set 转化 > 50 次 | 学习完成，考虑扩量 |
| CPA 超过目标 2 倍 | 降低预算或关停 |

**Step 7：扩量策略（Day 14+）**
- 对表现好的 Ad Set 逐步加预算（每 3-4 天加 20%）
- 复制成功 Ad Set 到新的 CBO Campaign
- 考虑启动 ASC（如果有足够数据）
- 制作新素材替换疲劳广告

### 2. 冷启动 SOP

1. 创建 1 个 ABO Campaign（销售目标）
2. 设 3-5 个 Ad Set：LAL 1% / Broad Interest / Advantage+ Audience
3. 每个 Ad Set 放 3 条素材（视频优先）
4. 预算：每组 $10-20/天（总 $50-100/天）
5. 不干预 4-7 天，等学习期结束
6. 第 7 天分析数据：保留 CPM 合理 + CTR >1% + 有转化的组
7. 关掉表现差的组，将预算集中到赢家

### 3. 测品 SOP

1. 每个产品建 1 个 ABO Campaign
2. 设 5-10 个 Ad Set × $5-10/组
3. 每个 Ad Set 放 3 条素材（统一素材方便对比受众效果）
4. 运行 4-5 天
5. 筛选标准：
   - 关：CPM > 行业均值 ×2
   - 关：CTR < 0.8%
   - 关：CPC > $3
   - 留：CTR > 1.5% 且有转化
6. 赢的组进入扩量阶段

### 4. 扩量 SOP — 三步扩量法

**垂直扩量**：赢的 Ad Set 预算每天增加 20%，例：$50 → $60 → $72 → $86

**水平扩量**：复制表现最好的 Ad Set 2-3 份，每份微调（受众/版位/出价）

**迁移到 ASC**：把赢的素材上传到 ASC，设预算 = 当前 CBO 的 1.5 倍

### 5. ASC 正确操作节奏

1. 创建 ASC → 上传 6-15 条素材（视频 ≥70%）
2. 日预算 ≥ $100
3. **前 7-10 天：完全不动**
4. 第 10 天：查看每条素材表现数据
5. 只换素材：关掉 CTR 最低的 2-3 条，上传 2-3 条新素材
6. 每 7-10 天轮换一次素材
7. 预算调整：每次 +20%，每周最多调 2 次

### 6. 日常运营节奏

| 频率 | 内容 | 耗时 |
|------|------|------|
| 每日（5分钟） | 看总花费和 ROAS、检查拒审、看账户余额 | 5min |
| 每周（30分钟） | 查看7天趋势、关疲劳广告、上新素材、调预算 | 30min |
| 每月（1小时） | 账户结构评估、新格式测试、竞品情报、预算调整 | 1hr |

### 7. 广告生命周期三阶段

| 阶段 | 时间 | 特征 | 操作 |
|------|------|------|------|
| 学习期 | 1-7天 | 数据波动大 | 不干预 |
| 稳定期 | 7-21天 | CPA 稳定，效果最佳 | 正常优化 |
| 疲劳期 | 21天+ | CTR↓ CPM↑ Frequency↑ | 换素材或复制新组 |

### 8. 杀广告硬指标（满足任意2条即关）

- CPM 涨幅 > 30%（vs 稳定期均值）
- CTR 跌幅 > 30%
- Frequency > 3.0
- CPA 连续 3 天超标

### 9. 避坑清单

**严重坑（浪费预算）**
- 一天内多次改广告 → 重置学习期
- 预算加太猛（> 50%）→ 重置学习期
- 频繁新建 Campaign → 数据分散
- 选错 Campaign 目标 → 系统优化错误行为
- 像素没配 CAPI → iOS 用户数据丢失 50%+

**中坑（影响效果）**
- 受众重叠 > 30% → 自己跟自己竞价
- 只用一个版位 → 错失 40-60% 展示机会
- 广告文案太长/没重点 → CTR < 0.8%
- 落地页加载 > 3 秒 → 跳出率 > 60%

### 10. 命名规范体系

```
Campaign: [目标]_[品类]_[受众类型]_[日期]_[状态]
示例：Sales_LED_LAL1%_20260501_ACTIVE

Ad Set: [受众描述]_[版位]_[预算]/day
示例：LAL1%_Purchase_90d_Auto_$50/day

Ad: [素材类型]_[Hook类型]_[时长]_[版本]
示例：UGC_Testimonial_QuestionHook_15s_V1
```

### 11. 不同业务类型的冷启动差异

**DTC 电商冷启动**
- Day 1-2：建 ABO Campaign，3 Ad Set
- Day 3-7：不干预，等学习期
- Day 7-10：关掉 CPA > 目标的组
- Day 10-14：赢家进 CBO，开始扩量
- 预算：$50-200/天起步

**B2B 线索冷启动**
- Day 1-2：建 Lead Gen Campaign（用 Facebook Lead Form）
- Day 3-7：跑 Broad + 行业兴趣，$30-50/天
- Day 7-14：分析线索质量（不是数量）
- Day 14-21：优化 Lead Form 问题
- 预算：$30-100/天起步

**高客单（>$500）冷启动**
- Day 1-2：建 Awareness Campaign（视频观看目标）
- Day 3-14：收集视频观看 > 50% 的受众
- Day 14-21：建 Retargeting Campaign
- Day 21-30：用 LAL 跑冷受众
- 预算：$50-150/天起步
- 预期周期：30-60 天才能出首单

## 可复用方法

### 扩量决策矩阵

| 指标状态 | 动作 | 幅度 |
|---------|------|------|
| CPA < 目标 × 0.7，频率 < 2 | 大胆扩量 | +30% 预算 |
| CPA < 目标 × 0.9，频率 2-3 | 正常扩量 | +20% 预算 |
| CPA ≈ 目标，频率 < 2 | 谨慎扩量 | +10-15% 预算 |
| CPA ≈ 目标，频率 > 3 | 先换素材再扩量 | - |
| CPA > 目标 × 1.2 | 不扩，先优化 | - |
| CPA > 目标 × 1.5 持续 5 天 | 减预算或关停 | -30% 或关停 |

- 有完整的从零到投放 SOP，新手也能按步骤执行

- 日常运营节奏清晰，避免过度干预
- 避坑清单覆盖 80% 的常见错误
