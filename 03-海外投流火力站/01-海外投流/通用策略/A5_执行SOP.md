# A5 执行SOP

## 一句话总结
从零到投放的7步SOP、冷启动/测品/扩量/ASC执行流程、日常运营节奏、避坑清单

## 核心结论
- **从零到投放 7 步 SOP**：账户基础设施（Day 1-2）→ 素材准备（Day 2-4）→ Campaign 搭建（Day 4）→ 广告创建（Day 4-5）→ 发布与学习期观察（Day 5-12）→ 首次优化决策（Day 14）→ 扩量策略（Day 14+）
- **学习期内绝对不干预**：发布后前 7 天不要大改，CPA 波动 30-50% 是正常的；提前关广告是最常见的错误
- **20% 规则**：每次预算调整不超过 20%，间隔 3-4 天，超过会重置学习期
- **杀广告硬指标**（满足任意 2 条即关）：CPM 涨幅 > 30%、CTR 跌幅 > 30%、Frequency > 3.0、CPA 连续 3 天超标
- **日常运营节奏**：每日 5-10 分钟快查 + 每周 30 分钟深度检查 + 每月 1 小时结构评估

## 适用场景
- 适合平台：Facebook/Meta Ads Manager
- 适合行业：DTC 电商（标准 7 步 SOP）、B2B 询盘（Lead Gen 冷启动 SOP）、高客单价（内容培育漏斗 SOP）
- 适合场景：新账户从零搭建、新产品冷启动、测品流程、扩量策略、日常运营节奏建立

## 投流要素拆解
- 平台：Facebook/Meta
- 执行流程：基础设施 → 素材 → 搭建 → 创建 → 观察 → 优化 → 扩量
- 关键节点：Day 7 首次看数据、Day 14 首次优化、Day 14+ 扩量
- 日常节奏：每日 5 分钟（花费/审核/余额）+ 每周 30 分钟（趋势/频次/素材）+ 每月 1 小时（结构/竞品）
- 关键指标：预算消耗率 > 70%、CTR > 1%、CPA < 目标、Frequency < 2.5
- 可复用策略：按业务类型选冷启动 SOP（DTC/B2B/高客单价）、三步扩量法（垂直/水平/ASC迁移）
- 风险点：学习期内频繁调整、预算加太猛（>50%）、受众重叠 >30%、Pixel 没配 CAPI
- 适合复用：[[食品加工机械]]B2B Lead Gen 冷启动、DTC 电商标准 SOP

## 可复用方法
1. **7 步上线 SOP**：从账户创建到首次优化的完整时间线，按天执行
2. **三步扩量法**：垂直扩量（+20% 预算）→ 水平扩量（复制 Ad Set 微调）→ ASC 迁移
3. **杀广告 4 指标**：CPM 涨、CTR 降、Frequency 高、CPA 超标，任意 2 条即关
4. **测品筛选法**：4-5 天 $100-200 花费，CTR > 1.5% 且有转化 = 留，否则关
5. **命名规范体系**：`[目标]_[品类]_[受众]_[日期]_[状态]`，确保后期可分析

## 对我的业务有什么价值
- 对跨境贸易的价值：B2B 食品机械用 Lead Gen 冷启动 SOP，$30-100/天起步，21 天分析线索质量而非数量
- 对 Facebook 投流的价值：有了明确的每日/每周/每月运营节奏，避免过度干预和遗漏检查
- 对巨量本地推的价值：巨量的执行 SOP 流程与 Meta 基本一致，扩量 20% 规则和杀广告指标可迁移
- 对客户开发的价值：高客单价冷启动需要 30-60 天培育周期，避免过早判断"广告没用"
- 对知识库沉淀的价值：执行 SOP 是投流体系的行动指南，所有策略最终落地为 SOP

## 1. 从零到投放的完整 SOP（7 步）

### Step 1：账户与基础设施准备（Day 1-2）

- [ ] 创建 Business Manager → 添加 Ad Account
- [ ] 添加支付方式
- [ ] 创建并安装 Pixel（Base Code 放全局 header）
- [ ] 部署 CAPI（优先通过平台原生集成）
- [ ] 验证域名（Business Settings → Brand Safety → Domains）
- [ ] 配置 Aggregated Event Measurement（8 事件优先级）
- [ ] 安装 Meta Pixel Helper 浏览器插件验证
- [ ] 创建至少 1 个自定义受众（网站访客）

### Step 2：素材与创意准备（Day 2-4）

- [ ] 制作 3-5 个广告图片（不同视觉角度）
- [ ] 制作 2-3 个视频广告（15-30 秒，含 Hook）
- [ ] 准备 5 个文案变体（不同 Hook 开头）
- [ ] 制作适配不同版位的素材变体（1:1, 4:5, 9:16, 16:9）
- [ ] 准备落地页（加载速度 < 3 秒，移动端优先）

### Step 3：Campaign 结构搭建（Day 4）

- [ ] Campaign 1：TOF 拉新（Sales 目标，ABO，3 个 Ad Set）
  - Ad Set A：兴趣定向（核心兴趣 1）
  - Ad Set B：兴趣定向（核心兴趣 2）
  - Ad Set C：Broad（仅地区 + 年龄 + 性别）
- [ ] Campaign 2：BOF 再营销（Sales 目标，CBO，1 个 Ad Set）
  - Ad Set：30 天网站访客 + 加购未购买

### Step 4：广告创建（Day 4-5）

- [ ] 每个 Ad Set 放 2-3 个广告变体
- [ ] 确保文案含明确 CTA
- [ ] 落地页 URL 加 UTM 参数追踪
- [ ] 检查广告预览在不同版位的效果

### Step 5：发布与学习期观察（Day 5-12）

- [ ] 发布后 24 小时：不碰！让系统开始学习
- [ ] Day 2-3：检查预算消耗率是否正常（> 70%）
- [ ] Day 4-5：看 CTR（是否 > 1%）、CPC、CPM
- [ ] Day 7：首次评估是否有转化数据
- [ ] Day 7-14：**绝对不要大改**，除非预算完全花不出去

### Step 6：首次优化决策（Day 14）

| 情况 | 动作 |
|---|---|
| Ad Set 转化 < 15 次 | 继续观察或关停（看 CTR 判断是素材问题还是定向问题） |
| Ad Set 转化 15-50 次 | 学习期中，等 |
| Ad Set 转化 > 50 次 | 学习完成，考虑扩量 |
| CPA 超过目标 2 倍 | 降低预算或关停 |

### Step 7：扩量策略（Day 14+）

- [ ] 对表现好的 Ad Set 逐步加预算（每 3-4 天加 20%）
- [ ] 复制成功 Ad Set 到新的 CBO Campaign
- [ ] 考虑启动 ASC（如果有足够数据）
- [ ] 制作新素材替换疲劳广告

## 2. 冷启动 SOP（新账户/新产品）

### 从零到第一笔订单的流程
1. 创建 1 个 ABO Campaign（销售目标）
2. 设 3-5 个 Ad Set：
   - Ad Set 1：LAL 1%（种子=Page互动或行业相关兴趣）
   - Ad Set 2：Broad Interest（3-5个相关兴趣叠加）
   - Ad Set 3：Advantage+ Audience（让Meta自动选）
3. 每个 Ad Set 放 3 条素材（视频优先）
4. 预算：每组 $10-20/天（总预算 $50-100/天）
5. **不干预** 4-7 天，等学习期结束
6. 第 7 天分析数据：保留 CPM 合理 + CTR >1% + 有转化的组
7. 关掉表现差的组，将预算集中到赢家
- **判断标准**：
  - 学习期通过标志：状态变为 "Active"
  - 继续标准：CPA 在可接受范围内（< 毛利的 50%）
- **避坑点**：前 3 天数据波动大属正常，**不要提前关广告**

## 3. 测品 SOP

### ABO 测品标准流程
1. 每个产品建 1 个 ABO Campaign
2. 设 5-10 个 Ad Set（不同受众方向）× $5-10/组
3. 每个 Ad Set 放 3 条素材（统一素材方便对比受众效果）
4. 运行 4-5 天
5. 按以下标准筛选：
   - ❌ 关：CPM > 行业均值 ×2（受众太贵）
   - ❌ 关：CTR < 0.8%（素材不吸引或受众不匹配）
   - ❌ 关：CPC > $3（成本太高）
   - ✅ 留：CTR > 1.5% 且有转化
6. 赢的组进入扩量阶段
- **判断标准**：4-5天内总花费 $100-200，能判断品是否有潜力

## 4. 扩量 SOP

### 三步扩量法

**方法一：垂直扩量（同一Campaign内加预算）**
1. 把赢的 Ad Set 预算每天增加 **20%**
2. 例：$50 → $60 → $72 → $86...
3. 观察 CPA 变化：如果 CPA 涨幅 <20% → 继续加；>30% → 停止
4. 不要一次加超 20%，否则触发重新学习

**方法二：水平扩量（复制赢的 Ad Set）**
1. 复制表现最好的 Ad Set 2-3 份
2. 每份微调：一个改受众（LAL 1%→3%），一个改版位，一个改出价
3. 每份预算 = 原始的 50-70%

**方法三：迁移到 ASC**
1. 把赢的素材上传到 Advantage+ Shopping Campaign
2. 设预算 = 当前 CBO 的 1.5 倍
3. 运行 7-10 天观察
4. 如果 ASC 效果 ≥ CBO → 逐步将更多预算转给 ASC

## 5. Advantage+ SOP

### ASC 的正确操作节奏
1. 创建 ASC → 上传 6-15 条素材（视频 ≥70%）
2. 日预算 ≥ $100（太少=学习太慢）
3. **前 7-10 天：完全不动**（不调预算、不换素材、不改受众）
4. 第 10 天：查看每条素材的表现数据
5. **只换素材**：关掉 CTR 最低的 2-3 条，上传 2-3 条新素材
6. 每 7-10 天轮换一次素材
7. 预算调整：每次 +20%，每周最多调 2 次
- **判断标准**：
  - ASC 好的信号：CPA 稳定或下降、自动发现新受众
  - ASC 差的信号：10天后 CPA 还在剧烈波动 → 可能数据量不够
- **避坑点**：ASC 里不要放太多低质素材，一条差素材会拉低整个 Campaign

## 6. 日常运营 SOP

### 每日检查（10 分钟）

1. 预算消耗率（是否 < 70% → 可能有投放问题）
2. CPM 是否异常飙升（> 历史均值 50% → 竞争加剧/受众疲劳）
3. 是否有广告被拒审
4. 关键指标趋势（对比昨天/上周）

### 每周检查（30 分钟）

1. ROAS/CPA 趋势
2. 各 Ad Set 转化数据
3. 广告频次（> 4 考虑换素材）
4. 受众重叠检查
5. 新素材替换决策

### 每月检查（1 小时）

1. 整体账户结构是否仍然最优
2. 是否有新广告格式/功能可以尝试
3. 竞争对手广告情报收集（Facebook Ad Library）
4. 预算分配调整

## 7. 优化窗口期管理

### 广告生命周期的三个阶段

| 阶段 | 时间 | 特征 | 操作 |
|---|---|---|---|
| 学习期 | 1-7天 | 数据波动大 | 不干预 |
| 稳定期 | 7-21天 | CPA稳定，效果最佳 | 正常优化 |
| 疲劳期 | 21天+ | CTR↓ CPM↑ Frequency↑ | 换素材或复制新组 |

### 杀广告的硬指标（满足任意2条即关）
- CPM 涨幅 > 30%（vs 稳定期均值）
- CTR 跌幅 > 30%
- Frequency > 3.0
- CPA 连续 3 天超标

## 8. 避坑清单

### 🔴 严重坑（浪费预算）

| 坑 | 后果 | 避免方法 |
|---|---|---|
| **一天内多次改广告** | 重置学习期，花冤枉钱 | 每 3-4 天做一次调整 |
| **预算加太猛（> 50%）** | 重置学习期 | 每次加 10-20%，间隔 3-4 天 |
| **频繁新建 Campaign** | 数据分散，学习永远完不成 | 精简结构，集中预算 |
| **选错 Campaign 目标** | 系统优化错误的行为 | 销售选 Sales，线索选 Leads |
| **像素没配 CAPI** | iOS 用户数据丢失 50%+ | 必须同时配 |

### 🟡 中坑（影响效果）

| 坑 | 后果 | 避免方法 |
|---|---|---|
| **受众重叠 > 30%** | 自己跟自己竞价 | 用 Audience Overlap 工具检查 |
| **只用一个版位** | 错失 40-60% 展示机会 | 用 Advantage+ Placements |
| **广告文案太长/没重点** | CTR < 0.8% | 前 125 字符放核心信息 |
| **落地页加载 > 3 秒** | 跳出率 > 60% | 优化图片、用 CDN |
| **忽视负面反馈** | 广告质量下降 | 定期查看评论、及时处理投诉 |

### 🟢 小坑（降低效率）

| 坑 | 后果 | 避免方法 |
|---|---|---|
| **素材尺寸不全** | 部分版位自动裁剪效果差 | 做全 1:1, 4:5, 9:16 |
| **没有 UTM 追踪** | 无法在 GA4 验证数据 | 所有链接加 UTM |
| **命名混乱** | 后期无法分析 | 统一命名规则 |
| **不看 Ad Library** | 不知道竞品在做什么 | 每月至少看一次 |

## 9. 常见问题速查

### Q: 广告审核被拒怎么办？
→ 检查广告政策（Meta Advertising Policies），修改后重新提交。常见原因：虚假承诺、歧视性内容、Before/After 对比图。

### Q: 预算花不出去？
→ 检查：①受众是否太窄 ②出价是否太低 ③广告质量是否太差 ④版位是否受限

### Q: CPA 突然飙升？
→ 检查：①季节性竞争 ②素材疲劳 ③受众重叠 ④Pixel 数据丢失 ⑤落地页是否挂了

### Q: 什么时候该关广告？
→ 花费 > 2 倍目标 CPA 仍无转化 → 关；CTR < 0.5% 持续 3 天 → 换素材；频次 > 5 → 轮换

## 10. 命名规范体系

### Campaign 命名规则

```
格式：[目标]_[品类]_[受众类型]_[日期]_[状态]

示例：
Sales_LED_LAL1%_20260501_ACTIVE
Leads_SaaS_Broad_20260503_TESTING
Purchase_Beauty_ASC_20260505_SCALING
RT_All_WarmAudience_20260507_PAUSE
```

### Ad Set 命名规则

```
格式：[受众描述]_[版位]_[预算]/day

示例：
LAL1%_Purchase_90d_Auto_$50/day
Broad_25-45_F_M_Auto_$30/day
Interest_Fitness_Auto_$20/day
```

### Ad 命名规则

```
格式：[素材类型]_[Hook类型]_[时长]_[版本]

示例：
UGC_Testimonial_QuestionHook_15s_V1
Brand_Demo_PatternInterrupt_30s_V2
Image_BeforeAfter_None_1x1_V1
```

## 11. 不同业务类型的冷启动差异

### DTC 电商冷启动

```
Day 1-2：建 ABO Campaign，3 Ad Set（LAL 1% / Broad / 核心兴趣）
Day 3-7：不干预，等学习期
Day 7-10：关掉 CPA > 目标的组
Day 10-14：赢家进 CBO，开始扩量
Day 14+：考虑 ASC
预算：$50-200/天起步
```

### B2B 线索冷启动

```
Day 1-2：建 Lead Gen Campaign（用 Facebook Lead Form）
Day 3-7：跑 Broad + 行业兴趣，$30-50/天
Day 7-14：分析线索质量（不是数量）
Day 14-21：优化 Lead Form 问题（加筛选问题）
Day 21+：用高质量线索建 LAL
预算：$30-100/天起步
```

### 高客单（>$500）冷启动

```
Day 1-2：建 Awareness Campaign（视频观看目标）
Day 3-14：收集视频观看 > 50% 的受众
Day 14-21：建 Retargeting Campaign 给观看用户
Day 21-30：用 LAL（种子=已成交客户）跑冷受众
Day 30+：根据成交数据优化
预算：$50-150/天起步
预期周期：30-60 天才能出首单
```

## 12. 扩量决策矩阵

| 指标状态 | 动作 | 幅度 |
|---------|------|------|
| CPA < 目标 × 0.7，频率 < 2 | 大胆扩量 | +30% 预算 |
| CPA < 目标 × 0.9，频率 2-3 | 正常扩量 | +20% 预算 |
| CPA ≈ 目标，频率 < 2 | 谨慎扩量 | +10-15% 预算 |
| CPA ≈ 目标，频率 > 3 | 先换素材再扩量 | - |
| CPA > 目标 × 1.2 | 不扩，先优化 | - |
| CPA > 目标 × 1.5 持续 5 天 | 减预算或关停 | -30% 或关停 |

### 水平扩量的 4 种方向

1. **新受众**：复制赢的 Ad Set，改受众类型（LAL 1% → LAL 3%）
2. **新市场**：相同素材和结构，投新国家/地区
3. **新版位**：如果只在 Feed 跑得好，加 Reels/Stories
4. **新素材方向**：相同受众，用完全不同的素材类型（品牌片 → UGC）
