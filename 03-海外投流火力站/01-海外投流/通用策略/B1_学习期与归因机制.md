# B1 学习期与归因机制

## 一句话总结
Meta归因模型演变、7天点击归因、iOS SKAN、学习期原理与退出条件、预算分配策略

## 核心结论
- **学习期门槛**：每个 Ad Set 需要 7 天内积累 50 次优化事件才能退出学习期，期间 CPA 波动 30-50% 是正常的，不要在学习期内做优化决策
- **归因窗口缩水影响**：iOS 14.5 后默认归因窗口从 28 天点击缩至 7 天点击，同一广告 ROAS 可能"看起来"降了 20-40%，但实际转化没变
- **跨工具数据差异正常**：BM 与 GA4 差异 20-40% 属正常（归因模型不同），差异 >50% 才需要排查 Pixel 部署
- **6 种加速学习期方法**：提高预算、降优化事件层级、合并 Ad Set、用 CBO 替代 ABO、Broad 替代精准定向、缩短归因窗口
- **数据信任优先级**：Meta Ads Manager（优化基准）> Shopify 后台（订单数最准）> GA4（趋势验证）> 第三方归因工具（参考）

## 适用场景
- 适合平台：Facebook/Meta Ads Manager（跨工具数据对比涉及 GA4、Shopify、Triple Whale）
- 适合行业：所有行业（学习期和归因是通用机制）
- 适合场景：新账户冷启动、学习期过不了排查、数据差异解读、归因窗口选择、预算分配优化

## 投流要素拆解
- 平台：Facebook/Meta
- 学习期门槛：50 转化/7 天/Ad Set
- 归因窗口：7 天点击（默认）/ 1 天浏览（可选）/ 7D Click + 1D View
- 数据源差异：BM vs GA4（低 15-30%）vs Shopify（低 10-20%）vs Triple Whale（高 10-20%）
- 关键指标：学习期状态（Learning/Active/Learning Limited）、事件数/周、预算消耗率
- 可复用策略：事件降级阶梯（Purchase→ATC→VC→LP View）、20% 渐进调整规则
- 风险点：频繁调整重置学习期、预算太低过不了学习期、归因窗口选错导致误判
- 适合复用：[[食品加工机械]]B2B 长周期归因、DTC 电商标准 7 天归因

## 可复用方法
1. **学习期预算计算器**：最低日预算 = (50 × 目标 CPA) ÷ 7，预算不够就降事件层级
2. **事件降级阶梯**：Purchase → InitiateCheckout → AddToCart → ViewContent → Link Click，逐级降直到有足够数据
3. **蜜月期扩量法**：退出学习期后 3-7 天 CPA 低 10-20%，此时 +20% 预算趁系统信心高
4. **归因窗口匹配法**：冲动购买用 7D Click、高客单价用 7D Click + 1D View、B2B 需增量测试
5. **跨工具验证 SOP**：以 BM 为优化基准，GA4 做趋势验证，Shopify 看实际订单数，接受 20-40% 差异

## 对我的业务有什么价值
- 对跨境贸易的价值：B2B 食品机械决策周期长（7-30 天），用 7D Click 归因会低估转化效果，需用增量测试验证真实 ROI
- 对 Facebook 投流的价值：理解学习期原理后不再误关正常波动中的广告，节省试错时间和预算
- 对巨量本地推的价值：巨量也有学习期机制，"50 转化"原理和 20% 调整规则可迁移
- 对客户开发的价值：正确解读 BM 数据 vs 实际订单差异，避免因数据不一致导致错误决策
- 对知识库沉淀的价值：学习期和归因是数据优化的基础，所有 CPA/ROAS 优化策略都依赖正确理解归因

## 1. Meta 归因模型演变

### 从 28 天到 7 天的归因缩水
- **核心结论**：iOS 14.5 后 Meta 被迫缩减排归因窗口。默认设置从"28天点击+1天浏览"缩到"7天点击"
- **演变时间线**：
  - 2021 前：28天点击 + 28天浏览 + 1天互动
  - 2021-2023：7天点击 + 1天浏览（仅限已ATT授权的iOS用户）
  - 2024+：7天点击为默认，浏览归因大幅减少
- **对 ROAS 的影响**：同一广告，归因窗口缩短后 ROAS 可能"看起来"降了 20-40%，但实际转化没变
- **操作建议**：在 Ads Manager → 列自定义 → 添加不同归因窗口对比数据

### 归因窗口对比

| 归因窗口 | 说明 | 何时使用 |
|---|---|---|
| **7-Day Click** | 点击广告后 7 天内的转化归因 | 默认推荐 |
| **1-Day View** | 观看广告后 1 天内的转化归因 | 叠加使用 |
| **7D Click + 1D View** | 两者都算 | 品牌类广告 |

### SKAdNetwork（SKAN）归因
- **核心结论**：iOS 用户如果选择不追踪（~75%选择不追踪），Meta 只能通过 SKAN 获得聚合数据（延迟、数值被 Apple 做了隐私化处理）
- **影响**：iOS 端的转化数据不完整，Meta 用 ML 建模来"补算"缺失数据
- **判断标准**：Events Manager 里看到 "Modeled Conversions" = Meta 在帮你补算

## 2. 跨工具数据差异

### 为什么 BM 数据 ≠ 第三方工具数据
- **核心结论**：BM 数据（Ads Manager）和第三方工具（Google Analytics / Triple Whale）永远对不上，差异来源：
  1. **归因模型不同**：FB 用"点击后7天内"vs GA 用"最后点击"
  2. **点击去重**：FB 不去重同一用户多次点击，GA 会
  3. **iOS 限制**：BM 有建模数据，GA 没有
  4. **时区差异**：BM 默认广告账户时区，GA 用网站时区
- **判断标准**：BM 和 GA 数据差异 20-40% 属正常。差异 >50% 需检查 Pixel 部署
- **避坑点**：不要用 GA 作为 Facebook 广告的"真实数据源"，它们度量方式不同

### 跨工具数据差异对照

| 工具 | 数据差异 | 原因 |
|---|---|---|
| **Meta Ads Manager** | 基准 | 用 Meta 归因模型 |
| **Google Analytics 4** | 通常低 15-30% | GA4 用最后点击归因，且丢失 fbclid |
| **Shopify 后台** | 通常低 10-20% | Shopify 有自己的归因逻辑 |
| **Triple Whale** | 通常高 10-20% | 使用第三方像素，归因更宽松 |

**建议**：以 Meta Ads Manager 数据为优化基准，用三方工具做趋势验证，不要用绝对值对比。

## 3. 增量测量

### 三种测量方法论对比

| 方法 | 原理 | 适用场景 | 成本 |
|---|---|---|---|
| MTA（多触点归因） | 追踪每个触点的贡献 | 中小规模，数据驱动优化 | 低 |
| MMM（营销组合模型） | 统计建模分析渠道贡献 | 大预算、多渠道 | 中 |
| Incrementality（增量测试） | A/B实验：看广告 vs 不看广告的差异 | 验证广告真实ROI | 高（需要对照组） |

- **核心结论**：小预算用 MTA（Meta 默认），大预算用 MMM，验证关键决策用 Incrementality

## 4. 学习期原理

### 什么是学习期？
- 每个 Ad Set 创建后进入 **Learning Phase（学习期）**
- Meta 的投放系统在探索：哪种人、什么时间、哪个版位展示你的广告效果最好
- 学习期内：表现不稳定，CPA 波动大，这是正常的

### 退出学习期条件
- **50 个优化事件 / 7 天**（按 Ad Set 计算）
- 例如：优化 Purchase → 7 天内需要 50 个购买
- 优化 Landing Page View → 7 天内需要 50 个落地页访问
- 退出后显示 **"Active"**，算法进入稳定优化阶段

### 什么会重置学习期（Significant Edits）
| 会重置 | 不会重置 |
|--------|---------|
| 大幅调整预算（> 20-30%） | 小幅预算调整（< 20%） |
| 更换广告素材 | 暂停/开启单条广告 |
| 改变受众定向 | 添加新广告到现有 Ad Set |
| 更换优化事件 | 改广告文案/标题（小幅） |
| 改变版位 | 改 CBO 预算（可能影响所有 Ad Set） |

### 如何确保退出学习期？

| 你的目标 CPA | 最低日预算（7天50转化） | 建议 |
|-------------|----------------------|------|
| $10 | $72/天 | 可行 |
| $20 | $143/天 | 可行 |
| $30 | $214/天 | 需要一定预算 |
| $50 | $357/天 | 预算压力大 |
| $100 | $714/天 | 考虑优化更浅层事件 |

**公式**：最低日预算 = (50 × 目标 CPA) ÷ 7

### 如果预算不够 50 转化怎么办？
1. **优化更浅层事件**：Purchase → AddToCart → Landing Page View
2. **减少 Ad Set 数量**：集中数据到 1-2 个 Ad Set
3. **提高预算**：集中预算，确保单个 Ad Set 有足够数据
4. **合并 Ad Set**：如果多个 Ad Set 都卡在有限学习

## 5. 预算分配策略

### 三层预算架构

| 层级 | 占比 | 用途 | 出价策略 |
|------|-----|------|---------|
| 勘探预算（Prospecting） | 60-70% | 获取新客户 | Lowest Cost / ASC |
| 再营销预算（Retargeting） | 30-40% | 转化高意向用户 | Target Cost / Cost Cap |
| 测试预算 | 10-15%（从各层出） | 新受众/创意测试 | Lowest Cost |

### 关键结论
1. **使用CBO的广告主CPA平均降低12%** — 算法比人工更懂如何分配预算
2. **勘探 vs 再营销预算比例 = 60-70% : 30-40%** — 再营销ROAS（3.61）远高于勘探（2.11）
3. **每日手动调整超过1次会导致ROAS下降15%** — 频繁干预会破坏算法学习

## 6. 诊断排查

| 症状 | 原因 | 解法 |
|------|------|------|
| Learning Limited | 7 天转化 < 50 | 提高预算 / 优化更浅层事件 / 减少 Ad Set |
| 学习期超 14 天 | 转化速度不够 | 提高预算 / 优化更浅层事件 / 减少 Ad Set |
| 学习期 CPA 好，退出后变差 | 学习期数据偶然性 | 等 5-7 天稳定再判断 |
| 一改就重学 | 改动幅度过大 | 每次改动 < 20%，渐进调整 |
| CBO 预算改了全重学 | CBO 预算影响所有 Ad Set | 通过加/减 Ad Set 来调整，而非改预算 |

## 7. 学习期进阶：Exit Strategy 与数据加速

> 学习期不仅是"等"，而是可以主动加速。核心思路：让系统更快积累 50 个优化事件。

### 加速学习期的 6 种方法

| 方法 | 适用场景 | 效果 | 风险 |
|------|---------|------|------|
| **提高日预算** | 当前预算无法在 7 天内积累 50 转化 | 最直接有效 | 如果素材差，会多花冤枉钱 |
| **降优化事件层级** | Purchase 数据太少 | 事件量可增加 3-5 倍 | 优化目标变浅，最终购买可能不够 |
| **合并 Ad Set** | 多个 Ad Set 都卡在有限学习 | 数据集中到 1 个 Ad Set | 丧失受众细分能力 |
| **用 CBO 替代 ABO** | ABO 下某个 Ad Set 独大 | 预算自动向高效组倾斜 | 可能忽视有潜力但慢热的 Ad Set |
| **缩短归因窗口** | 长周期转化产品 | 数据更快归因 | 可能低估真实转化数 |
| **Broad 替代精准定向** | 受众太小导致展示不足 | 展示量可增加 5-10 倍 | 初期转化率可能下降 |

### 事件降级阶梯（从深到浅）

```
Purchase → InitiateCheckout → AddToCart → ViewContent → Link Click
```

**降级原则**：
1. 每次只降一级，不要直接跳到最浅
2. 降了之后，日预算也要对应调整（浅事件 CPA 更低，可以降预算）
3. 积累足够数据后，逐步回深

### 学习期退出后的"蜜月期"

- 退出学习期后 3-7 天，算法通常会有一个"蜜月期"——CPA 比稳定期低 10-20%
- **策略**：利用这个窗口期加预算（+20%），趁系统信心高、投放效率高时扩量
- **注意**：蜜月期结束后 CPA 会回升到稳定水平，这是正常的

## 8. 归因窗口深度对比

> 不同归因窗口下的 ROAS 差异可以很大。理解这一点，才能正确解读报表。

### 归因窗口对数据的影响

| 归因窗口 | 特点 | 转化覆盖率 | 适用场景 |
|---------|------|-----------|---------|
| **7-Day Click** | 点击后 7 天内转化都归因 | ~60-80% | 默认推荐，快速消费品 |
| **1-Day View** | 仅看广告后 1 天内转化 | ~5-15% | 品牌曝光类广告 |
| **7D Click + 1D View** | 两者叠加 | ~65-90% | 综合评估 |
| **28-Day Click**（旧制）| 点击后 28 天 | ~90-100% | 长决策周期产品 |

### 归因窗口对比 SOP

1. 在 Ads Manager 中，列设置 → 添加不同归因窗口的转化数列
2. 对比同一广告在 7D Click vs 1D View 下的转化数差异
3. 如果差异 > 50%，说明有相当多的转化发生在点击后多天
4. 对于长决策周期产品（B2B、高客单价），用更长的归因窗口评估

### 归因窗口与产品决策周期的匹配

| 产品类型 | 决策周期 | 推荐归因窗口 |
|---------|---------|-------------|
| 冲动购买（<$20） | 0-24 小时 | 7-Day Click |
| 普通电商（$20-100） | 1-7 天 | 7-Day Click |
| 高客单价（$100-500） | 3-14 天 | 7D Click + 1D View |
| B2B/SaaS（$500+） | 7-30 天 | 需要增量测试验证 |

## 9. GA4 对比 SOP

> 很多人发现 GA4 和 Meta 数据对不上就慌。其实对不上是正常的。

### 数据对比流程

1. **选同一天范围**：确保时区一致
2. **选同一归因**：GA4 用"最后点击"，Meta 用"7 天点击"——本来就不该一致
3. **接受 20-40% 差异**：这是正常的
4. **差异 > 50% 时排查**：
   - Pixel 是否正确安装？
   - GA4 是否正确安装？
   - 是否有广告拦截器影响？
   - UTM 参数是否正确？

### 为什么 GA4 数据通常低于 Meta？

| 原因 | 影响程度 | 说明 |
|------|---------|------|
| 归因模型不同 | ~15-20% | GA4 最后点击 vs Meta 7 天点击 |
| iOS ATT 限制 | ~10-15% | Meta 有建模数据，GA4 没有 |
| Cookie 限制（ITP） | ~5-10% | Safari 7 天 cookie 限制 |
| 广告拦截器 | ~5% | 部分用户拦截了 Pixel |
| 时区差异 | ~2-3% | 如果没对齐 |

## 10. 数据驱动决策清单

> 什么时候该信哪个数据源？

| 决策场景 | 信哪个 | 原因 |
|---------|--------|------|
| 判断单个广告好不好 | **Meta Ads Manager** | 用 Meta 归因，广告是在 Meta 上跑的 |
| 判断整体投放效率 | **MER（总收入÷总花费）** | 不依赖归因，看整体 |
| 判断自然流量+广告的综合效果 | **GA4** | 看所有渠道的综合效果 |
| 判断支付/订单数 | **Shopify/后台** | 订单系统是最准确的 |
| 验证 Pixel 是否正确 | **三方对比** | 如果三者都显示"有转化"，Pixel 正常 |

### 每日监控优先级

1. **Meta Ads Manager** → CPA、ROAS、消耗率
2. **Shopify 后台** → 实际订单数、收入
3. **GA4** → 趋势验证（不纠结绝对值）
