## 1. 数据驱动决策树

### ROAS 下滑诊断树
```
ROAS 下滑
  ├── CPM 上涨 > 30%？ → 竞争问题（季节性/竞品加大）
  ├── CTR 下降 > 30%？ → 素材疲劳（换新素材）
  ├── CVR 下降 > 30%？ → 落地页/产品问题
  └── 都正常？ → 归因偏差/客单价下降
```

### 学习期 6 大原因排查

| 原因 | 判断标准 | 解法 |
|---|---|---|
| 事件太少 | 优化事件 < 50 次/周 | 换优化目标（Purchase→AddToCart）或加预算 |
| 预算太低 | 日预算 < $5/Ad Set | 每组至少 $10-15/天 |
| 受众太窄 | 受众 < 1000 人 | 扩大到至少 10 万 |
| 素材差 | CTR < 0.5% | 换素材 |
| 落地页差 | 跳出率 > 80% | 优化落地页 |
| 出价太低 | 竞拍排名很低 | 提高出价或换最低成本 |

## 2. AB 测试方法论

### 核心原则
- **一次只测一个变量**：测素材就别动受众，测受众就别动出价
- **等量预算**：每个变量必须分配相同预算，否则无法判断差异来自变量还是预算
- **统计显著性**：差异 < 20% 时需要更长时间或更大样本量来判断
- **为什么不能在同一个 Campaign 里测？**：CBO 会自动把钱给表现好的 ad set，其他变量得不到足够数据

### 测试优先级排序
1. **素材**（影响力最大，占广告效果的 70-80%）
2. **Hook**（前 3 秒决定 80% 效果）
3. **Offer**（价格/套餐/赠品）
4. **受众**（在 Broad 时代重要性降低）
5. **版位**（Advantage+ 通常最优）
6. **出价策略**（有基准数据后再测）

### AB 测试标准流程
1. **写假设**："X 会比 Y 产生更低 CPA，因为……"
2. **建测试 Campaign**（ABO）：
   - 每个变量一个 Ad Set
   - 每个 Ad Set 等额预算
   - 相同受众、相同优化事件
   - 只改变要测的那个变量
3. **运行 5-7 天**：不干预、不暂停、不加素材
4. **每天记录**：CPA、花费分布、频率、学习期状态
5. **分析决策**：
   - CPA 差异 > 20% 且一致 → 采用赢家
   - CPA 差异 10-20% → 延长测试 3-5 天
   - CPA 差异 < 10% → 无显著差异，保留两者

### 素材测试 SOP（每周例行）
1. 每周一：在测试 Campaign 加 2-3 条新素材
2. 每条素材一个 Ad Set（ABO，等额预算）
3. 跑 3 天（到周四）
4. 判断标准：
   - CTR > 1.5% + CPA < 目标 → 晋级 Scaling Campaign
   - CTR < 1.0% → 淘汰
   - CTR 1.0-1.5% → 再跑 2 天
5. 周五：将赢家导入 Scaling Campaign（CBO）
6. 逐步淘汰 Scaling Campaign 中表现垫底的 20%

## 3. 预算分配框架

### 70/20/10 法则

| 类别 | 占比 | 内容 | 目标 |
|------|------|------|------|
| **已验证** | 70% | 已跑 3 周+ 的盈利 Campaign | 稳定产出 |
| **测试** | 20% | 新素材 / 新受众（每次一个变量） | 找新赢家 |
| **实验** | 10% | 全新角度 / 新漏斗 / 新格式 | 探索性发现 |

### 最低预算计算
- **单 Ad Set 最低日预算** = 目标 CPA × 3
- **推荐日预算** = (50 × 目标 CPA) ÷ 7
- 例：目标 CPA $25 → 最低 $75/天，推荐 $178/天

### 预算扩展 SOP
1. **20% 规则**：每 48-72 小时加预算 ≤ 20%
2. 扩展前检查：
   - CPA 是否稳定在目标以下？
   - 频率是否 < 3？
   - 学习期是否已退出？
3. 扩展后监控：
   - 如果 CPA 上涨 > 30% → 回退到之前预算，等 3 天
   - 如果 CPA 稳定 → 继续下一次扩展
4. **水平扩展**（加新 Ad Set / 新市场） vs **垂直扩展**（加预算）
   - 预算 > $500/天时优先水平扩展

## 4. 数据指标体系

| 指标 | 测试期标准 | 扩展期标准 |
|------|-----------|-----------|
| 测试运行时长 | 5-7 天最低 | - |
| 最低转化数 | 每变量 ≥ 15-20 | - |
| CPA 差异显著性 | > 20% 为显著 | - |
| 预算增幅 | - | ≤ 20% / 48-72h |
| 频率上限 | 测试期 < 3 | 扩展期 < 5 |

## 5. 实战案例

### 案例 1：素材 Hook A/B 测试
- 测试同一视频的两个不同前 3 秒 Hook
- Ad Set A: "90% 的人不知道这个护肤秘诀"（反问式）
- Ad Set B: "我用 30 天把这个产品的退货率降了一半"（数字式）
- 结果：B Hook CTR 2.3%，A Hook CTR 1.4%，B 胜出，CPA 低 35%

### 案例 2：70/20/10 预算重组
- 原：100% 预算跑老素材
- 改为：70% 老素材 + 20% 测试新素材 + 10% 实验 Reels 格式
- 结果：每月找到 2-3 条新赢家素材，整体 ROAS 提升 25%

### 案例 3：20% 规则扩展
- 初始日预算 $100，CPA $25
- Day 3: $120（+20%），CPA $26
- Day 6: $144（+20%），CPA $24
- Day 9: $173（+20%），CPA $27（可接受）
- Day 12: $207（+20%），CPA $25
- 12 天从 $100 → $207，CPA 保持稳定

## 6. 素材-受众-出价三维优化框架

### 三维交叉诊断

| 维度组合 | 症状 | 解法 |
|---------|------|------|
| **素材好 + 受众差** | CTR 高但转化低 | 换受众，保留素材 |
| **素材差 + 受众好** | CTR 低但转化率高 | 换素材，保留受众 |
| **素材好 + 受众好 + CPA 高** | CTR 高 + CVR 高但 CPM 极高 | 换出价策略（Cost Cap → Lowest Cost） |
| **素材差 + 受众差** | 全指标都差 | 先换素材（素材影响最大），再换受众 |
| **素材好 + 受众好 + CVR 差** | 点击多但不转化 | 落地页问题，非广告问题 |

### 优化优先级决策

```
遇到多指标同时异常时：
1. 先看 CTR → 低于 1% 优先换素材
2. 再看 CVR → 低于 1.5% 查落地页
3. 再看 CPM → 高于行业 2 倍查受众
4. 最后看 CPA → 前面 3 个都 OK 但 CPA 高 → 出价策略问题
```

## 7. 季节性优化节奏

### 全年投放节奏

| 月份 | 市场特征 | 策略调整 |
|------|---------|---------|
| **1-2 月** | 淡季，CPM 低 | 测试新素材/新受众，低成本积累 |
| **3-4 月** | 回暖 | 加大测试期赢家的预算 |
| **5-6 月** | 618 前后 | 电商提预算 30-50% |
| **7-8 月** | 暑期 | 旅游/教育品类放量，其他品类 CPM 微涨 |
| **9-10 月** | 旺季预热 | 开始准备黑五素材和受众 |
| **11 月** | 黑五旺季 | CPM 涨 50-100%，预算最大化 |
| **12 月** | 圣诞 | 延续旺季，后半段快速回落 |

### Q4 旺季准备时间线

```
T-12 周（8 月）：测试新素材方向，确定 Q4 素材主线
T-10 周：筛选 WINNER 素材，开始裂变（改 Hook/文案/CTA）
T-8 周：建测试 Campaign 验证 Q4 素材
T-6 周：确定最终素材包（至少 15-20 条）
T-4 周：逐步加预算到目标值（20% 规则）
T-2 周：停止任何大改动
T-0（黑五当周）：只监控，不调整
```

## 8. 亏损 Campaign 止损 SOP

### 止损决策树

```
Campaign ROAS < 1.0 持续 5 天+
  ├── 学习期状态？
  │     ├── Learning → 再等 2-3 天
  │     └── Active → 继续排查
  ├── CTR 是否正常？
  │     ├── 正常（> 1.5%）→ 落地页问题 → 查落地页
  │     └── 低（< 1%）→ 素材问题 → 换素材
  ├── CPA 是否超过目标 2 倍？
  │     ├── 是 → 减预算 50% 或关停
  │     └── 否 → 再观察 3 天
  └── 已经换了 2 轮素材仍无改善？
        ├── 是 → 关停整个 Campaign
        └── 否 → 继续测试
```

### 关停 vs 保留的判断

| 情况 | 动作 | 原因 |
|------|------|------|
| ROAS < 0.5 持续 7 天 | 立即关停 | 每多花一天就多浪费一天 |
| ROAS 0.5-0.8 持续 7 天 | 减预算 50%，观察 3 天 | 可能还有救 |
| ROAS 0.8-1.0 持续 7 天 | 减预算 20%，换素材 | 接近盈亏平衡 |
| ROAS 1.0-1.5 | 保持，继续优化 | 已盈利，只是不够好 |
| ROAS > 1.5 | 扩量 | 赢家 |

## 9. 数据优化中的心理学陷阱

### 常见决策偏差

| 偏差 | 表现 | 避免方法 |
|------|------|---------|
| **确认偏差** | 只看支持自己判断的数据 | 先写假设再验证，记录所有数据 |
| **损失厌恶** | 不肯关停亏损 Campaign（"再给它几天"） | 设硬性止损线，到线就关 |
| **近因效应** | 过分关注最近 1-2 天的数据 | 看 7 天趋势，不看单日 |
| **幸存者偏差** | 只分析赢家，不研究输家 | 每周复盘输家素材/受众/出价 |
| **锚定效应** | 被初始 CPA 锚定，不接受市场变化 | 每月更新行业基准 |
