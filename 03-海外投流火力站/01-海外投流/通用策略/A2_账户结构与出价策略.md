# A2 账户结构与出价策略

## 一句话总结
Meta广告账户架构原则、Campaign/Ad Set/Ad三层结构、Advantage+体系、CBO/ABO/ASC选择、六种出价策略对比与迁移SOP

## 核心结论
- Meta 广告三层架构：Campaign（目标+预算）→ Ad Set（定向+出价+版位）→ Ad（素材+文案+链接），**一个业务目标 = 一个 Campaign**
- CBO 适合成熟账户（3+ Ad Set），ABO 适合测试阶段（精确控制每个受众组），ASC 适合电商放量（全自动）
- 出价策略迁移路径：Lowest Cost（冷启动）→ Cost Cap（稳定期控制 CPA）→ ROAS Target（成熟期优化利润）
- 预算集中是关键——宁用 $100/天 × 2 Ad Set，不用 $20/天 × 10 Ad Set，数据密度决定算法效率
- 上线前 20+ 项检查清单，漏一项可能浪费一周预算

## 适用场景
- 适合平台：Facebook/Meta Ads Manager
- 适合行业：DTC 电商（ASC 首选）、B2B 询盘（Lead 目标）、SaaS、教育、本地服务
- 适合场景：新账户搭建、预算结构调整、出价策略迁移、Campaign 结构优化

## 投流要素拆解
- 平台：Facebook/Meta
- 账户架构：BM → Ad Account → Campaign → Ad Set → Ad
- 预算模式：CBO（AI 分配）/ ABO（手动分配）/ ASC（全自动）
- 出价策略：Lowest Cost / Cost Cap / Bid Cap / ROAS Target
- 广告形式：Feed / Reels / Stories / Marketplace / Audience Network
- 转化路径：广告 → 落地页/表单 → 询盘/购买
- 关键指标：CPA < 客单价 × 毛利率、预算消耗率 70-100%
- 可复用策略：按预算等级直接套用 4 套模板（$20-50/$50-200/$200-1000/>$1000）
- 风险点：受众自竞争（Auction Overlap）、频繁改预算重置学习期、选错目标优化错误行为
- 适合复用：[[食品加工机械]]B2B Lead Gen、[[教育培训]]留资、[[装修家居]]同城获客

## 可复用方法
1. **预算模板直接套用**：按日预算等级选 A/B/C/D 模板，不需要重新设计账户结构
2. **Cost Cap 公式**：Cost Cap = 目标 CPA × 1.2（预留 20% 浮动），渐进收紧每 7 天下调 5%
3. **ROAS Goal 计算**：ROAS Goal = 客单价 ÷ 目标 CPA，先算盈亏平衡 ROAS = 1 ÷ 毛利率
4. **Lowest Cost → Cost Cap 迁移**：积累 50-100 转化 → 计算 7-14 天平均 CPA → Cost Cap = 平均 CPA × 1.15 → 监控一周
5. **三层预算架构**：勘探 60-70% + 再营销 30-40% + 测试 10-15%（从各层出）

## 对我的业务有什么价值
- 对跨境贸易的价值：B2B 食品机械用 Lead 目标 + Facebook Lead Form 收集询盘，按模板 B 结构 $50-200/天起步
- 对 Facebook 投流的价值：直接套用 4 套预算模板，减少试错时间；Cost Cap 迁移SOP 确保平稳过渡
- 对巨量本地推的价值：巨量账户结构（计划→单元→创意）与 Meta 三层架构概念一致，预算集中原则通用
- 对客户开发的价值：正确的 Campaign 目标选择（Sales 不是 Traffic）直接影响线索质量，避免"点击多但询盘少"
- 对知识库沉淀的价值：账户结构是投流体系的骨架文件，所有 SOP 和优化策略都依赖正确的结构

## 1. Meta 广告三层架构

```
广告账户 (Ad Account)
├── Campaign（广告系列）← 选目标 + 预算模式
│   ├── Ad Set（广告组）← 定向 + 出价 + 版位
│   │   ├── Ad（广告）← 素材 + 文案 + 链接
│   │   └── Ad（广告）← A/B 测试变体
│   └── Ad Set（广告组）
│       └── ...
└── Campaign（广告系列）
    └── ...
```

### 核心原则
- **一个业务目标 = 一个 Campaign**：多个 Campaign 针对同一产品/受众会导致拍卖重叠（Auction Overlap），互相抬高成本
- **精简结构 = 更好的算法表现**：Meta 算法需要足够的数据密度，结构越分散，每个 ad set 分到的数据越少
- **2026 年更新**：Meta 已合并 Manual 和 Advantage+ 为统一界面，AI 驱动优化成为默认选项
- **预算集中**：宁可用 $100/天 × 2 个 Ad Set，不用 $20/天 × 10 个 Ad Set
- **避免自竞争**：同一受众不要在多个 Ad Set 中重叠

### 黄金法则

| 法则 | 说明 |
|---|---|
| **精简结构** | 每个广告账户 3-6 个 Campaign，每个 Campaign 1-4 个 Ad Set |
| **预算集中** | 集中火力在少数 Ad Set 上，不要过度分散 |
| **避免自竞争** | 同一受众不要在多个 Ad Set 中重叠 |
| **命名规范** | `[目标]_[市场]_[受众]_[日期]` 例：`PURCHASE_US_LAL_Purchasers_May2025` |
| **隔离学习** | 每个 Ad Set 需要独立的转化数据来完成学习期 |

## 2. 账户层级设置清单

### 2.1 BM 创建到可投放的 10 步

1. 创建 Business Manager → business.facebook.com
2. 验证 Business 信息（地址/电话/法律实体）
3. 创建 Facebook Page（或关联已有 Page）
4. 添加广告账户（新建或 Claim）
5. 添加支付方式（建议用信用卡，避免被封后余额冻结）
6. 设置 Pixel（见下方）
7. 设置 Conversions API（见下方）
8. 验证域名（Business Settings → Brand Safety → Domains）
9. 配置 Aggregated Event Measurement（最多 8 个优先事件）
10. 设置 Two-Factor Authentication + 团队成员权限

### 2.2 Campaign Objective 选择

| 目标 | 适用场景 | 备注 |
|---|---|---|
| **Sales** | 电商转化、独立站购买 | 需要 Pixel + CAPI 数据支撑 |
| **Leads** | B2B 留资、表单收集、WhatsApp 消息 | 即时表单 + 后续跟进 |
| **Engagement** | 视频观看、帖文互动、Messenger 对话 | 品牌曝光 + 暖受众 |
| **Traffic** | 引流到网站/落地页 | 仅做点击优化，不保证转化 |
| **Awareness** | 品牌认知、大范围曝光 | 最低 CPM，不做转化 |

**⚠️ 重要**：选 Sales 时，系统会优化"最可能购买"的人；选 Traffic 时，系统优化"最可能点击"的人。目标选错 = 流量质量不对。

### 2.3 预算模式选择

| 模式 | 适用场景 | 优点 | 缺点 |
|---|---|---|---|
| **CBO（Campaign Budget Optimization）** | 稳定投放、成熟广告账户 | AI 自动分配预算到最优 Ad Set | 单个 Ad Set 可能被边缘化 |
| **ABO（Ad Set Budget Optimization）** | 测试阶段、需要精确控制 | 精确控制每个 Ad Set 预算 | 需要手动调优 |
| **ASC（Advantage+ Shopping）** | 电商放量 | 全自动，效果通常优于手动 | 黑盒、无法精确控制定向 |

## 3. Ad Set 层级设置清单

### 3.1 定向策略（2025 更新）

| 策略 | 做法 | 适合阶段 |
|---|---|---|
| **宽定向（Broad）** | 只选年龄 + 性别 + 地区，无兴趣定向 | 成熟账户 + 有足够 Pixel 数据 |
| **兴趣定向（Interest）** | 选 3-5 个相关兴趣标签 | 新账户冷启动 |
| **自定义受众（Custom）** | 网站访客/视频观看者/客户名单 | 再营销、追单 |
| **相似受众（Lookalike）** | 基于种子用户做 1%-5% LAL | 扩展新客 |
| **Advantage+ Audience** | 给 AI 建议受众 + 详细定向作为"参考" | 替代手动定向，推荐用于 2025 |

### 3.2 版位设置

| 选项 | 建议 | 理由 |
|---|---|---|
| **Advantage+ Placements（自动版位）** | ✅ 推荐默认 | 让 AI 选择最优版位，覆盖最广 |
| **手动版位** | 仅当有明确素材适配策略 | 比如只做 Reels 广告时 |

## 4. 出价策略详解

### 4.1 四种出价策略对比

| 策略 | 控制级别 | 放量能力 | 适合阶段 |
|------|---------|---------|---------|
| **Lowest Cost（最低价）** | 无控制 | 最大 | 冷启动/测试期 |
| **Cost Cap（成本上限）** | 平均成本控制 | 中等 | 稳定期/扩展期 |
| **Bid Cap（出价上限）** | 每次拍卖上限 | 受限 | 严格单位经济模型 |
| **ROAS Target（ROAS目标）** | 价值优化 | 中等 | 电商多SKU/价格差异大 |

### Meta 拍卖系统总价值公式

```
Total Value = Bid Amount（出价金额） × Estimated Action Rate（预估行动率） + Ad Quality（广告质量）
```

- **出价金额**：你愿意为一次转化付多少钱
- **预估行动率**：算法预测该用户采取行动的概率
- **广告质量**：用户反馈、互动率、负面反馈等

**核心启示**：低出价 + 高相关素材 > 高出价 + 差素材

### 4.2 何时选什么出价策略？

| 阶段 | 推荐出价 | 切换条件 |
|---|---|---|
| **冷启动（0-50 转化）** | Lowest Cost | 默认 |
| **学习期（50-100 转化）** | Lowest Cost | 保持稳定，不改 |
| **稳定期（>100 转化）** | 尝试 Cost Cap | 当 CPA 数据稳定 7 天以上 |
| **扩量期** | Cost Cap 或 ROAS Goal | 当需要控制利润率时 |
| **竞价高峰期（黑五等）** | Bid Cap | 当需要保证展示份额时 |

### 4.3 Cost Cap 设置公式

```
Cost Cap = 目标 CPA × 1.2（预留 20% 浮动空间）
```

**示例**：如果你的目标 CPA 是 $20，则 Cost Cap 设 $24。

### 4.4 ROAS Goal 计算

```
ROAS Goal = 客单价 ÷ 目标 CPA
```

**示例**：客单价 $80，目标 CPA $20 → ROAS Goal = 4.0

### 4.5 Lowest Cost → Cost Cap 迁移流程

1. **运行 Lowest Cost** 直到积累 50-100 个转化
2. **计算过去 7-14 天平均 CPA**
3. **设置 Cost Cap = 平均 CPA × 1.15**
4. **监控第一周**：
   - 花费骤降 → Cap 太紧，每次 +10% 上调
   - 花费正常、CPA 稳定 → 找到平衡点
5. **渐进收紧**：每 7 天下调 Cap 5%，观察放量是否受影响

### 4.6 ROAS Target 设置步骤

1. **计算 Break-even ROAS**：1 ÷ 利润率
   - 例：利润率 25% → Break-even ROAS = 4x
2. **设置目标 ROAS = Break-even × 1.2**
   - 例：4x × 1.2 = 4.8x
3. **确认 Pixel/CAPI 已传 value 参数**（否则无法运行）
4. **保守起步**：目标不要设得比历史 ROAS 高太多
5. **渐进上调**：算法稳定后每次 +5-10%

## 5. Advantage+ 体系

### Advantage+ 产品线

| 功能 | 说明 | 适用场景 |
|------|------|---------|
| Advantage+ Placements | 自动版位分配 | 所有 campaign（默认开启） |
| Advantage+ Audience | AI 自动扩量超出设定受众 | Broad 定向时效果最佳 |
| Advantage+ Shopping (ASC) | 电商全自动投放 | DTC 电商，有 Catalog |
| Advantage+ Creative | 自动优化素材呈现 | 素材素材较少时 |
| Advantage+ App Campaigns | 全自动 App 安装/活跃推广 | App 推广 |

### 2025/2026 推荐结构

#### 结构 A：精简型（适合大多数广告主）
```
Campaign (CBO) — "TS Prospecting"
  └── Ad Set (1个) — Broad 定向（仅设地区+年龄+性别）
      └── Ad 1-5 条（从历史表现最好的广告用 Post ID 导入）
```
- 初始日预算：$100/天起
- 优化目标：Purchase
- 版位：自动（Advantage+ Placements）
- 定向：不设兴趣、不设 Lookalike、不设 Custom Audience

#### 结构 B：测试+扩展双轨型（成熟广告主）
```
Campaign A (ABO) — "Testing"
  ├── Ad Set 1 — 新受众 A ($50/天)
  ├── Ad Set 2 — 新受众 B ($50/天)
  └── Ad Set 3 — 新素材 C ($50/天)

Campaign B (CBO) — "Scaling"
  ├── Ad Set 1 — 已验证受众 A
  ├── Ad Set 2 — 已验证受众 B
  └── 3-5 条已验证广告
```

### ASC 设置清单

1. **选目标**：Sales → Conversions → Purchase
2. **选 Advantage+ Shopping Campaign**
3. **设置预算**：建议日预算 ≥ 历史最佳 Ad Set 预算的 2-3 倍
4. **设置建议受众（非强制）**：
   - 过去 180 天购买者自定义受众
   - 过去 180 天网站访客
   - 客户名单上传
5. **上传素材**：至少 5-10 个创意变体（图片 + 视频混合）
6. **设置广告文案**：准备 3-5 个文案变体
7. **选择出价策略**：初期用 Lowest Cost，稳定后切 Cost Cap
8. **发布并观察**：给 7-14 天学习期，不要频繁改

### ASC 效果评估标准

| 指标 | 达标线 | 优秀线 |
|---|---|---|
| ROAS | ≥ 2.5x | ≥ 4x |
| CPA | ≤ 历史手动广告 CPA 的 80% | ≤ 历史手动广告 CPA 的 60% |
| 新客占比 | ≥ 50% | ≥ 65% |
| 预算消耗率 | ≥ 95%（预算能花完） | 100% |

## 7. Lookalike Audience 创建

### 种子受众质量决定 LAL 质量
- **核心结论**：种子人数 ≥100（最低），推荐 ≥1000。种子质量：Purchase 客户 > AddToCart > Page Likes
- **操作步骤**：
  1. 准备种子受众（客户文件上传 / 网站自定义受众 / Page 互动受众）
  2. 受众 → 创建 Lookalike → 选种子 → 选国家
  3. 百分比设置：
     - 1% = 最精准，约 200 万人（美国）
     - 2-3% = 平衡精准和规模
     - 5-10% = 拓量阶段用
  4. 创建后等 6-24h 生效
- **判断标准**：LAL 1% 的 CPA 通常比 Broad 低 30-50%
- **避坑点**：种子太小（<100）= LAL 不稳定；种子太杂（混合多种行为）= LAL 不精准
- **来源**：madgicx.com, balistro.com, pixis.ai

## 8. 预算分配与 CBO 策略

### 三层预算架构

| 层级 | 占比 | 用途 | 出价策略 |
|------|-----|------|---------|
| 勘探预算（Prospecting） | 60-70% | 获取新客户 | Lowest Cost / ASC |
| 再营销预算（Retargeting） | 30-40% | 转化高意向用户 | Target Cost / Cost Cap |
| 测试预算 | 10-15%（从各层出） | 新受众/创意测试 | Lowest Cost |

### 关键结论

1. **使用CBO的广告主CPA平均降低12%** — 算法比人工更懂如何分配预算
2. **勘探 vs 再营销预算比例 = 60-70% : 30-40%** — 再营销ROAS（3.61）远高于勘探（2.11）
3. **每日手动调整超过1次会导致ROAS下降15%** — 频繁干预会破坏算法学习

### CBO vs ABO 选择

| 场景 | 推荐 |
|------|------|
| 3个以上Ad Sets | ✅ 用CBO |
| 预算<$10K/天 | ✅ 用CBO |
| 账户刚过学习期 | ✅ 用CBO |
| 单个Ad Set预算测试 | ❌ 不用CBO（用ABO） |

### 避坑点
- ⚠️ 避免每天手动调整预算超过1次（会破坏算法学习，导致ROAS下降15%）
- ⚠️ 不要用Lifetime Budget做测试阶段（新Campaign还没数据，算法无法智能分配）
- ⚠️ 受众重叠会让多个广告互相抬价，浪费预算
- ⚠️ 勘探和再营销必须严格分开——用排除功能防止受众重叠
- ⚠️ 预算调整幅度不要超过20%（超过会重置学习期）

```
    ├── Ad 1: UGC_Video_01
    └── Ad 2: Image_01
```
- **运行节奏**：7-14 天不碰，只看数据
- **评估节点**：Day 7 看 CTR，Day 14 做首次优化
- **适合**：新账户冷启动、新品测试

```
Campaign A: [PURCHASE]_Testing_$150/day (ABO)
```

- **运行节奏**：测试组跑 4-7 天，表现好的复制到 Scaling
- **评估节点**：每周评估一次
- **适合**：有一定数据积累，开始扩量

## 12. 上线前最终检查清单

> 每次发布或大改 Campaign 前，逐项检查。漏一项可能浪费一周预算。

### 基础设施

- [ ] Pixel 已安装且正常回传（用 Pixel Helper 验证）
- [ ] CAPI 已启用且 Event Match Quality ≥ 6/10
- [ ] 域名已验证（Business Settings → Brand Safety → Domains）
- [ ] Aggregated Event Measurement 已配置，Purchase 为优先级 1
- [ ] 支付方式有效（信用卡未过期）

### Campaign 层级

- [ ] 目标选对（Sales 不是 Traffic）
- [ ] 预算模式选对（CBO vs ABO）
- [ ] 预算金额 ≥ 目标 CPA × 3
- [ ] 出价策略正确（初期 Lowest Cost）

### Ad Set 层级

- [ ] 受众大小 ≥ 10 万人
- [ ] 没有和其他 Campaign 受众重叠 > 30%
- [ ] 版位设为 Advantage+ Placements
- [ ] 排除已购买用户（再营销除外）
- [ ] 每个 Ad Set 至少有 2-3 条素材

### Ad 层级

- [ ] 素材尺寸适配版位（1:1, 9:16）
- [ ] 文案含明确 CTA
- [ ] 落地页 URL 加了 UTM 参数
- [ ] 链接能正常打开（移动端测试）
- [ ] 落地页加载 < 3 秒
- [ ] 落地页内容与广告一致（避免审核拒登）

### 合规

- [ ] 广告图片无 Before/After 对比
- [ ] 文案无绝对化用语（"100%"、"Guaranteed"）
- [ ] 落地页无虚假促销/隐藏费用
- [ ] 不 targeting 个人属性（"你胖吗"）
- [ ] 受限品类已查政策并合规

## 13. 常见错误结构及修正

### 错误 1：受众碎片化

**问题结构：**
```
Campaign A: LAL-1% ($20/day)
Campaign B: Interest-Sports ($20/day)
Campaign C: Interest-Tech ($20/day)
Campaign D: Broad ($20/day)
Campaign E: LAL-3% ($20/day)
```
**问题**：5 个 Campaign 互相竞争，每个都拿不到足够数据过学习期。
**修正**：合并到 1-2 个 Campaign：
```
Campaign A: [PURCHASE]_Prospecting_$100/day (CBO)
├── Ad Set 1: LAL-1%
├── Ad Set 2: Interest-Sports+Tech
├── Ad Set 3: Broad
└── Ad Set 4: LAL-3%
```

### 错误 2：单素材 Campaign

**问题结构：**
```
Campaign → Ad Set → Ad（仅 1 条广告）
```
**问题**：没有任何测试空间，素材差就全军覆没。
**修正**：每个 Ad Set 至少 3 条素材变体。

### 错误 3：再营销没有排除购买者

**问题结构：**
```
Retargeting Ad Set → 30d Website Visitors
```
**问题**：已购买用户仍在看广告，浪费预算且可能引起反感。
**修正**：
```
Retargeting Ad Set → 30d Website Visitors
                        └── Exclude: 365d Purchasers
```

### 错误 4：CBO 里 Ad Set 太多

**问题结构：**
```
Campaign (CBO) → 10+ Ad Sets（每个 $10-20/day）
```
**问题**：CBO 会把预算集中在 1-2 个 Ad Set，其余几乎不花。
**修正**：CBO 里放 2-4 个已验证的 Ad Set 即可。

### 错误 5：多 Campaign 投同一受众

**问题结构：**
```
Campaign A: Broad_US
Campaign B: Broad_US（不同素材）
```
**问题**：拍卖重叠，自己抬自己的 CPM。
**修正**：合并到同一 Campaign，用 Post ID 导入不同素材到同一 Ad Set。
