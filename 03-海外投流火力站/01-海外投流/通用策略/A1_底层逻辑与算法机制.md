# A1 底层逻辑与算法机制

## 一句话总结
Meta广告竞拍机制、学习期原理、Advantage+ AI体系、Pixel/CAPI双端数据回传

## 核心结论
- Meta 广告是**价值拍卖**而非**出价拍卖**——总价值 = 出价 × 预估行动率 × 广告质量，低出价+高质量可以打败高出价+低质量
- 学习期是算法的"探索阶段"，需要 7 天内 50 次优化事件才能进入稳定态，期间数据波动 30-50% 是正常的
- Advantage+ 代表 Meta 从"人工精细控制"到"AI 黑盒优化"的范式转移，当转化数据 >1000/月时 ASC 通常优于手动
- Pixel+CAPI 双端部署将数据覆盖率从 ~55% 提升到 ~95%+，Event Match Quality ≥6 是 CPA 优化的前置条件
- iOS 14.5 ATT 后，"素材即定向"成为核心逻辑——宽受众+好素材 > 精准定向+差素材

## 适用场景
- 适合平台：Facebook/Meta Ads Manager（Instagram/Audience Network 同理）
- 适合行业：DTC 电商、B2B 询盘、SaaS、教育、高客单价服务、本地服务
- 适合场景：新账户冷启动、投放策略选择、ASC vs 手动决策、Pixel/CAPI 部署诊断

## 投流要素拆解
- 平台：Facebook/Meta
- 核心机制：价值拍卖（Total Value = Bid × Estimated Action Rate × Ad Quality）
- 学习期门槛：50 转化/7 天
- 数据追踪：Pixel（浏览器）+ CAPI（服务器）双端回传
- 归因窗口：7 天点击为默认，1 天浏览可选
- 关键指标：CPM、CTR、CVR、CPA、ROAS、Event Match Quality
- 可复用策略：宽定向 + 好素材 > 精确定向；ASC 适合 50+ 转化/周的账户
- 风险点：频繁调整重置学习期、iOS 信号丢失、受众自竞争
- 适合复用：[[食品加工机械]]B2B 询盘、[[教育培训]]留资、[[韩国医美]]高客单转化

## 可复用方法
1. **拍卖价值公式法**：用 Total Value = Bid × EAR × Quality 诊断广告——CPA 高不一定是出价问题，先看质量排名
2. **学习期预算计算器**：最低日预算 = (50 × 目标 CPA) ÷ 7，预算不够就降级优化事件
3. **ASC 决策树**：转化 <50/周 → 手动CBO；50-200/周 → ASC+手动并行；>200/周 → 70% 预算给 ASC
4. **事件降级阶梯**：Purchase → InitiateCheckout → AddToCart → ViewContent → Link Click，逐级降直到有足够数据
5. **信号质量 Checklist**：8 项参数传递确保 Event Match Quality ≥6，包括 hashed email/phone/fbc/fbp/IP/UA/地址

## 对我的业务有什么价值
- 对跨境贸易的价值：B2B 食品机械外贸获客理解学习期原理，避免因学习期数据波动误关广告；Pixel+CAPI 确保询盘数据完整回传
- 对 Facebook 投流的价值：理解价值拍卖机制，不再盲目提高出价；通过优化素材质量和相关度来降低 CPA
- 对巨量本地推的价值：巨量的 oCPM 与 Meta 的价值拍卖逻辑相通，学习期的"50 转化"原理可直接迁移
- 对客户开发的价值：理解归因窗口和数据差异，正确解读 BM 数据 vs Shopify 后台的差距，做出正确优化决策
- 对知识库沉淀的价值：作为投流知识体系的基础文件，所有投放策略、数据优化、素材方法论的底层逻辑来源

## 1. Meta 广告竞拍机制

### 1.1 广告排名公式（2025 版）

**总价值得分 = 出价 × 预估行动率 × 广告质量 + 广告相关度加分**

- **出价（Bid）**：你愿意为一次转化付多少钱，不等于实际扣费（实际扣费 = 下一名总价值 ÷ 你的预估行动率）
- **预估行动率（Estimated Action Rate）**：AI 预估该用户看到广告后完成目标转化的概率
- **广告质量（Ad Quality）**：基于用户反馈（隐藏/举报/互动）+ 落地页质量综合评分
- **广告相关度**：广告内容与目标受众的匹配程度

**关键结论**：不是出价最高就能赢。高质量 + 高相关度的广告可以用更低出价赢得展示。

### 知识点：出价不是唯一决定因素
- **核心结论**：出价低但质量高的广告可以赢得竞拍。Meta 优先展示"用户最可能互动"的广告，而非"广告主最愿意付钱"的。
- **判断标准**：如果你的 CTR 持续高于行业均值，CPM 会逐步下降
- **适用场景**：所有广告投放
- **避坑点**：不要只靠提高出价来解决展示量问题，先检查素材质量和受众匹配度
- **来源**：facebook.com/business

### 知识点：广告质量排名（Quality Ranking）
- **核心结论**：Meta 对每条广告给出三个诊断指标：Quality Ranking / Engagement Rate Ranking / Conversion Rate Ranking，均为"广告 vs 同受众竞品"的相对排名
- **操作步骤**：
  1. 打开 Ads Manager → 自定义列 → 添加 Competitive Metrics
  2. 看 Quality Ranking 是否在 Average 以上
  3. 如果 Below Average，说明素材/落地页体验差
- **判断标准**：三个指标都 Above Average = 优质广告；任一 Below Average = 需优化
- **来源**：facebook.com/business

### 1.2 价值模型（Value Model）

Meta 的价值模型有两层优化：

| 层级 | 说明 |
|---|---|
| **广告竞价层** | 决定谁能赢展示机会 |
| **转化价值层** | 预测转化后带来的价值（购买金额、LTV 等），Advantage+ Shopping Campaign 尤其依赖此层 |

### 1.3 信号学习与数据反馈循环

- 每次广告展示 → 收集信号（展示/点击/停留/转化）
- AI 模型用这些信号更新**预估行动率**
- 转化越多 → 模型越准 → CPA 越低 → 正向循环
- 反之：转化太少 → 模型无法学习 → 陷入"学习期受限"状态

## 2. 学习期原理

### 知识点：学习期的触发与退出条件
- **核心结论**：新建或大幅修改的 Ad Set 进入学习期，需在 7 天内积累 **50 次优化事件**（如 Purchase/AddToCart）才能退出。退出后算法进入"稳定态"
- **判断标准**：
  - 事件数 ≥ 50/周 = 可退出学习期
  - 事件数 < 50/周 = 持续学习中，数据波动大属正常
- **适用场景**：所有新 Campaign / 新 Ad Set / 预算大幅调整（>20%）
- **避坑点**：
  - ❌ 学习期内频繁调预算/出价 → 重置学习期
  - ❌ 用不稳定的优化事件（如 ViewContent 当目标，太容易触发导致数据噪音大）
  - ✅ 学习期内不要动任何设置，耐心等 4-7 天
- **来源**：facebook.com/business, jonloomer.com

### 知识点：学习期的数据解读
- **核心结论**：学习期内的 CPM/CPA/CVR 波动高达 30-50%，这是正常的。不要在学习期内做优化决策
- **操作步骤**：在 Ads Manager 里看 "Delivery" 列，状态显示 "Learning" 或 "Active"
- **判断标准**：等 Ad Set 状态变为 "Active" 且稳定 3 天后，再根据数据优化
- **来源**：jonloomer.com

### 什么会重置学习期（Significant Edits）
| 会重置 | 不会重置 |
|--------|---------|
| 大幅调整预算（> 20-30%） | 小幅预算调整（< 20%） |
| 更换广告素材 | 暂停/开启单条广告 |
| 改变受众定向 | 添加新广告到现有 Ad Set |
| 更换优化事件 | 改广告文案/标题（小幅） |
| 改变版位 | 改 CBO 预算（可能影响所有 Ad Set） |

### 如何确保退出学习期？

| 你的目标 CPA | 最低日预算（7天50转化） | 建议 |
|-------------|----------------------|------|
| $10 | $72/天 | 可行 |
| $20 | $143/天 | 可行 |
| $30 | $214/天 | 需要一定预算 |
| $50 | $357/天 | 预算压力大 |
| $100 | $714/天 | 考虑优化更浅层事件 |

**公式**：最低日预算 = (50 × 目标 CPA) ÷ 7

## 3. Advantage+ 的 AI 自动化原理

### 3.1 什么是 Advantage+？

Advantage+ 是 Meta 在 2023-2024 年推出的一系列 AI 驱动的自动化投放工具：

| 产品 | 功能 | 适用场景 |
|---|---|---|
| **Advantage+ Shopping Campaigns (ASC)** | 全自动电商转化广告，AI 负责定向、出价、素材分配 | 电商独立站（Shopify 等），有 Pixel 数据支撑 |
| **Advantage+ Audience** | AI 自动扩展受众，替代手动定向 | 想要放量但手动定向容易疲劳的广告组 |
| **Advantage+ Creative** | 自动优化素材格式、音乐、剪裁 | 多素材混合投放时 |
| **Advantage+ App Campaigns** | 全自动 App 安装/活跃推广 | App 推广 |
| **Advantage+ Placements** | 自动版位分配 | 所有 Campaign（默认开启） |

### 知识点：Advantage+ Shopping Campaigns (ASC) 工作原理
- **核心结论**：ASC 是 Meta 的 AI 驱动广告类型，自动组合受众 + 素材 + 版位 + 出价，减少人工决策。它利用你已有的 Pixel 数据和转化历史来寻找高概率购买者
- **操作步骤**：
  1. 创建 ASC → 选择销售目标
  2. 上传 6-15 条素材（视频优先）
  3. 设定日预算（建议 ≥$100/天）
  4. **前 7-10 天不要干预**
  5. 之后只换素材，不改受众/出价
- **判断标准**：
  - 适合：日预算 >$100、已有 Pixel 数据（>50次转化）、素材 ≥6 条
  - 不适合：全新无数据账户、预算太小、素材不足
- **避坑点**：ASC 需要大量数据喂养，新账户冷启动不建议直接上 ASC
- **来源**：jonloomer.com, bir.ch

### 3.2 ASC 与传统手动广告的本质区别

| 维度 | 传统手动广告 | Advantage+ Shopping |
|---|---|---|
| 定向 | 手动选兴趣/自定义/相似 | AI 自动选择，可设建议受众但非强制 |
| 出价 | 多种出价策略可选 | 主要是最低成本 / 成本上限 |
| 素材 | 一个广告组放 1-3 个广告 | 最多放 150 个素材，AI 自动组合测试 |
| 学习期 | 每个广告组独立学习 | 整组共享学习，数据聚合更快 |
| 透明度 | 高（可精确控制） | 低（黑盒化，需信任 AI） |

### 知识点：Advantage+ vs 手动 Campaign 的选择阈值
- **核心结论**：当你有 **≥1000 次转化事件/月** 且 **日预算 ≥$100** 时，ASC 通常优于手动设置
- **决策树**：
  - 转化事件 <50/周 → 手动 CBO
  - 50-200/周 → 可以尝试 ASC，同时保留手动
  - >200/周 → 优先 ASC，用 70% 预算
- **来源**：jonloomer.com, askneedle.com

### ASC 适用条件
- 电商产品≥50个SKU
- 日预算 ≥ $100（给算法足够学习空间）
- 已配置CAPI和Pixel（确保归因准确）

### ASC 不适用场景
- 高单价决策周期长产品（如房产/汽车）
- B2B询盘类业务（转化事件稀疏）
- 受众极小的细分市场

## 4. Pixel + CAPI 双端数据回传

### 4.1 为什么必须同时配 Pixel + CAPI？

| 数据源 | 工作原理 | 数据丢失率 | 优势 |
|---|---|---|---|
| **Pixel（浏览器端）** | JS 追踪，依赖浏览器 Cookie | iOS 14.5+ 下丢失 30-50% | 实时、含用户行为数据 |
| **CAPI（服务器端）** | 服务器直接回传转化事件给 Meta | 丢失 < 5% | 不受 ITP/ATT 影响 |

**最佳实践**：两者同时部署 → Meta 自动去重 → 数据覆盖率从 ~55% 提升到 ~95%+

### 知识点：Pixel 和 CAPI 的区别与互补
- **核心结论**：
  - **Pixel（浏览器端）**：JS 代码在用户浏览器执行，追踪点击、浏览等行为。iOS 14.5+ 后被 Safari/WebKit ITP 限制
  - **CAPI（服务端）**：从你的服务器直接发送事件到 Meta，不受浏览器限制
  - **两者结合**：Meta 自动去重（event_id 匹配），数据更完整
- **操作步骤**：
  1. 先部署 Pixel（8步清单见 [[02-海外投流/账户策略/A2_账户结构与出价策略]]）
  2. 在 Events Manager → 设置 → CAPI → 选择"手动安装"或"GTM"
  3. 确保每条事件都有 `event_id`（用于去重）
  4. 设置事件优先级：Purchase > AddToCart > InitiateCheckout > ViewContent
- **判断标准**：CAPI 部署后，Events Manager 里看到 "Browser" 和 "Server" 双渠道数据，且去重后总数 > 纯 Pixel 数据
- **避坑点**：不要只部署 CAPI 不部署 Pixel，两者互补效果最佳
- **来源**：developers.facebook.com

### Event Match Quality（事件匹配质量）
- Meta 用 0-10 分评估事件匹配用户的准确度
- **关键参数**：email、phone、client_user_agent、fbc、fbp、client_ip_address、first_name、last_name、city、state、zip、country
- **目标分数**：≥ 6.0 为良好，≥ 8.0 为优秀
- **影响**：匹配质量越高 → 算法越精准 → CPA 越低

### 4.2 事件优先级设置（Aggregated Event Measurement）

iOS 14.5 后，每个域名最多只能优化 8 个事件。必须设置优先级：

1. **Purchase**（购买）→ 优先级 1
2. **Initiate Checkout**（发起结账）→ 优先级 2
3. **Add to Cart**（加入购物车）→ 优先级 3
4. **View Content**（查看内容）→ 优先级 4
5. 其他事件按业务重要性排列

### 4.3 CAPI 部署方式对比

| 方式 | 技术难度 | 成本 | 推荐场景 |
|---|---|---|---|
| **Shopify 原生集成** | ⭐ 零配置 | 免费 | Shopify 卖家首选 |
| **Zapier/Make 中转** | ⭐⭐ 低 | $20-50/月 | 非 Shopify 独立站 |
| **服务器直接开发** | ⭐⭐⭐⭐ 高 | 开发成本 | 自建站/定制化需求 |
| **Partner 方案（Stape 等）** | ⭐⭐ 中 | $10-30/月 | 通用独立站，推荐性价比方案 |

### 知识点：iOS 14.5+ 信号丢失与应对
- **核心结论**：iOS 14.5 后用户可选择不追踪（ATT），导致约 30-50% 的 iOS 用户数据丢失。CAPI 可找回约 10-15%
- **操作步骤**：
  1. 确保已验证域名并设置 Aggregated Event Measurement（最多8个优先事件）
  2. 部署 CAPI 补充服务端数据
  3. 关注 "建模转化"（Modeled Conversions）— Meta 用 ML 推算丢失数据
  4. 增加 First-Party Data（客户文件上传、网站注册数据）
- **来源**：facebook.com/business, triplewhale.com

## 5. iOS 14.5+ ATT 影响后的应对框架

### 核心影响
- IDFA 获取率从 ~70% 降至 ~10-15%
- 归因窗口从 28 天缩短至 7 天点击 / 1 天浏览
- 自定义受众覆盖缩小 30-50%
- 再营销效果下降

### 应对策略
1. **升级 CAPI**：弥补浏览器端数据丢失
2. **一触数据（First-Party Data）**：通过 Lead Form、客服对话收集客户数据
3. **扩大受众池**：从精准定向转为宽定向，让 AI 找人群
4. **内容驱动再营销**：用内容互动（视频观看、IG 互动）替代 Pixel 再营销
5. **多渠道归因**：结合 GA4、Triple Whale 等三方工具做交叉验证

## 5. 拍卖机制深度解析

### 5.1 实际扣费公式（Second-Price Auction 变体）

Meta 不是"你出多少付多少"。实际扣费基于第二名竞争者的总价值：

```
你的实际 CPC = (第二名总价值 ÷ 你的预估行动率) + $0.01
```

**示例**：
- 你的出价：$2.00，预估行动率：5%，质量分：8
- 你的总价值 = 2.00 × 0.05 × 8 = 0.80
- 第二名总价值 = 0.60
- 你的实际扣费 = (0.60 ÷ 0.05) + 0.01 = $12.01（每次行动）

**关键结论**：提高质量分比提高出价更有效——质量分翻倍，实际扣费可能降低 40%。

### 5.2 广告质量分的三个子维度

| 子维度 | 权重 | 影响因素 | 在哪看 |
|--------|------|---------|--------|
| **Quality Ranking** | 40% | 用户负面反馈（隐藏/举报） | Ads Manager → Competitive Metrics |
| **Engagement Rate Ranking** | 30% | 点赞/评论/分享/视频观看 | Ads Manager → Competitive Metrics |
| **Conversion Rate Ranking** | 30% | 点击后转化率 | Ads Manager → Competitive Metrics |

### 5.3 质量分与 CPM 的关系

```
高质量分广告（Top 20%）：CPM 比均值低 15-30%
低质量分广告（Bottom 20%）：CPM 比均值高 25-50%
```

**提升质量分的 5 个杠杆**：
1. 素材与受众精准匹配（避免"广撒网"）
2. 落地页加载速度 <3 秒（GTmetrix 测试）
3. 减少"标题党"Hook（点击率高但跳出率也高会降低质量分）
4. 鼓励正向互动（评论、分享、保存）
5. 避免负面信号（隐藏率 < 0.1%，举报率 < 0.01%）

## 6. Advantage+ 算法 internals

### 6.1 ASC 的 AI 决策链路

```
输入信号
  ├── Pixel 历史转化数据（谁买了、买了什么、什么时候买）
  ├── 当前素材表现（各素材的 CTR/CVR/Thumb-Stop）
  ├── 市场竞争态势（当前时段 CPM/CPC 水平）
  ├── 用户实时行为（最近 7 天的浏览/互动/搜索模式）
  └── 季节性/周期性信号（节假日、促销季、品类周期）
        ↓
AI 决策
  ├── 定向决策：从全量用户中筛选高概率转化人群
  ├── 素材决策：为每个用户选择最可能点击的素材版本
  ├── 版位决策：Feed / Reels / Stories / Audience Network 最优组合
  ├── 出价决策：根据竞争强度实时调整实际出价
  └── 频次决策：对每个用户控制展示次数，避免疲劳
        ↓
输出结果
  └── 每次展示都是一次独立的 AI 决策
```

### 6.2 ASC vs 手动广告的性能差异

| 指标 | 手动 CBO | ASC | 差异 |
|------|---------|-----|------|
| CPA（电商） | $15-30 | $12-25 | ASC 低 15-20% |
| 学习期时长 | 7-14 天 | 5-10 天 | ASC 快 30% |
| 素材利用率 | 30-40% | 60-80% | ASC 高 2x |
| 人工调优频率 | 每天 1-2 次 | 每周 1-2 次 | ASC 省 80% 时间 |
| 放量天花板 | 中 | 高 | ASC 可放大量 |

### 6.3 ASC 冷启动加速策略

```
Day 1-3：观察期
  - 不做任何调整
  - 检查素材数量 ≥ 6 条
  - 确认 Pixel 数据正常回传

Day 4-7：评估期
  - 如果 CPA > 目标 2x：检查素材质量（3s 留存率 > 40%？）
  - 如果 CPA 在目标 1-1.5x：继续观察
  - 如果 CPA < 目标：准备加预算

Day 8-14：优化期
  - 替换表现最差的 20% 素材
  - 如果 CPA 稳定：预算 +20%
  - 每周补充 2-3 条新素材
```

## 7. Pixel 事件追踪进阶配置

### 7.1 标准事件优先级矩阵

| 优先级 | 事件 | 适用场景 | 优化目标 |
|--------|------|---------|---------|
| P0 | Purchase | 电商 | 直接优化购买 |
| P1 | Lead | B2B/服务 | 优化表单提交 |
| P2 | AddToCart | 测试期（数据不足时） | 优化加购行为 |
| P3 | InitiateCheckout | 高客单价 | 优化结账发起 |
| P4 | ViewContent | 内容/教育 | 优化内容消费 |
| P5 | CompleteRegistration | SaaS/App | 优化注册完成 |

### 7.2 事件降级策略（解决数据不足）

当深层事件（Purchase）周转化 < 50 时，按以下阶梯降级：

```
Purchase（周 < 50 次）
  ↓
InitiateCheckout（周 < 100 次）
  ↓
AddToCart（周 < 200 次）
  ↓
ViewContent / Lead Form
  ↓
Landing Page Views（最后手段）
```

**降级原则**：
- 降级事件必须与最终转化有强关联（加购→购买的转化率 > 10%）
- 降级后积累的 Pixel 数据仍可帮助 AI 学习用户特征
- 达到 50 次深层事件/周后，立即切回深层优化

### 7.3 Pixel 参数传递最佳实践

```javascript
// Purchase 事件标准参数
fbq('track', 'Purchase', {
  content_ids: ['SKU123', 'SKU456'],
  content_type: 'product',
  value: 89.99,        // 必须包含运费
  currency: 'USD',     // 必须与广告账户一致
  num_items: 2,        // 购买件数
  event_id: 'unique_id_123'  // CAPI 去重必需
});
```

**关键参数要求**：
- `value`：必须包含运费和税费，不能只含商品金额
- `currency`：必须与广告账户货币一致，否则 ROAS 计算错误
- `event_id`：Pixel 和 CAPI 必须使用相同的 ID 才能去重
- `content_ids`：用于 Dynamic Ads 再营销，必须与 Product Catalog 匹配

### 8.1 多渠道信号补偿矩阵

| 渠道 | 补偿比例 | 成本 | 延迟 | 准确度 |
|------|---------|------|------|--------|
| Pixel（浏览器） | 50-60% | 免费 | 实时 | 高 |
| CAPI（服务器） | 30-40% | $10-30/月 | 实时 | 高 |
| 建模转化 | 5-10% | 免费 | 24-48h | 中 |
| 三方归因（Triple Whale 等） | 交叉验证 | $100-300/月 | 实时 | 中 |
| GA4 交叉验证 | 交叉验证 | 免费 | 24-48h | 中 |
| 客户文件上传 | 5-15% | 人工 | 手动 | 高 |

### 8.2 数据信任优先级

```
当不同数据源出现冲突时：
1. CAPI + Pixel 去重后数据（最可信）
2. 广告管理器内显示数据（含建模转化）
3. GA4 数据（注意归因窗口差异）
4. 第三方归因工具（参考用）
5. 独立站后台订单数据（最准确但不包含广告归因）
```

**MER 作为最终校准器**：
- MER = 总收入 ÷ 总广告花费
- 当 MER 稳定时，即使广告后台数据有偏差，整体盈利能力是可信的
- MER 与 ROAS 差异 > 20% → 需要排查数据源

## 9. 算法优化的 7 个高阶技巧

### 9.1 利用"广告组拍卖重叠"优化

当同一账户内多个 Ad Set 竞争同一用户时，Meta 会内部"竞价"，只展示一个。结果：
- 自我竞争推高自己的 CPM
- 分散转化信号，延长学习期

**解决方案**：
- Ad Set 之间受众重叠 < 20%
- 或用 CBO 自动分配预算（减少自我竞争）
- 合并相似受众的 Ad Set

### 9.2 预算分配的"时间窗口效应"

Meta 算法在不同时段的学习效率不同：
- **0:00-6:00**：流量低谷，学习慢，适合低预算
- **6:00-12:00**：流量上升，学习加速
- **12:00-18:00**：流量高峰，竞争加剧但转化也高
- **18:00-24:00**：流量峰值，CPM 最高但转化意愿最强

**应用**：
- 新广告上线选在 6:00-9:00，利用全天学习窗口
- 不要在 22:00 后上线新广告（浪费 6 小时学习）
- 预算调整选在凌晨，避开学习高峰期

### 9.3 素材多样性对算法的影响

Meta 算法需要素材多样性来探索新用户：
- 同一 Ad Set 放 3-5 条不同风格的素材
- 算法会自动为不同用户分配不同素材
- 避免所有素材使用同一 Hook 类型

**素材多样性检查**：
- Hook 类型 ≥ 3 种（数字钩/问题钩/反常识钩）
- 视觉风格 ≥ 2 种（实拍 + 图文/动画）
- 时长 ≥ 2 种（6s 短 + 15-30s 标准）
