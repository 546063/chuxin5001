# 获客内容方法论总路由

## 一句话定位

本系统用于解决"内容如何带来有效咨询、询盘、私信、加微信、WhatsApp、成交机会"的问题。

## 一、这个系统解决什么问题

本系统不是单纯的律师获客知识库，而是**通用获客内容方法论**。律师短视频获客课程只是本系统的原始课程来源与应用样本之一。

核心解决以下问题：
- 内容发布后播放高但没有咨询/询盘/私信
- 不知道什么样的内容能筛选出好成交客户
- 不知道如何建立信任促进成交
- 不清楚内容策略是否跑通，是否需要调整方向

## 二、通用获客底层方法论（优先调用）

> 所有行业、所有平台的获客内容，都应先走这两篇底层方法论。

- [[10-通用内容获客方法论：从优质用户到成交]] — 从定义优质用户到内容成交的完整链路
- [[11-通用获客复盘系统：用五条内容判断用户、内容和产品是否跑通]] — 用最小成本验证用户-内容-产品匹配度

通用方法论适用于：装修获客、教育培训获客、消杀获客、美业获客、食品机械 B2B 获客、外贸询盘、本地服务咨询、私域转化、短视频客资内容等。

## 四、调用规则

### 规则 1：通用获客问题

用户问"怎么获客 / 怎么提高咨询 / 怎么提高询盘 / 怎么提高私信 / 怎么让客户加微信 / WhatsApp 询盘怎么提升"

→ 先调用：
- [[10-通用内容获客方法论：从优质用户到成交]]
- [[11-通用获客复盘系统：用五条内容判断用户、内容和产品是否跑通]]

### 规则 2：律师获客问题

用户问"律师账号怎么做 / 律师文案怎么写 / 法律服务怎么获客"

→ 先调用：
- [[00-获客内容方法论总路由]]
再进入律师获客应用模块（01-08）。

### 规则 3：其他行业获客迁移

用户问"装修、教育培训、消杀、美业、食品机械、B2B 外贸、本地服务怎么做获客内容"

→ 先调用通用获客底层方法论（10、11），再根据行业场景迁移：
- 优质用户定义
- 排他性痛点
- 高价值需求
- 产品前置
- 信任建立
- 复盘迭代

### 规则 4：播放高但咨询少

→ 先调用复盘系统：
- [[11-通用获客复盘系统：用五条内容判断用户、内容和产品是否跑通]]
再配合诊断清单：
- [[07-律师获客文案诊断清单]]（非律师行业只借用诊断维度，不照搬律师表达）

### 规则 5：写一条能带来咨询的文案

→ 先判断：
- 目标优质用户是谁
- 用户处于什么阶段
- 排他性痛点是什么
- 高价值需求是什么
- 产品是否前置
- 信任元素是否足够
- 行动引导是否清晰

## 五、不适合调用的场景

- 用户需要具体的广告投放 SOP（应去投流知识库）
- 用户需要具体的销售话术（应去客户沟通复盘）
- 用户需要平台操作教程（应去对应平台 SOP）

## 六、与主库其他获客文案系统的关系

本系统与主库中 `04-内容素材弹药库/获客文案方法论` 形成互补：
- **获客内容方法论**（本系统）：侧重"内容如何获客"的战略层、定位层、选题层、诊断层
- **获客文案方法论**：侧重"文案如何成交"的执行层、转化层、话术层、链路层

两者结合使用：先用本系统确定内容方向和选题，再用获客文案方法论优化具体文案表达和转化链路。
