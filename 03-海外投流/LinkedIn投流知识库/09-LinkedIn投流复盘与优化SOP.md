# LinkedIn投流复盘与优化SOP

## 一句话总结

LinkedIn 优化看四个数字：CPC → CTR → CVR → SQL率，三步定位问题环节，6 棵决策树覆盖所有常见问题。

## 核心结论

- LinkedIn 与 Facebook 优化逻辑不同：LinkedIn 关注 CPC 和 SQL 率，Facebook 关注 CPM 和 CVR
- 80% 的问题出在受众太宽或素材不专业
- CPL 偏高 = CPC 高 或 CTR 低 或 CVR 低，先定位是哪个环节
- SQL 率低 = 受众不精准或表单字段太少

## Step 0：先问 4 个数字

```
CPC = ?
CTR = ?
CVR = ?
SQL率 = ?

CPC > 正常值 30%+   → 问题在竞争层（受众太宽/出价太高/质量分低）
CPC 正常，CTR 低    → 问题在点击层（素材不专业/受众不匹配）
CPC 和 CTR 正常     → 问题在转化层（落地页/表单/CTA）
CVR 正常，SQL率低   → 问题在受众质量（字段太少/定向太宽）
```

核心公式：`CPL = CPC ÷ CVR`

## 决策树 A：不起量（消耗 < 预期 30%）

```
广告不消耗 / 消耗极慢
├── 1. 查审核状态 → Not Approved → 修复 → 申诉
├── 2. 查出价 → 手动CPC太低 → 调高或切自动出价
├── 3. 查受众规模 → LinkedIn预估受众 < 5万人 → 扩展行业/职位
├── 4. 查预算 → 日预算 < 目标CPL × 3 → 提高预算
├── 5. 查定向组合 → AND条件太多 → 减少交叉限制
└── 6. 查支付方式 → 付款失败 → 更新
```

**第一优先级动作**：确认审核状态和受众规模。LinkedIn 受众池比 Facebook 小很多，过度交叉定向会导致受众过小。

## 决策树 B：成本高（CPL > 目标 30%+）

```
CPL 偏高
├── Step 1：查 CPC → 涨 30%+ → 排查竞争层（受众竞争/出价/质量分）
├── Step 2：查 CTR → < 0.5% → 排查点击层（素材不专业/受众不匹配/频次>4）
└── Step 3：查 CVR → < 10% → 排查转化层（表单太长/CTA不清/落地页问题）
```

## 决策树 C：CTR 低（< 0.5%）

```
CTR 偏低
├── 检查运行时长 → 跑了 3 周+ → 素材疲劳 → 换新素材
├── 检查频次 → Frequency > 4 → 扩展受众
├── 检查素材 → 不够专业/不像LinkedIn风格 → 重做素材
├── 检查受众 → 职位/行业不匹配 → 重新定向
└── 检查版位 → 右侧/底部版位CTR低 → 仅投信息流
```

CTR 基准：优秀 > 1%，正常 0.5-1%，偏低 0.3-0.5%，危险 < 0.3%

## 决策树 D：转化率低（CVR < 10%）

```
CVR 偏低
├── 表单太长（> 7字段） → 减少到 3-5 个
├── CTA 不清晰 → 改为明确行动指令
├── 落地页加载慢（> 3秒） → 优化速度
├── 表单与广告不匹配 → 确保广告承诺与表单一致
└── 移动体验差 → 测试移动端表单
```

## 决策树 E：素材疲劳

**疲劳判断（满足任一即需换）**：
- Frequency > 4
- CTR 连续 5 天下降 > 15%
- CPL 连续 5 天上涨 > 20%
- 跑了 4-6 周以上

```
素材疲劳
├── 立即动作：关掉疲劳广告，保留预算，新建广告组
├── 换素材策略：保留赢的框架，只改一个维度，一次上 2-3 条新素材
└── 预防措施：高预算+小受众 4-6 周换一轮，中等预算 6-8 周
```

## 决策树 F：SQL 率低（< 20%）

```
SQL率偏低
├── 受众太宽 → 增加Seniority限制（Manager+）
├── 行业不匹配 → 缩小到Food & Beverages / Food Production
├── 表单字段太少 → 增加1-2个 qualifying 问题
├── 排除Entry级 → 排除Entry/Training/Unpaid
├── 公司规模不匹配 → 按产品定位调整公司规模
└── 线索响应太慢 → 5分钟内首次触达
```

**这是 LinkedIn 特有的决策树** —— Facebook 没有 SQL 率这个维度，但 LinkedIn 必须关注，因为线索质量直接影响销售效率。

## 30 秒快速诊断卡

| 顺序 | 指标 | 正常范围 | 异常动作 |
|------|------|---------|---------|
| 1 | 审核状态 | Approved/Running | Not Approved → 立即申诉 |
| 2 | 预算消耗率 | 70-100% | < 50% → 查出价/受众规模 |
| 3 | CPL | < 目标值 | > 目标 30% → 走决策树 B |
| 4 | Frequency | 1-4 | > 4 → 准备换素材 |
| 5 | SQL率 | > 25% | < 20% → 走决策树 F |

**原则：前 2 个数字正常 + 后 3 个数字正常 = 今天不需要做任何操作。**

## 关停判断标准

满足以下**任意 2 条**即关停：
- 花费 > 目标 CPL × 3 且 0 转化
- CTR < 0.3% 持续 5 天
- CPL > 目标 × 2 持续 7 天
- Frequency > 6 且 CTR 下降 > 25%
- 跑了 30 天 + SQL率 < 10%

## 行业基准对照

| 指标 | LinkedIn（食品机械） | LinkedIn（全行业） | Facebook（食品机械） |
|------|---------------------|-------------------|---------------------|
| CPC | $3-8 | $5-10 | $0.5-2 |
| CTR | 0.5-1.2% | 0.4-0.8% | 1-2% |
| CVR | 10-25% | 5-15% | 5-15% |
| CPL | $25-50 | $50-100 | $5-20 |
| CPL（S级案例） | $139 | - | $8-20 |
| SQL率 | 25-40% | 15-25% | 10-20% |

## 操作节奏

| 频率 | 可以做的操作 |
|------|-------------|
| 每天 | 看CPC/审核状态/预算消耗率（不动根） |
| 每周 | 关疲劳广告、上 1-2 条新素材、调整预算 ±20%、检查受众重叠 |
| 每月 | 全面复盘、关低效Ad Group、扩展高效Ad Group、评估SQL率 |

### 绝对不要做的
- 一天内多次调整广告（影响算法学习）
- 预算一次加超过 30%（LinkedIn对大幅调整敏感）
- 前7天内做重大改动（给算法时间）
- 用 CTR 低于 0.3% 的素材继续跑

## 四维度评分卡

| 维度 | 5分（优秀） | 3分（及格） | 1分（危险） |
|------|------------|------------|------------|
| CPL | < 目标 60% | 目标 60-100% | > 目标 |
| CTR | > 1% | 0.5-1% | < 0.3% |
| SQL率 | > 35% | 20-35% | < 15% |
| 预算消耗 | 80-100% | 60-80% | < 60% |

**总分 16-20**：优秀 → 加预算扩量
**总分 12-15**：及格 → 找失分维度，针对性优化
**总分 8-11**：危险 → 按诊断 SOP 排查
**总分 < 8**：紧急 → 建议停掉重做

## 与 Facebook 优化核心差异

| 维度 | Facebook 优化 | LinkedIn 优化 |
|------|--------------|--------------|
| 核心指标 | CPM → CTR → CVR | CPC → CTR → CVR → SQL率 |
| 学习期 | 50转化/7天 | 无明确概念，但前7天关键 |
| 素材更换频率 | 7-14天 | 14-30天 |
| 预算调整幅度 | ±20-50% | ±20-30% |
| 受众规模要求 | > 10万人 | > 5万人 |
| 质量判断 | CVR/ROAS | SQL率/成交率 |
| 优化重点 | 素材+落地页 | 受众+素材 |
