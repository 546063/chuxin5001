# 投流真实资料导入流程

> **适用范围**：所有投流真实资料从用户准备到证据层归档的完整流程。
> **核心原则**：不编造、不泄露、不混放。

---

## 1. 用户准备资料

用户需要准备以下类型的资料：

| 资料类型 | 来源 | 说明 |
|----------|------|------|
| 广告后台截图 | Facebook/TikTok/LinkedIn Ads Manager | Campaign/Ad Set 级别数据面板 |
| 广告数据 CSV/Excel | Ads Manager 导出 | 花费、展示、点击、CPL 等指标 |
| 素材截图 | 信息流中看到的广告 | 图片广告、视频广告、轮播 |
| 竞品广告截图 | Meta Ad Library / TikTok Creative Center | 竞品投放的广告素材和文案 |
| WhatsApp 聊天截图 | 手机端截图 | 客户询盘、报价、成交沟通 |
| Messenger 聊天截图 | Facebook Messenger | 广告带来的私信咨询 |
| 客户报价记录 | 报价单/合同 | PDF、截图或文本 |
| 成交记录 | 付款确认/合同 | 确认成交的证据 |
| 未成交原因 | 客户反馈/内部记录 | 丢单原因分析 |
| 客户反馈 | 聊天记录/邮件 | 客户对产品/服务的评价 |

## 2. 文件命名规范

### 广告后台数据 CSV/Excel

```
platform_country_industry_product_metrics_YYYYMMDD.csv
```

- `platform`: facebook / tiktok / linkedin / google
- `country`: sa / us / ae / ph（ISO 3166-1 alpha-2）
- `industry`: food-machine / packaging / textile
- `product`: dumpling-machine / meat-grinder
- 示例：`facebook_saudi_foodmachine_metrics_20260522.csv`

### 广告后台截图

```
platform_country_industry_product_dashboard_YYYYMMDD_001.png
```

- 示例：`facebook_saudi_foodmachine_dashboard_20260522_001.png`

### 素材截图

```
platform_country_industry_product_creative_YYYYMMDD_001.png
```

- 示例：`facebook_saudi_foodmachine_creative_20260522_001.png`

### 竞品广告截图

```
platform_country_industry_product_competitor_YYYYMMDD_001.png
```

- 示例：`facebook_saudi_foodmachine_competitor_20260522_001.png`

### 客户聊天截图

```
platform_country_product_leadchat_YYYYMMDD_001_sensitive.png
```

- `_sensitive` 表示含隐私信息，必须先脱敏
- 示例：`facebook_saudi_dumplingmachine_leadchat_20260522_001_sensitive.png`

## 3. 导入步骤

```
1. 用户把资料放入导入箱：
   20-原始资料与下载暂存/投流证据导入箱/
   ↓
2. Claude Code 扫描导入箱中的文件
   ↓
3. 判断文件类型（图片/CSV/Excel/PDF）
   ↓
4. 检查文件名是否含 _sensitive 标记
   ↓
5. 需要脱敏的文件：
   - 标注敏感字段位置
   - 执行脱敏（姓名/手机号/邮箱/地址）
   - 生成脱敏版本
   ↓
6. 数据文件处理：
   - 读取 CSV/Excel 数据
   - 分配 data_id（DATA-YYYYMMDD-001）
   - 写入投流真实数据记录表
   - 更新 CSV 文件
   ↓
7. 素材截图处理：
   - 移动到 20-原始资料与下载暂存/投流素材截图/
   - 分配 creative_id（CREATIVE-YYYYMMDD-001）
   - 写入广告素材截图索引
   ↓
8. 竞品截图处理：
   - 移动到 20-原始资料与下载暂存/投流素材截图/
   - 分配 competitor_id（COMP-YYYYMMDD-001）
   - 写入竞品广告素材截图索引
   ↓
9. 客户聊天处理：
   - 脱敏后归档
   - 分配 lead_id（LEAD-YYYYMMDD-001）
   - 分配 customer_id（Customer-A）
   - 写入客户转化记录库
   - 写入询盘跟进记录库
   ↓
10. 关联到真实案例：
    - 如果有对应 Campaign，分配 case_id（CASE-YYYYMMDD-001）
    - 关联到投流真实案例库
    ↓
11. 更新总路由：
    - 更新路由 JSON 中的案例数/数据量
    - 更新证据层总入口的统计汇总
```

## 4. 禁止事项

| 禁止 | 原因 |
|------|------|
| 编造数据 | 证据层的核心价值是真实性 |
| 保留完整客户联系方式 | 违反隐私保护原则 |
| 把未脱敏聊天截图嵌入公开文档 | 泄露客户隐私 |
| 把客户隐私上传公开仓库 | 法律风险 |
| 把模拟数据放入真实数据表 | 混淆数据性质 |
| 把竞品素材直接照搬成自有素材 | 版权侵权 |

---

## 5. 导入箱状态

| 状态 | 说明 |
|------|------|
| 空箱 | 无待处理文件 |
| 有待处理 | 有文件等待扫描处理 |
| 处理中 | Claude Code 正在处理 |
| 已清空 | 处理完毕，导入箱已清空 |

> 导入箱是**临时中转站**，处理完毕后文件会被归档到对应位置。
