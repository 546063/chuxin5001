# 采购决策链

## 一句话定义

> **采购决策链 = 在一个采购决定中，不同角色各自扮演什么角色 = 找对人比说对话重要100倍**

---

## 为什么采购决策链重要

> [!warning] 核心问题
> 很多人做外贸失败，是因为：
> - 找错了人（跟一个没有决定权的人聊了3个月）
> - 不知道谁说了算
> - 只跟中间人转达，不跟决策人直接聊
>
> **结果：浪费大量时间，客户说"我再问问老板"就消失了**

---

## 决策链中的角色

### 1. 决策人 (Decision Maker)

| 角色 | 说明 | 特点 |
|------|------|------|
| 老板/CEO | 最终掏钱的人 | 直接决策，中小公司 |
| 采购总监 | 预算和供应商选择 | 大公司 |
| 投资人 | 大额采购需要批准 | 周期长 |

**怎么识别：**
- 问"这个价格你能决定吗？"
- 问"要不要问其他人？"
- 看对话语气（老板通常直接）

### 2. 影响者 (Influencer)

| 角色 | 说明 | 特点 |
|------|------|------|
| 工程师 | 技术评估，说"能不能用" | 关心参数、认证 |
| 使用部门 | 说"我需要什么功能" | 关心操作便捷 |
| 财务 | 说"预算够不够" | 关心价格 |
| 运营 | 说"效率怎么样" | 关心ROI |

**怎么识别：**
- 问很多技术问题
- 问认证、测试报告
- 关心细节

### 3. 中间人 (Gatekeeper)

| 角色 | 说明 | 特点 |
|------|------|------|
| 采购专员 | 收集信息，不决策 | 传话、过滤 |
| 秘书/助理 | 安排会议，筛选 | 很难见到真正决策人 |
| 贸易商 | 中间转手，不负责 | 可能截胡 |

**怎么识别：**
- 总是说"我要问一下"
- 不直接回答问题
- 问你很多问题但不谈订单

### 4. 使用者 (User)

| 角色 | 说明 | 特点 |
|------|------|------|
| 操作工 | 实际使用设备 | 关心操作难度 |
| 维修工 | 负责维修保养 | 关心维护方便 |
| 厨师/技术员 | 实际操作 | 关心功能 |

**怎么识别：**
- 问操作问题
- 问功能问题
- 关心使用体验

---

## 识别决策人的方法

### 方法1：直接问

> [!tip] 直接问
> "请问您这边采购是您自己决定，还是需要其他人批准？"
>
> "这个项目大概预算是多少，需要几个人同意？"

### 方法2：看对话内容

| 对话内容 | 可能的角色 |
|---------|-----------|
| "价格多少？MOQ多少？" | 可能是决策人 |
| "技术参数能发吗？" | 可能是影响者 |
| "我要问一下老板" | 不是决策人 |
| "客户需求是什么？" | 可能是中间人 |

### 方法3：看公司规模

| 公司规模 | 决策人 |
|---------|--------|
| 小型（<10人） | 老板直接决策 |
| 中型（10-50人） | 老板或采购经理 |
| 大型（50+人） | 采购委员会，多人决策 |

---

## 决策链图示

```
小型公司（老板直接管）：

老板 ← 决定价格、付款、订单
↑
客户问价格 → 老板回答 → 下单

中型公司（采购经理管）：

老板 ← 批准预算 ← 最终决策
↑
采购经理 ← 询价、比价、谈判
↑
工程师 ← 技术评估
↑
客户问价格 → 采购经理 → 传话给老板

大型公司（采购委员会）：

CEO/CFO ← 最终批准（大额）
↑
采购总监 ← 选择供应商
↑
采购专员 ← 收集信息
↑
工程师 ← 技术评估
↑
使用者 ← 提供需求
↑
客户 → 不知道找谁（危险！）
```

## 客户为什么不成交

### 情况1：跟中间人聊了很久

> [!warning] 问题
> 跟一个采购专员聊了2个月，他说他跟老板说了，但老板没反应。
>
> **真相：采购专员可能根本没有把你的方案传给老板**

**怎么解决：**
- 直接问：能不能约老板聊？
- 看对话：采购专员是不是一直说"我跟老板汇报一下"？
- 施压：如果2周内没有回复，就说"那我看有没有其他客户在等"

### 情况2：客户说"我再想想"

> [!warning] 问题
> 客户问完价格，说"我再想想"，然后就消失了。
>
> **真相：可能不是真的在想，而是在问别人**

**怎么解决：**
- 问：你还跟谁在讨论这个项目？
- 问：有什么我需要解释给其他人听的？
- 给：给他发总结邮件，让他在会上转发

### 情况3：工程师说"可以"但老板说"太贵"

> [!warning] 问题
> 工程师已经测试了设备，说"没问题"，但老板说价格太高。
>
> **真相：技术通过了，但商务没通过**

**怎么解决：**
- 早介入：让工程师帮你说话
- 给ROI：让工程师能跟老板算账
- 分批：先小单，建立信任后再大单

## 不同场景的决策链

### 场景1：墨西哥小进口商（1-5人公司）

```
老板（也是采购员）→ 直接决定
```

**策略：**
- 跟老板直接聊
- 快速响应，WhatsApp
- 价格要灵活，可以谈
- 老板一个人说了算

### 场景2：德国中型食品厂（50-200人）

```
老板 → 批准（大额）
↑
运营总监 → 评估效率
↑
采购经理 → 询价、比价
↑
工程师 → 技术评估
```

**策略：**
- 找采购经理，但要让他帮你见到工程师
- 技术资料要专业，让工程师认可
- 大额订单要让老板看到ROI

### 场景3：美国连锁餐厅（100+门店）

```
总部采购委员会 → 最终决定
↑
区域经理 → 推荐
↑
门店经理 → 提出需求
```

**策略：**
- 从区域经理开始
- 要进入供应商库很难，要时间
- 一旦进入，量稳定
- 标准化是关键

## 快速判断决策链的方法

### 问题1：问决策

> "您这边采购设备，是您自己决定，还是要其他人批准？"

- **"我自己决定"** → 决策人，可以聊
- **"我需要问一下"** → 不是决策人，要找别人
- **"要看金额"** → 问多大金额需要批准

### 问题2：问流程

> "您这边采购流程是什么？一般要几个人同意？"

- **"问一下老板就行"** → 老板说了算
- **"要走招标流程"** → 大公司，周期长
- **"要采购委员会批准"** → 多人决策，要时间和关系

### 问题3：问时间

> "您希望什么时候用到这个设备？"

- **"越快越好"** → 决策快，真实需求
- **"3-6个月后"** → 可能是半年后才决定
- **"不着急"** → 不是真需求

## 实战话术

### 识别决策人

**问：**
> "跟您沟通很顺畅。请问这个项目，除了您之外，还有谁参与决策？我怕遗漏了需要提供信息的人。"

**客户答1：**
> "就我说了算，我直接下单。"
> → 决策人，明确，快速推进

**客户答2：**
> "我要问一下老板，他最后批准。"
> → 老板是最终决策人，想办法见到老板

**客户答3：**
> "我们有采购委员会，要评估。"
> → 大公司，需要时间和关系介入

### 让中间人帮你

**如果对方是采购专员：**
> "您这边负责收集信息很好。不过这个项目最终是谁决定？能不能让我直接跟他沟通一下技术细节？这样可以节省您的时间。"

**如果对方是工程师：**
> "您这边技术评估没问题了。我想问一下，老板那边最关心什么？我来帮他准备ROI计算，这样您跟他汇报的时候更方便。"

---

## 总结

> [!success] 采购决策链核心
> 1. **找对人** - 跟没有决定权的人聊浪费时间
> 2. **识别角色** - 决策人、影响者、中间人、使用者
> 3. **直接问** - "您自己能决定吗？"
> 4. **帮助中间人** - 让他帮你见到决策人
> 5. **给决策人需要的** - 工程师要技术，老板要ROI
