# Engineer（工程师）

## 一句话定义

> **Engineer = 评估设备技术可行性的人 = 说"能用"才能过第一关**

工程师不管价格，只管技术参数能不能满足工艺要求。

---

## 工程师分类

| 类型 | 说明 | 特点 |
|------|------|------|
| 工艺工程师 | 评估设备能不能满足生产要求 | 最关键 |
| 设备工程师 | 评估设备性能和维护 | 关心MTBF |
| 电气工程师 | 评估电气系统、控制面板 | 关心PLC、安全 |
| 质量工程师 | 评估食品安全、材料合规 | 关心认证 |

---

## 他真正关心什么

### 1. 技术参数

> [!tip] 核心关注
> 工程师最关心：**这个设备能不能完成我的工艺要求？**
>
> - 速度、精度、能力
> - 材料、尺寸、功率
> - 能不能配合现有产线

### 2. 可靠性

> - 设计寿命多久
> - MTBF多少
> - 主要部件品牌
> - 故障率多少

### 3. 维护

> - 维护周期多久
> - 备件通用性
> - 诊断功能
> - 远程支持

---

## 他会问什么问题

> - "切菜精度是多少？误差多少？"
> - "切割速度能到多少？"
> - "用的是什么PLC？"
> - "材料是什么？食品级吗？"
> - "设计寿命多少小时？"
> - "MTBF是多少？"
> - "电气系统符合什么标准？"
> - "有控制面板吗？能接入我们系统吗？"

---

## 他最怕什么

> [!warning] 工程师最怕的 5 件事

| 恐惧 | 说明 |
|------|------|
| 设备不达标 | 用不了，要重新买 |
| 质量问题 | 食品污染风险 |
| 维护复杂 | 频繁故障，难维修 |
| 参数虚假 | 说明书写的和实际不一样 |
| 不兼容 | 和现有系统对接不了 |

---

## 他最容易被什么打动

| 打动点 | 说明 |
|--------|------|
| **详细技术文件** | 规格表、图纸、测试报告 |
| **参数真实** | 实测数据，不是"约等于" |
| **部件清单** | 用了什么品牌型号 |
| **测试验证** | 第三方测试报告 |
| **案例数据** | 同行用的效果数据 |

---

## 怎么和工程师沟通

### 提供的资料

| 资料 | 说明 |
|------|------|
| 完整规格表 | 所有参数，误差范围 |
| CAD图纸 | 尺寸、接口 |
| 电气原理图 | 控制逻辑 |
| 材料证书 | 食品接触材料合规 |
| 测试报告 | 第三方认证 |
| 操作手册 | 使用和维护 |

### 沟通话术

**客户问参数：**
> "Our slicer achieves:
> - Cutting speed: 60-120 slices/min (adjustable)
> - Slice thickness: 1-30mm (accuracy ±0.5mm)
> - Belt speed: 5-20m/min (adjustable)
> - Material: SUS304 stainless steel
>
> These parameters are verified by our in-house testing and third-party inspection report from [testing lab]."

**客户问PLC：**
> "We use Siemens S7-1200 PLC, same as European brands. The control system supports:
> - Ethernet/IP communication
> - PROFIBUS if required
> - Remote monitoring via cloud
> - Integration with existing SCADA systems"

**客户担心质量：**
> "Our quality control:
> - Incoming material inspection
> - In-process quality checks
> - Final testing before shipment
> - We provide batch test reports for each production run
> - Third-party inspection available (SGS/BV)"

---

## 成交关键

> [!success] 让工程师认可的关键
> 1. **参数要真实** - 不要虚标，工程师会验证
> 2. **文件要完整** - 规格表、图纸、测试报告
> 3. **部件要透明** - 用什么品牌要说清楚
> 4. **案例要有数据** - 有客户用的实际数据
> 5. **测试要能验证** - 允许客户测试样品
