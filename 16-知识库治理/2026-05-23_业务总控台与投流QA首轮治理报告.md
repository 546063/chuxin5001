# 业务总控台与投流QA首轮治理报告

## 本轮目标

把用户提出的六个方向先收敛成可执行入口：

- 整理 Obsidian 知识库
- 搭建业务总控台
- 做投流知识库 QA 系统
- 做食品机械 B2B 增长系统
- 维护 AI 工具链
- 为本地项目开发和修复建立入口

## 扫描结论

- 当前 vault 约有 1755 个 Markdown 文件。
- 已存在较完整的总指挥部、MOC、投流总路由、Facebook/TikTok/LinkedIn/巨量/自然流量知识库。
- 食品机械、客户开发、客户沟通、销售复盘、成交案例等模块已有基础，但入口之间仍需要业务级串联。
- 当前 git 工作区已有大量既有改动，本轮不做批量移动、删除、去重和格式化。

| 文件 | 用途 |
|---|---|
| [[业务总控台]] | 把海外投流、短视频获客、食品机械、客户成交、案例复盘接成业务作战地图 |
| [[投流知识库QA系统]] | 建立“问题 -> 路由 -> SOP -> 案例 -> 检查清单”的投流问答入口 |
| [[食品机械B2B增长系统]] | 建立食品机械选品、素材、投流、承接、成交、复盘闭环 |

## 暂缓事项

| 事项 | 暂缓原因 | 下一步 |
|---|---|---|
| 批量修 wikilink | 需要先跑断链报告，避免误链 | 生成 broken link dry-run |
| 去重和归档 | 当前存在大量历史归档和同名文档，需要人工确认主版本 | 先列候选重复组 |
| 大规模 frontmatter 统一 | 之前已有 YAML 修复记录，继续批量改风险较高 | 先抽样检查字段 |
| 项目代码修复 | 需要逐项目读取 README 和测试命令 | 下一阶段选择一个项目开工 |

## 下一轮建议

1. 按 [[2026-05-23_断链去重DryRun治理报告]] 先修旧目录 MOC 和明确总入口链接。
2. 对重复标题做主版本判断，先处理入口类和高频 SOP 类重复。
3. 对 `claude-to-im` 或 `food-machinery-hero` 选一个项目做代码级体检。
4. 每周更新 [[投流知识库QA系统]]，把新问题沉淀成路由，而不是散落在聊天记录。
