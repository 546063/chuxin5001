# 输出格式总模板

## 标准输出结构

每次回答业务问题，必须包含以下部分：

```markdown
## 问题分类
[判断问题属于哪个业务场景]

## 调用模型
- [[XX模型]]：为什么调用这个模型
- [[XX模型]]：为什么调用这个模型
- [[XX模型]]：为什么调用这个模型

## 综合判断
[用调用的模型思考后得出的判断]

## 行动建议
1. [具体可执行的步骤 1]
2. [具体可执行的步骤 2]
3. [具体可执行的步骤 3]

## 需要补充的信息
[回答这个问题的缺失信息，建议用户补充]

## 风险提醒
[可能的风险和需要注意的点]
```

## 不同场景的特殊要求

### 选品判断
必须输出：目标客户、市场容量、竞品分析、差异化机会、最小测试方案

### 跨境B2B获客
必须输出：目标客户画像、渠道选择、内容方向、私信路径、WhatsApp转化路径

### 投流判断
必须输出：账户结构建议、素材方向、预算分配、数据判断标准、优化节奏

### 文案创作
必须输出：目标受众、核心信息、钩子设计、CTA、平台适配建议

### WhatsApp成交
必须输出：客户分层、跟进节奏、话术框架、信任建设材料、转化路径

### 老板决策
必须输出：核心矛盾、能力圈评估、风险边界、最小行动建议、止损线

## 格式要求

- 用 markdown 格式
- 用表格对比选项
- 用列表列步骤
- 用引用标注关键结论
- 所有数据必须标注来源
