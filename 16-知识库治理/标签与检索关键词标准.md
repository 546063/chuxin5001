# 标签与检索关键词标准

生成时间：2026-05-16
主控角色：iMac 主控 Agent

---

## 一句话解释

标签和关键词必须精准，不能泛泛。AI 检索靠这些词定位文档。

---

## 标签标准

### 禁止使用的泛标签

以下标签过于宽泛，禁止单独使用：
- `#Facebook` — 太宽泛
- `#投流` — 太宽泛
- `#客户` — 太宽泛
- `#外贸` — 太宽泛
- `#AI` — 太宽泛
- `#复盘` — 太宽泛
- `#食品机械` — 太宽泛
- `#LinkedIn` — 太宽泛
- `#WhatsApp` — 太宽泛

### 必须使用的细分标签格式

**格式**：`#场景/动作` 或 `#产品/场景/动作`

**原则**：
- 场景 + 动作 + 决策价值
- 细分到具体业务动作
- 让 AI 能精准定位

## retrieval_keywords 标准

### 要求

1. **关键词必须能帮助 AI 精准检索**
2. **不要只写单个大词**
3. **优先写业务问题词、场景词、动作词、产品词、客户异议词**
4. **S/A 级建议 8-12 个，B/C 级 5-8 个，D 级 3-5 个**

### 好关键词示例

| 类型 | 关键词示例 |
|------|-----------|
| 业务问题词 | WhatsApp价格异议处理、LinkedIn首轮私信话术 |
| 场景词 | 食品机械B2B客户开发、Facebook线索广告投放 |
| 动作词 | 客户需求挖掘、询盘判断、成交推进 |
| 产品词 | 果蔬清洗机海外客户、肉类加工设备选型 |
| 客户异议词 | 客户嫌价格高、客户要求折扣、客户不回复 |
| 方法论词 | 周度销售复盘、话术优化、数据分析 |

### 坏关键词示例

| 类型 | 问题 |
|------|------|
| 单个大词 | Facebook、投流、客户、外贸 |
| 太泛 | AI、知识库、复盘 |
| 无业务意义 | 测试、学习、研究 |

## 标签数量标准

| 质量级别 | 标签数量 | 说明 |
|----------|----------|------|
| S级 | 3-5 个 | 核心标签 + 场景标签 |
| A级 | 2-4 个 | 场景标签 + 动作标签 |
| B级 | 1-3 个 | 基础标签 |
| C级 | 0-1 个 | 可选 |
| D级 | 0 个 | 不添加 |
