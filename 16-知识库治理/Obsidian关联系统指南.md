## 概述
本指南介绍 Obsidian vault 的完整关联系统，包括 MOC、标签、关键词、元数据、脚注和相关笔记索引。

## 系统架构

### 核心 MOC 文件
1. **[[04-跨境选品与产品/MOC]]** - 跨境社媒出海、外贸选品、海关数据
2. **[[01-记忆与配置/MOC]]** - 记忆管理、长期记忆、每日记忆
3. **项目 MOC** - 项目管理、银羽、博哥IP、鲸驰、跨境项目
4. **技能 MOC** - 技能目录、工具链、技术栈
5. **运营方法论 MOC** - 运营方法论、私域转化、社群运营

### 关联管理文件
1. **标签索引** - 标签分类、使用指南、热门标签
2. **关键词关联** - 关键词网络、关联规则、核心关键词
3. **元数据模板** - 元数据标准、模板、使用指南
4. **脚注与补充说明** - 脚注系统、补充说明层、使用示例
5. **相关笔记索引** - 相关笔记索引、关联规则、维护指南

## 使用流程

### 1. 创建新笔记
1. 使用 元数据模板 添加元数据
2. 从 标签索引 中选择合适的标签
3. 在 关键词关联 中登记关键词
4. 在相关 MOC 中添加链接

### 2. 关联现有笔记
1. 检查 相关笔记索引 寻找关联
2. 建立双向链接
3. 更新相关 MOC 文件
4. 添加补充说明和脚注

### 3. 维护关联系统
1. 每月检查一次关联关系
2. 更新过时的链接
3. 清理死链
4. 优化关联网络

## 最佳实践

### 标签使用
- 每个笔记至少使用1-2个核心标签
- 使用层级标签如 `#跨境社媒/选品`
- 保持标签一致性
- 定期整理和合并相似标签

### 关键词管理
- 在笔记中突出显示核心关键词
- 建立关键词之间的层级关系
- 使用双向链接关联相关笔记
- 定期更新关键词网络

### 元数据规范
- 必填字段：title、created、tags
- 推荐字段：type、status、related
- 使用统一的元数据格式
- 及时更新元数据

### 脚注和补充说明
- 使用统一的脚注格式
- 保持脚注简洁明了
- 及时更新过时的脚注
- 建立脚注之间的关联

## 示例工作流

### 创建跨境选品分析笔记
1. 创建新笔记 `智能锁选品分析.md`
2. 添加元数据（参考 元数据模板）
3. 添加标签：`#跨境社媒`、`#跨境选品`、`#智能锁`、`#分析`
4. 在 [[04-跨境选品与产品/MOC]] 中添加链接
5. 在 关键词关联 中登记关键词
6. 在 相关笔记索引 中添加关联
7. 添加脚注说明数据来源和方法

### 更新现有笔记关联
1. 打开需要更新的笔记
2. 检查 相关笔记索引 寻找新关联
3. 添加新的双向链接
4. 更新元数据中的 related 字段
5. 添加补充说明和脚注
6. 更新相关 MOC 文件

## 维护计划

### 每日维护
- 检查新笔记的关联关系
- 更新当日的记忆笔记

### 每周维护
- 检查所有 MOC 文件的完整性
- 更新 相关笔记索引
- 整理标签系统

### 每月维护
- 全面检查关联系统
- 清理死链和过时内容
- 优化关联网络结构
- 更新系统指南文档

## 故障排除

### 常见问题
1. **找不到相关笔记**：检查 相关笔记索引 和 关键词关联
2. **标签混乱**：参考 标签索引 的标签分类
3. **元数据不一致**：使用 元数据模板 统一格式
4. **关联关系复杂**：使用 Obsidian 图形视图可视化

### 解决步骤
1. 确定问题类型
2. 参考相关指南文件
3. 按照最佳实践操作
4. 记录解决方案供后续参考
