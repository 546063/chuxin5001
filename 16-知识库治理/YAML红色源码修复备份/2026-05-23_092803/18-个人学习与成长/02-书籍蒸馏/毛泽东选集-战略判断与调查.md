# 《毛泽东选集》｜战略判断、调查研究与实践闭环工作模型

## 1. 一句话结论

《毛泽东选集》对我最有价值的地方，不是照搬具体历史语境，而是学习一种工作方法：先调查真实情况，再判断主要矛盾，然后用实践验证判断，最后把经验重新沉淀成方法。用在今天的跨境社媒、一人公司和做人做事里，就是不要靠感觉决策，不要靠情绪行动，不要靠别人说法判断市场，而要从客户、平台、同行、供应链、成交反馈中找事实。

## 2. 这本书最值得我学的内容

第一，调查研究。很多人失败不是因为不努力，而是没有搞清楚事实就开始努力。做跨境社媒也是一样，不能只凭“我觉得这个内容会火”“我觉得这个产品好卖”“我觉得这个客户会买”。必须先看客户真实问题、同行真实打法、平台真实反馈、询盘真实质量。

第二，矛盾分析。一个项目表面上有很多问题：内容不好、播放少、询盘少、报价难、客户不回复、供应链不稳定。但真正要抓的是主要矛盾。比如一个账号不成交，主要矛盾可能不是内容数量不够，而是客户信任不足；也可能不是信任不足，而是选题吸引了错误人群；也可能不是人群错误，而是私信承接没有标准话术。

第三，实践闭环。正确认知不是想出来的，而是做出来、测出来、复盘出来的。短视频脚本、广告素材、落地页、WhatsApp话术都不能只停留在理论层面，必须发布、测试、看数据、看客户反应，再回到知识库更新判断。

第四，反对本本主义。书可以学，课程可以学，方法论可以学，但不能把任何方法当成固定答案。跨境社媒不同平台、不同国家、不同产品、不同客单价，打法都不同。真正成熟的做法，是把书里的方法变成自己的“问题分析工具”，而不是变成口号。

## 3. 核心模型提炼

### 3.1 调查研究模型

调查研究不是简单收集资料，而是为了找到真实矛盾。对你来说，调查研究至少包括五类：

1. 客户调查：客户是谁、在哪里、为什么买、怕什么、比较什么、什么时候下单。  
2. 平台调查：Facebook、TikTok、LinkedIn、YouTube、小红书、抖音各自适合什么内容和承接路径。  
3. 同行调查：同行怎么做内容、怎么展示案例、怎么报价、怎么引导客户咨询。  
4. 产品调查：产品核心卖点、应用场景、交付周期、售后风险、认证和案例。  
5. 成交调查：哪些客户最终成交，成交前问了什么，没成交的人卡在哪里。

调查研究的关键不是“资料越多越好”，而是找出能够影响决策的事实。比如客户是否真的在意价格？还是更在意售后？客户是否真的看设备参数？还是更看工厂视频和真实案例？

### 3.2 主要矛盾模型

任何项目都要先找主要矛盾。跨境社媒项目常见主要矛盾包括：

- 需求不成立：内容吸引的人没有真实购买需求。  
- 信任不足：客户看到了你，但不相信你能交付。  
- 渠道不通：平台不适合这个产品或人群。  
- 表达错误：你讲的是自己想讲的，不是客户想知道的。  
- 承接断裂：内容有人看，但没有私信路径、询盘表单、WhatsApp话术。  
- 产品不清：没有讲清楚适用场景、规格、服务和差异。  
- 供应链不稳：前端拿到客户，后端无法稳定报价和交付。

主要矛盾不是固定的，它会随着阶段变化而变化。冷启动阶段主要矛盾可能是内容方向；有流量后主要矛盾可能是询盘质量；有询盘后主要矛盾可能是信任证据；有成交后主要矛盾可能是交付和复购。

### 3.3 实践—认识—再实践模型

这个模型可以直接变成你的工作流：

1. 先形成假设：这个客户群可能关心某个问题。  
2. 做最小内容：用一条短视频、一篇帖子、一组广告素材测试。  
3. 看真实反馈：播放、停留、评论、私信、询盘、客户问题。  
4. 复盘判断：哪里验证了，哪里错了，哪里不确定。  
5. 修正方法：更新脚本库、话术库、案例库、投流规则。  
6. 再次实践：用更新后的方法继续测试。

这套模型对一人公司非常重要。因为你资源有限，不能用大公司打法烧钱试错，只能用更短的反馈链路降低试错成本。

## 4. 跟我工作的连接

### 4.1 用在跨境社媒

跨境社媒最容易犯的错误是把“发内容”当成工作本身。真正的工作应该是：通过内容验证客户需求，通过互动发现客户疑虑，通过数据识别内容方向，通过私信和询盘测试成交路径。

你可以把每一条内容都当成一次调查：

- 这个标题是否吸引目标客户？  
- 评论区出现了哪些真实问题？  
- 私信客户问的是价格、规格、认证、交期，还是案例？  
- 哪类内容带来的是同行围观，哪类内容带来的是采购咨询？

如果内容没有带来有效信息，就算播放高，也只是热闹；如果内容能暴露客户疑虑，即使播放不高，也有战略价值。

### 4.2 用在一人公司

一人公司不能追求“大而全”。你要像做运动战一样，快速识别机会，快速测试，快速转向。最重要的是建立自己的小型情报系统：

- 每天记录客户问题。  
- 每周记录高潜选题。  
- 每月记录成交和未成交原因。  
- 每个项目记录主要矛盾变化。  
- 每个失败动作记录错误假设。

一人公司不是没有组织，而是组织要轻。你的知识库、AI工作流、内容库、客户库，就是你的组织能力。

### 4.3 用在做人做事

做人做事也要避免“没调查就判断”。很多冲突来自误判：误判对方动机、误判事情难度、误判自己能力、误判合作成本。更成熟的做法是先搞清事实，再判断矛盾，再决定行动。

遇到复杂关系时，可以问：

- 我看到的是事实，还是情绪？  
- 当前主要矛盾是什么？  
- 对方真正想要什么？  
- 我能给什么，不能给什么？  
- 这件事有没有必要投入更多精力？

## 5. 可以直接使用的判断问题

### 项目启动前

1. 我有没有做足调查，还是只是在凭感觉？  
2. 这个项目的主要矛盾是什么？  
3. 如果只能解决一个问题，应该先解决哪一个？  
4. 当前信息够不够支持投入资源？  
5. 有没有更小的测试动作？

### 内容发布前

1. 这条内容是在解决客户问题，还是在表达我自己？  
2. 它验证哪个假设？  
3. 它希望客户产生什么动作？  
4. 它能不能产生评论、私信、收藏、询盘？  
5. 发布后我看什么指标？

### 复盘时

1. 哪个判断被事实证明是对的？  
2. 哪个判断被事实推翻？  
3. 客户真实反馈和我原来的想象有什么差异？  
4. 下一步要加大投入，还是停止？  
5. 这次经验应该沉淀到哪个知识库模块？

## 6. 主用模型

### 模型一：主要矛盾优先级模型

每个项目只抓一个当前最关键矛盾。不要同时解决十个问题。比如一个账号没有询盘，先判断是流量问题、内容问题、人群问题、信任问题还是承接问题。抓错矛盾，越努力越乱。

### 模型二：调查研究四步法

事实收集 → 分类整理 → 找矛盾 → 做验证。不要一边收集资料一边兴奋地行动，先把事实整理成可判断的结构。

### 模型三：实践闭环模型

假设 → 小测试 → 数据 → 复盘 → 修正 → 再测试。这是你做内容、投流、客户开发、AI知识库治理都应该使用的底层流程。

## 7. 备选模型

### 统一战线迁移模型

在商业里可以理解为资源协同：客户、工厂、渠道、内容平台、服务商、AI工具，不一定都要由你控制，但你要让它们围绕同一个目标协同。

### 根据地模型

对一人公司来说，“根据地”不是地盘，而是稳定可复用的资产：知识库、案例库、客户库、脚本库、话术库、数据复盘库。没有根据地，就只能每次从零开始。

### 群众路线迁移模型

不是你想讲什么，而是客户真正关心什么；不是你觉得高级什么，而是客户能听懂什么；不是你认为专业什么，而是客户愿意为哪个问题付钱。

## 8. 不建议强行套用的部分

《毛泽东选集》的历史语境非常特殊，不能把政治斗争、军事斗争、阶级斗争的语言直接搬到商业沟通里。你需要学的是方法：调查、矛盾、实践、复盘、组织，而不是把表达方式原样套用到客户、同行或合作伙伴身上。

商业世界里，过度斗争化会伤害合作。你要用它提升判断力，而不是让自己变得攻击性过强。

## 10. 我的行动版总结

《毛泽东选集》给我的最大启发是：不要闭门造系统，不要凭感觉做市场，不要被表面问题牵着走。所有内容、投流、客户开发、合作判断，都要回到事实、矛盾和实践。对我最实用的一句话可以改写成：没有调查，就没有发言权；没有测试，就没有决策权；没有复盘，就没有增长权。

## 11. 放进知识库后的使用方法

这篇文稿不要当成普通读书笔记使用，而要当成一个“决策前置检查器”。以后遇到新项目、新账号、新客户、新投流计划、新合作机会时，先不要急着执行，先问三个问题：

1. 这个问题的本质是什么？
2. 我现在掌握的信息够不够？
3. 我应该用什么最小动作验证？

建议调用方式：

```text
请基于《毛泽东选集》这篇工作模型笔记，帮我分析当前这个跨境社媒项目：
1. 当前主要矛盾是什么？
2. 哪些是假问题？
3. 哪些动作应该先做？
4. 哪些动作应该停止？
5. 给我一个7天最小验证计划。
```

## 12. 和我当前业务的连接

### 跨境社媒

跨境社媒不是发内容这么简单。它本质上是：用内容建立信任，用平台分发找到潜在客户，用私域承接完成沟通，再用案例和证据推动成交。任何书籍模型最终都必须落到这四个环节：内容、分发、承接、成交。

### 一人公司

一人公司最大的敌人不是没想法，而是想法太多、动作太散、系统太重、复盘太少。所以每一本书都必须帮我做减法：减少无效项目、减少无效内容、减少无效工具、减少无效沟通，把注意力放到能产生真实反馈和现金流的动作上。

### 做人做事

做人做事不是单纯讲情商，而是长期稳定地建立判断力、信用、节奏和边界。能长期合作的人，往往不是最会表达的人，而是能调查清楚、判断准确、交付稳定、复盘及时的人。

## 13. 最终行动清单

- 每次做新项目，先写一页“问题判断表”。
- 每次做内容，先明确它解决的是客户哪一个真实疑虑。
- 每次投流，先判断是否已经具备信任证据和承接路径。
- 每次合作，先判断对方资源、能力、动机和风险。
- 每周复盘一次：哪些判断被证实，哪些判断被推翻。
- 每月沉淀一次：把成功案例、失败案例、客户问题和高转化话术放进知识库。
