# 《孙子兵法》｜竞争判断、低成本获胜与平台作战模型

## 1. 一句话结论

《孙子兵法》对我最有价值的地方，是教我不要盲目开战、不要正面硬刚、不要用高成本证明自己，而是先计算胜算，找到局部优势，再用最小代价拿到结果。用在跨境社媒、一人公司和做人做事里，就是先判断竞争格局，再选择战场，再决定动作。

## 2. 这本书最值得我学的内容

第一，先算后打。很多人做生意和做内容，一上来就发、一上来就投流、一上来就建系统，但没有算清楚客户是谁、对手是谁、平台规则是什么、自己有什么优势、成本能不能承受。《孙子兵法》开篇强调“不可不察”，对你来说就是：每个项目启动前必须先做胜算评估。

第二，知己知彼。知己不是自信，知彼不是看几个同行账号。知己是清楚自己的资源、能力、成本、时间、供应链、内容能力；知彼是清楚客户需求、竞争对手打法、平台规则、市场价格、成交门槛。

第三，避实击虚。一人公司不要跟大公司硬拼预算、团队、品牌和大市场。应该寻找小切口、小人群、小场景、小关键词、小渠道，用局部优势换结果。

第四，先胜后战。不是打了再看有没有机会，而是先创造有利条件，再投入资源。内容、投流、客户开发、合作谈判，都应该先建立胜算，再扩大投入。

第五，以全争胜。最好的胜利不是打得很惨才赢，而是用策略减少消耗。商业里最好的增长，也不是把自己累死，而是让内容、案例、知识库、自动化、客户口碑形成复利。

## 3. 核心模型提炼

### 3.1 五事七计商业版

《孙子兵法》的“五事”可以转成你的商业检查表：

1. 道：目标是否一致。你的内容、产品、客户、平台、承接是否指向同一个成交目标。  
2. 天：时机是否合适。平台红利、季节需求、市场趋势、客户采购周期是否有利。  
3. 地：战场是否正确。选择 Facebook、TikTok、LinkedIn、YouTube，还是私域社群，不同战场打法不同。  
4. 将：执行能力是否匹配。你是否有脚本能力、剪辑能力、私信承接能力、复盘能力。  
5. 法：流程是否稳定。有没有内容生产 SOP、询盘处理 SOP、客户标签、复盘表、案例库。

商业里的“七计”可以问：

- 谁更懂客户？  
- 谁更懂平台？  
- 谁内容更稳定？  
- 谁信任证据更强？  
- 谁承接更快？  
- 谁成本更低？  
- 谁复盘更及时？

### 3.2 先胜后战模型

先胜后战不是保守，而是先把可控条件准备好。比如你要做一个跨境社媒项目，开打之前至少要准备：

- 目标客户画像。  
- 20个客户真实问题。  
- 10条核心内容脚本。  
- 3个信任案例。  
- 1套私信承接话术。  
- 1个简单落地页或资料包。  
- 1个复盘表。

这些没有准备好就投流，很容易烧钱；没有准备好就大量发内容，很容易吸引错人；没有准备好就谈合作，很容易承诺过度。

### 3.3 避实击虚模型

大公司强在哪里？预算、团队、品牌、成熟案例。你不能从这些地方硬打。你要找他们虚的地方：

- 他们响应慢，你响应快。  
- 他们内容标准化，你内容更真实。  
- 他们不愿意做小客户，你可以服务细分客户。  
- 他们不愿意解释基础问题，你可以做教育型内容。  
- 他们只讲品牌，你可以讲细节、过程、避坑、对比、案例。

跨境社媒里，“虚”常常藏在长尾需求里。比如大品牌打大词，你打具体场景；他们讲宏大卖点，你讲客户真实问题；他们堆参数，你讲选择标准；他们发宣传片，你发真实过程。

### 3.4 情报系统模型

《孙子兵法》的“知”不是口号，而是一整套情报系统。你的一人公司也必须有自己的情报系统：

- 同行内容库：记录同行爆款选题、标题、结构、评论区问题。  
- 客户问题库：记录客户最常问的问题和成交前疑虑。  
- 竞品案例库：记录不同服务商、工厂、品牌的表达方式。  
- 平台数据库：记录不同平台内容表现和询盘质量。  
- 失败案例库：记录自己踩过的坑，避免重复犯错。

没有情报系统，所有动作都会变成临时发挥。

## 4. 跟我工作的连接

### 4.1 用在跨境社媒

跨境社媒不是谁更努力谁赢，而是谁更会选择战场。不同平台就是不同战场：

- TikTok：适合视觉冲击、过程展示、快速认知。  
- Facebook：适合社群、广告、兴趣人群、再营销。  
- LinkedIn：适合B2B背书、专业身份、公司决策人连接。  
- YouTube：适合长内容、搜索流量、专业解释和案例沉淀。  
- WhatsApp：适合私域承接、报价、关系推进。

不要把同一套内容复制到所有平台。要根据平台战场调整打法。

### 4.2 用在一人公司

一人公司最重要的战略不是扩张，而是集中。你要避免同时开太多战场：今天做食品机械，明天做法律获客，后天做课程转写，大后天做视频剪辑。可以研究多个方向，但真正投入时必须阶段性集中。

一人公司的低成本获胜路径是：

1. 选择细分战场。  
2. 找到客户高频痛点。  
3. 做一批高质量内容。  
4. 建立信任证据。  
5. 用私域承接。  
6. 复盘成交链路。  
7. 再扩大。

### 4.3 用在做人做事

做人做事也需要判断“战场”。有些争论没有价值，有些关系不值得消耗，有些机会看起来热闹但成本极高。成熟的人不是每件事都赢，而是知道哪些事不该打。

可以问：

- 这件事值得我投入注意力吗？  
- 我有没有优势？  
- 对方的真实诉求是什么？  
- 有没有不争而胜的方法？  
- 继续纠缠会不会消耗我的主线？

## 5. 可以直接使用的判断问题

### 战场选择

1. 这个平台是不是我的优势战场？  
2. 这个客户群是不是我能服务的人？  
3. 这个关键词是不是竞争太强？  
4. 有没有更小、更具体、更容易切入的场景？  
5. 我有没有能力承接这个流量？

### 竞争分析

1. 对手强在哪里？  
2. 对手弱在哪里？  
3. 客户对现有方案最不满意什么？  
4. 我能不能用更快、更真实、更具体的内容切入？  
5. 我是否在用自己的短板打别人的长板？

### 投流前检查

1. 有没有明确目标客户？  
2. 有没有信任素材？  
3. 有没有私信话术？  
4. 有没有落地页或资料包？  
5. 有没有复盘指标？

## 6. 主用模型

### 模型一：五事七计检查表

每个项目启动前，用五事七计做一次胜算判断。判断结果分三类：立即执行、补齐条件后执行、放弃。

### 模型二：避实击虚打法

不要从竞争最强的地方切入。找到客户需求里被忽略的小点，做具体、真实、专业的内容，让对手的大资源失效。

### 模型三：先胜后战流程

先建立内容、案例、话术、承接、复盘，再扩大动作。没有胜算不重仓，没有承接不投流，没有案例不强推成交。

## 7. 备选模型

### 奇正模型

“正”是稳定内容：产品介绍、案例、FAQ、工厂过程、客户问题。“奇”是突破内容：反常识、对比、挑战、拆解、热点借势。账号不能只有奇，也不能只有正。

### 致人而不致于人

商业里可以理解为掌握节奏，不被客户、同行、平台牵着走。你要主动设计内容节奏、报价节奏、沟通节奏，而不是客户问什么你被动答什么。

### 用间模型

不能理解成阴谋，而要理解成情报收集。通过公开资料、同行内容、客户反馈、展会信息、广告库、评论区，建立市场感知。

## 8. 不建议强行套用的部分

《孙子兵法》是战争经典，但商业不是战争的简单复制。不要把客户当敌人，不要把同行都当敌人，不要为了赢而牺牲信用。你要学习的是计算、选择、节奏、成本控制，而不是把商业关系全部斗争化。

## 9. AI调用提示词

```text
请基于《孙子兵法》的五事七计、知己知彼、避实击虚、先胜后战，分析我当前这个跨境社媒项目：
1. 当前战场选择是否正确？
2. 我的优势和短板是什么？
3. 对手强在哪里，虚在哪里？
4. 应该避开什么正面竞争？
5. 给我一个低成本获胜的内容和获客计划。
```

## 10. 我的行动版总结

《孙子兵法》给我的最大启发是：不要盲打，不要硬刚，不要高成本试错。先算胜算，再选战场；先找优势，再投入资源；先建立信任，再放大流量。对一人公司来说，战略不是做更多，而是用更少动作拿到更确定的结果。

## 11. 放进知识库后的使用方法

这篇文稿不要当成普通读书笔记使用，而要当成一个“决策前置检查器”。以后遇到新项目、新账号、新客户、新投流计划、新合作机会时，先不要急着执行，先问三个问题：

1. 这个问题的本质是什么？
2. 我现在掌握的信息够不够？
3. 我应该用什么最小动作验证？

建议调用方式：

```text
请基于《孙子兵法》这篇工作模型笔记，帮我分析当前这个跨境社媒项目：
1. 当前主要矛盾是什么？
2. 哪些是假问题？
3. 哪些动作应该先做？
4. 哪些动作应该停止？
5. 给我一个7天最小验证计划。
```

## 12. 和我当前业务的连接

### 跨境社媒

跨境社媒不是发内容这么简单。它本质上是：用内容建立信任，用平台分发找到潜在客户，用私域承接完成沟通，再用案例和证据推动成交。任何书籍模型最终都必须落到这四个环节：内容、分发、承接、成交。

### 一人公司

一人公司最大的敌人不是没想法，而是想法太多、动作太散、系统太重、复盘太少。所以每一本书都必须帮我做减法：减少无效项目、减少无效内容、减少无效工具、减少无效沟通，把注意力放到能产生真实反馈和现金流的动作上。

### 做人做事

做人做事不是单纯讲情商，而是长期稳定地建立判断力、信用、节奏和边界。能长期合作的人，往往不是最会表达的人，而是能调查清楚、判断准确、交付稳定、复盘及时的人。

## 13. 最终行动清单

- 每次做新项目，先写一页“问题判断表”。
- 每次做内容，先明确它解决的是客户哪一个真实疑虑。
- 每次投流，先判断是否已经具备信任证据和承接路径。
- 每次合作，先判断对方资源、能力、动机和风险。
- 每周复盘一次：哪些判断被证实，哪些判断被推翻。
- 每月沉淀一次：把成功案例、失败案例、客户问题和高转化话术放进知识库。
