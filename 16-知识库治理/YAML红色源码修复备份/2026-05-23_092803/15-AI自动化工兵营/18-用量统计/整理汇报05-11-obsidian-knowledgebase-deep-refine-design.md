# Obsidian 知识库深度整理设计

## 概述

对已完成文件夹重组的 Obsidian 知识库（1,947 文件）进行深度整理：标准化 frontmatter、注入双向链接、构建标准文章结构、创建索引页、建立 QA 问题库、优化知识图谱连通性。

**三阶段策略：**
- Phase 1：脚本批量处理全部 ~1,900 文件（frontmatter + 标签 + 双链 + 结构骨架）
- Phase 2：Agent 并行精修 ~200 核心文件（内容丰富 + 精准双链 + 投流/案例拆解）
- Phase 3：图谱验证 + 索引构建 + QA 问题库

---

## Phase 1：脚本批量处理

### 脚本 1：Frontmatter 补全

**目标：** 确保每个文件包含完整的 14 字段 frontmatter

**现有 frontmatter 字段（已有 97% 覆盖率）：**
```yaml
title, aliases, type, status, category, platform, industry, scenario, tags, created
```

**缺失字段（需补全）：**
```yaml
related:
  - [[01-全库工作总览MOC]]          # 双链列表，格式: ["[[文章名]]", ...]
updated: "2026-05-11" # 整理日期
source: "AI整理"      # 来源
summary: ""          # 一句话总结，从标题和前3段提取
```

**自动填充规则：**
- `summary:` 取文件 `## 一句话总结` 下的内容，若无则取第一个段落的前 50 字
- `related:` 由脚本 3 注入
- `updated:` 固定为今天日期
- `source:` 默认 "AI整理-2026-05-11"，若已有 source 则保留

### 脚本 2：标签标准化

**目标：** 统一 2,384 个现有标签，消除同义重复

**标准标签体系（12 大类 × 细分标签）：**

| 大类 | 标准标签 | 合并目标 |
|------|---------|---------|
| 跨境社媒 | `#跨境社媒` `#Facebook投流` `#LinkedIn开发` `#TikTok运营` | `#facebook-ads` → `#Facebook投流` |
| 国内投流 | `#巨量本地推` `#抖音本地推` | `#本地推` → `#巨量本地推` |
| 行业 | `#食品加工机械` `#教育培训` `#韩国医美` `#装修家居` `#B2B外贸` | 统一中文 |
| 素材 | `#广告素材` `#素材优化` `#Hook设计` `#UGC策略` | 合并变体 |
| 客户 | `#客户开发` `#客户服务` `#询盘转化` `#私域跟进` | 统一 |
| 投流策略 | `#投流策略` `#投流复盘` `#数据优化` `#账户策略` | 合并 |

**处理方式：** 扫描所有文件，执行标签替换（sed），同时更新 frontmatter 中的 tags 数组

### 脚本 4：标准结构注入

**目标：** 为缺少标准章节的文章注入骨架

**注入位置：** 在 frontmatter 之后、第一个 `# 标题` 之后

**标准章节骨架：**
```markdown
## 一句话总结
<!-- 从 summary 字段填充 -->

## 核心结论
<!-- 若文件已有列表，保留；否则留空待补充 -->

## 适用场景
- 适合平台：{{platform}}
- 适合行业：{{industry}}
- 适合场景：{{scenario}}

## 具体内容
<!-- 保留原文内容 -->

## 可复用方法
<!-- 留空，待 Phase 2 填充 -->

## 对我的业务有什么价值
<!-- 留空，待 Phase 2 填充 -->

## 相关案例
<!-- 从 related 字段生成 -->

## 相关知识点
<!-- 从 related 字段生成 -->

## 后续可提问的问题
<!-- 基于内容生成 3-5 个问题 -->

## 待补充
<!-- 标记 #待补充 -->
```

**投流文件额外注入：**


**案例文件额外注入：**
```markdown

### 投放目标：<!-- 待补充 -->
```

**防护规则：** 已有章节的文件不覆盖，只追加缺失章节

### 精修内容

每篇核心文件需完成：

1. **内容丰富：** 填充所有标准章节，确保每个章节有实质内容
2. **精准双链：** 10-15 条精准双链，覆盖向上/横向/向下/项目/AI 五个维度
3. **投流要素/案例拆解：** 完整填写所有字段
4. **可复用方法：** 提炼 3-5 条可复用方法
5. **业务价值：** 结合 6 大业务线说明价值
6. **提问问题：** 生成 5-10 个可向知识库提问的问题
7. **标签优化：** 至少 8 个精准标签

- 运行 `obsidian-link-graph` 扫描全库
- 识别孤立文件（wikilink < 3）
- 为孤立文件补充链接
- 验证核心节点连通性

   - 按问题查看（6 个）
