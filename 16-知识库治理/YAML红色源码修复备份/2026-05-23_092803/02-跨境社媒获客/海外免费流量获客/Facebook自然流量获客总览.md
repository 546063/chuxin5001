## 一、Facebook 免费流量生态全景

Facebook 提供 6 种免费流量渠道，各有不同定位和适用场景：

### 1.1 六大渠道概览

| 渠道 | 核心定位 | 触达机制 | B2B设备适配度 | 成人用品适配度 |
|------|---------|---------|-------------|-------------|
| **Facebook Page** | 品牌信任基础 | 粉丝推送 + 算法推荐 | 高 | 低（政策限制） |
| **Facebook Group** | 精准社群获客 | 群成员互动 + 搜索 | 高 | 中（需合规群） |
| **个人号（Profile）** | 人设建立 + 直接沟通 | 好友动态 + 算法推荐 | 中高 | 中（私信受限） |
| **Facebook Reels** | 短视频爆发获客 | 推荐算法（非粉丝也可触达） | 中高 | 低（审核严格） |
| **Facebook Marketplace** | 本地商品交易 | 搜索 + 位置推荐 | 低 | 低（政策严禁） |
| **Facebook Live** | 实时互动展示 | 粉丝通知 + 推荐 | 中 | 低 |

### 1.2 各渠道核心逻辑

```
Page（品牌基础）──→ 建立信任 → 引导询盘
     │
     ├── Group（社群）──→ 精准 B2B 人群 → 评论/私信 → WhatsApp
     │
     ├── Reels（短视频）──→ 算法推荐爆发 → 主页 → 询盘
     │
     ├── 个人号（人设）──→ 好友互动 → 私信 → WhatsApp
     │
     ├── Live（直播）──→ 实时展示 → 评论互动 → 询盘
     │
     └── Marketplace（交易）──→ 本地搜索 → ⚠️ 不适合B2B设备
```

### 1.3 渠道优先级推荐

| 优先级 | 渠道 | 原因 |
|--------|------|------|
| P0 | Group | 精准 B2B 人群聚集，意向高 |
| P0 | Page | 品牌信任基础，必须配置 |
| P1 | Reels | 算法推荐触达非粉丝，爆发潜力大 |
| P1 | 个人号 | 人设建立，适合中小团队 |
| P2 | Live | 实时展示产品/工厂，适合定期活动 |
| P3 | Marketplace | 不适合 B2B 设备（详见[[Facebook Marketplace获客可行性分析]]） |

## 二、适合 / 不适合的产品类型

### 2.1 食品机械

| 维度 | 评估 |

| **适配度** | 高 |

## 六、触达率提升关键技巧

### 6.1 Facebook 算法核心逻辑（2025-2026）

Facebook 算法基于 4 个核心信号排序内容：

1. **Inventory（内容池）**：粉丝关注 + 推荐系统可扩展的内容
2. **Signals（信号）**：发布时间、互动历史、内容类型、设备
3. **Predictions（预测）**：用户与你内容互动的概率
4. **Score（评分）**：综合计算后决定是否展示

> **来源**：[Facebook How Feed Works](https://transparency.fb.com/policies/feed-algorithm/)

### 6.2 提升触达率的 10 个技巧

| # | 技巧 | 说明 |
|---|------|------|
| 1 | **前 30 分钟互动是关键** | 发帖后 30 分钟内的互动量决定后续推荐量，发帖后立即在 Group 分享 |
| 2 | **回复每一条评论** | 评论回复会触发二次推送，提升帖子的 Engagement Score |
| 3 | **视频优先** | Facebook 优先推荐视频内容，尤其是 Reels（完播率 > 停留时长） |
| 4 | **原生视频上传** | 不要贴 YouTube 链接，上传原生视频获得更高触达 |
| 5 | **避免站外链接** | 帖子中带站外链接会降低触达，放在评论区第一条 |
| 6 | **提问式结尾** | 以问题结尾的帖子评论率提升 2-3 倍 |
| 7 | **使用 Facebook 功能** | Poll、Event、Live 等功能内容获得算法加权 |
| 8 | **稳定频率** | 算法偏好稳定发布的账号，忽多忽少会降低权重 |
| 9 | **长文+图片组合** | 带 1-3 张图片的长文帖子（300+ 字）比纯链接帖触达高 3-5 倍 |
| 10 | **避免 Engagement Bait** | "Like this post if..." 类内容会被算法降权 |

> **来源**：[Facebook News Feed FYI](https://about.fb.com/news/2021/07/providing-more-transparency-into-how-facebook-works/)、[Facebook Ranking Signals](https://transparency.fb.com/policies/feed-algorithm/)

### 6.3 避免降权的内容

| 会被降权 | 原因 |
|---------|------|
| "点赞/分享/评论"引导 | Engagement Bait 政策 |
| 站外链接在正文 | Facebook 不希望用户离开平台 |
| 重复内容 | 算法识别重复会降低触达 |
| 低质量图片/模糊视频 | 用户停留时间短，算法判定低质量 |
| 过多文字的图片 | Facebook 曾限制图片文字比例（虽取消限制但仍影响触达） |

## 七、评论区/私信/WhatsApp 承接路径总览

### 7.1 完整承接路径

```
用户看到内容
    ↓
用户评论/私信
    ↓
第一回合：快速回复（< 1 小时）
    ↓
提供价值：回答问题/给资料/给报价
    ↓
引导到 WhatsApp：wa.me/[号码]
    ↓
WhatsApp 承接：需求确认 → 报价 → 跟进 → 成交
```

### 7.2 各渠道承接方式

| 渠道 | 承接方式 | 说明 |
|------|---------|------|
| Page | 私信 → WhatsApp | 设置自动回复 + 手动引导 |
| Group | 评论 → 私信 → WhatsApp | 先提供公开价值，再私聊 |
| 个人号 | 直接私信 → WhatsApp | 人设优势，直接沟通 |
| Reels | 评论 → 主页 → Bio 链接 → WhatsApp | Reels 评论区不可发链接 |
| Live | 评论 → 私信 → WhatsApp | 直播中引导私信 |

### 7.3 关键话术模板

详见 [[免费流量转WhatsApp承接SOP]]

**快速模板**：
```
Hi [Name], thanks for your interest! I can share more details and a custom quote on WhatsApp. 
Here's the link: wa.me/[number]
Looking forward to helping you!
```

### 7.4 承接注意事项

| 注意事项 | 原因 |
|---------|------|
| 不在 Group 公开帖子发 WhatsApp 链接 | 会被管理员删除/封禁 |
| 先提供价值再引导 | 直接发链接像 spam，效果差 |
| 回复速度 < 1 小时 | 超过 4 小时转化率下降 60%+ |
| 使用 WhatsApp Business | 专业形象 + 快捷回复 + 自动回复 |
