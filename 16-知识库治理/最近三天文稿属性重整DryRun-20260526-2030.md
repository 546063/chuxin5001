# 最近三天文稿属性重整 - DryRun 报告

- **生成时间**: 2026-05-26
- **抽样文件数**: 15 篇
- **待处理总数**: 404 篇

---

## 一、发现的问题类型

### 1. 非标准字段泛滥
多数文件存在大量非标准字段，包括但不限于：
- `topic`, `module`, `category`, `platform`, `industry`, `scenario`
- `routing_target`, `model_route`, `use_case`, `source`, `source_quality`, `source_files`
- `relations`, `review_note`, `one_sentence_summary`, `merged_from`, `old_versions`, `archived_reason`
- `updated_at`（应为 `updated`）

### 2. type 字段不匹配标准枚举
发现的非标准 type 值：
- `content-card` → 应为 `方法论`/`SOP`/`案例` 等
- `工具技能` → 应为 `SOP` 或 `方法论`
- `book_distillation` → 应为 `研究报告` 或 `原始资料`
- `框架` → 应为 `方法论`
- `moc` → 应为 `索引`
- `case` → 应为 `案例`
- `路由表` → 应为 `索引`

### 3. one_sentence_summary 中泄漏 Markdown 源码
部分文件的 `one_sentence_summary` 字段包含完整 Markdown 正文内容（如 `# 案例-TikTok...`），属于严重的数据污染。

### 4. aliases 过泛或无意义
- 直接复制文件名（如 `"01-核心框架__逻辑性推进骨架"`）
- 空数组 `[]`
- 单个别名与 title 完全相同

### 5. tags 过泛
- `["MOC", "知识库", "导航"]` 过于宽泛
- `["外贸调研", "VR成人用品", "SEO"]` 缺少平台/场景搭配
- 嵌套数组格式：`- - - - - #客资优化`（YAML 格式错误）

### 6. related 双链问题
- 空值：`related: ` 或 `related: []`
- 全部链接到总索引/MOC，无业务相关链
- 重复链接同一文档
- 双链目标文档可能不存在

### 7. quality_level 非标准值
- `imported` 应转为 `B`（需要验证）
- `"中"` 应转为 `B`
- `"high"` 应转为 `A`

### 8. 内容极薄/占位文档
- 多处"待补充"占位
- 内容只有模板框架，无实际业务内容

---

## 二、准备采用的修复规则

1. **删除所有旧 frontmatter**，不保留任何非标准字段
2. **读取正文**，根据实际内容生成 11 个标准字段
3. **type 映射**：
   - content-card/框架/工具技能/路由表 → 按内容判断为 `方法论`/`SOP`/`索引`
   - book_distillation → `研究报告`
   - moc → `索引`
   - case → `案例`
4. **quality_level 映射**：
   - imported → `B`
   - "中"/"high" → 按内容质量判断 `A` 或 `B`
   - 内容极薄/大量"待补充" → `C` 或 `D`
5. **tags**：根据正文内容生成 4-8 个精准业务标签
6. **aliases**：提取 3-6 个用户可能搜索的真实说法
7. **related**：优先链接同业务链路的 3-8 篇文档
8. **summary_cn**：100-200 字，包含问题+场景+业务价值
9. **清理正文残留 YAML 源码**

---

## 三、风险点

1. **related 双链**：需要对全库文件有索引才能精准链接。将通过全库文件清单匹配。
2. **大量"安先生"导入文档**：336 篇集中在 04-内容素材弹药库，内容相似度高，需逐篇判断。
3. **时间消耗**：404 篇逐篇处理预计需要较长时间。

---

## 四、建议

**建议继续正式执行**。发现的问题类型明确，修复规则清晰。主要风险是 related 双链精准度，可通过全库索引匹配降低。
