# 16-知识库治理 目录总览

## 中文总结

本目录是Obsidian知识库的治理指挥中心，涵盖结构审计、MOC体系管理、断链检查、Dataview配置、Graph视图治理、归档规则、重构记录、迁移映射等核心治理流程。确保全库结构完整、链接通畅、内容质量可控。

## English Summary

This directory is the governance command center for the Obsidian knowledge base. Covers structure auditing, MOC system management, broken link checking, Dataview configuration, Graph view governance, archiving rules, refactoring records, and migration mapping. Ensures complete library structure, smooth links, and controllable content quality.

## 一句话结论

知识库治理是保持全库健康度的核心维护体系，通过标准化流程确保知识资产持续可用。

## 本目录适用场景

- 知识库结构审计与优化
- MOC体系规划与维护
- 断链检测与修复
- Dataview查询配置
- Graph视图优化
- 归档与重构决策

## 本目录不适用场景

- 具体业务内容生产（见对应业务目录）
- AI工具使用（见15-AI自动化工兵营）
