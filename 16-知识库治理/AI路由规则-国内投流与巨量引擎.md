## 回答格式
回答必须包含：
1. **直接结论** — 先给一句话答案
2. **判断依据** — 基于哪个文档/数据
3. **执行方案** — 具体步骤
4. **风险点** — 注意事项
5. **下一步动作** — 紧接着做什么
6. **相关文档** — 链接到知识库文档

## 禁止行为
1. 不允许只根据通用经验回答，必须引用知识库文档。
2. 不允许忽略本地知识库，优先从知识库读取。
3. 不允许只搜全库不走路由。
4. 不允许不区分行业直接套模板。
5. 不允许把模拟案例当成真实案例。
6. 不允许在用户问行业案例时跳过该行业的案例文档。
7. 不允许在用户问数据复盘时跳过复盘指标体系文档。

## 行业差异处理
不同行业的投放策略差异巨大，回答时必须区分：
- **低客单高频**（餐饮/商超）：团购券 + POI + 到店
- **中客单中频**（美容美发/教培/宠物）：体验券 + 私信 + 到店
- **高客单低频**（家装/婚纱/医美/口腔）：表单留资 + 电话跟进 + 到店
- **B2B线索**（消防/除甲醛/招商/B2B设备/工厂机械）：表单留资 + 招商经理跟进 + 实地考察

## 更新规则
当知识库新增/修改文档时：
1. 检查是否需要更新本路由入口的链接
2. 检查是否需要更新问题路由表的映射
3. 检查是否需要更新 JSON 路由文件的 preferred_notes
4. 更新后重新运行测试问题集验证
