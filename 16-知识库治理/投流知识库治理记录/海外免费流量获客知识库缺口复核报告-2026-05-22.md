## 六、后续同步规则

### 强制规则（必须写入同步配置）

**规则 1：iMac 优先原则**
- iMac 是主库，Mac mini 是 Worker 副本
- 同步方向必须是 iMac → Mac mini（推送），不是 Mac mini → iMac（拉取覆盖）
- 任何从 Mac mini 到 iMac 的同步，禁止使用 --delete 标志

**规则 2：合并前检查**
- iMac 新增知识库未确认合并到 Mac mini 前，禁止用 Mac mini --delete 覆盖 iMac
- 每次同步前，先 dry-run 检查会被删除的文件
- 如果 dry-run 显示有删除操作，先 git commit 再同步

**规则 3：git commit 保护**
- iMac 新增文件后必须先 git add + git commit
- 这样即使 rsync --delete 也不会丢失（可 git restore）

**规则 4：同步前备份**
- 同步前在 iMac 执行：`git add -A && git commit -m "pre-sync backup"`
- 同步后检查：`git status` 确认关键文件仍在
