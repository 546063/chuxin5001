## 二、认证问题根因

### 为什么报 `Not logged in`

1. 系统实际使用**阿里百炼 Qwen3.6-plus API**，不是官方 Anthropic API
2. `.zshrc` 启动时会加载 `ANTHROPIC_*` 环境变量（指向 DashScope）
3. **launchd 不加载 `.zshrc`**，所以运行时没有这些环境变量
4. `claude` 在没有 env 的情况下走官方 API，需要交互式 `/login`
5. 脚本 `run_douyin_juliang_update.sh` 直接调用 `claude` 而没有 source 环境变量

### 修复方式

3个脚本统一修改：
```bash
# 在调用 claude 之前，source qwen.env
ENV_FILE="$HOME/.config/claude-code/providers/qwen.env"
if [ -f "$ENV_FILE" ]; then
    source "$ENV_FILE"
fi

# 同时添加 --model qwen3.6-plus 参数
${CLAUDE_BIN} --model qwen3.6-plus --dangerously-skip-permissions ...
```

plist 修改：添加 `HOME=/Users/mac` 环境变量，确保脚本能找到 `$HOME`。
