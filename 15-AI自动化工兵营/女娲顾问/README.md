## 具体内容
# 女娲顾问知识库

> **核心原则：** 资料可以多，但进入决策系统的内容必须少。
> 原始资料放仓库，决策时只调用压缩后的顾问卡。

## 系统架构

```
顾问资料（丰富）
  ↓ 压缩
心智模型（精炼）
  ↓ 压缩
决策卡片（极简）
  ↓ 调用
女娲Skill决策
  ↓ 沉淀
个人原则（你的资产）
```

## 目录结构

| 目录 | 用途 | 调用优先级 |
|------|------|-----------|
| `00_原始资料/` | 一手/二手资料索引 | 不调用 |
| `10_顾问心智模型/` | 8核心董事决策卡 | 高 |
| `20_业务工具顾问/` | 场景专用顾问卡 | 中 |
| `30_观察型顾问/` | 保留参考 | 低 |
| `40_我的业务系统/` | 个人原则库 | 高 |
| `50_决策委员会/` | 委员会调用规则 | 高 |

| `70_案例库/` | 成功/失败案例 | 中 |

## 调用规则

1. 不要每次问15个人，按问题调用对应委员会（3-6人）
2. 优先读取：决策委员会 → 顾问调用卡 → 个人原则 → 复盘
3. 不要读取：原始资料全文、长篇证据、无关顾问卡
