# 羽毛球招生视频剪辑师版本风格分析与 SOP V3

## 0. 本 SOP 的核心结论

这四个参考视频不是普通素材，而是**剪辑师正确版本**。以后 600.mp4、601.mp4 这类 AI 错误剪辑，必须按这四个版本的风格重做。

最重要的规则：

1. **口播画面必须使用原始同期声**，不能拿别的声音套嘴型。
2. **字幕必须强制全程存在**，并且和声音完全同步。
3. **字幕样式要按剪辑师版本来**，不要再用 600/601 里那种小字幕。
4. **B-roll 只服务文案**，不能为了切镜头而乱切。
5. **逐字稿讲完，视频就结束**，不需要额外空镜、黑尾、音乐尾巴或空白结尾。
6. **男声提醒、拍摄者声音、场外提示音全部删除或静音**。

# 2. 四个剪辑师版本的共同点

## 2.1 画幅与整体观感

统一为竖屏短视频：

```text
512×910，接近 9:16
30fps
人物居中或略居中
字幕居中
画面干净，不乱堆元素
```

剪辑师版本不是花哨剪法，而是**清晰、干净、可信、招生转化导向**。

---

## 2.2 人物口播画面特点

参考 256、258、408、416 的口播画面：

| 项目 | 标准 |
|---|---|
| 构图 | 人物居中，脸部清楚，胸口以上或半身为主 |
| 镜头 | 口播时尽量稳定，不乱缩放 |
| 嘴型 | 画面中能看到嘴巴时，必须和声音同步 |
| 眼神 | 正对镜头或自然看镜头 |
| 停顿 | 不保留明显卡壳、等待、提示声 |
| 口播时长 | 核心表达时不频繁切走 |

正确做法：

```text
女生正在说重点 → 保留女生正脸口播。
女生声音继续，但画面切到训练/B-roll → 只在非关键口型处切。
```

错误做法：

```text
女生嘴巴在说 A 句，声音却是 B 句。
女生口播刚开始就切走，观众看不到真实表达。
为了画面丰富，频繁切掉关键口型。
```

---

## 2.3 B-roll 使用共同点

四个参考视频的 B-roll 不是乱切，而是围绕内容服务：

| 文案内容 | 对应画面 |
|---|---|
| 讲零基础 | 女生拿球拍口播、训练入门画面 |
| 讲第一节课 | 教练示范、学员练习、基础动作 |
| 讲孩子训练 | 儿童训练、发球、教练指导、球场全景 |
| 讲俱乐部/教练员 | 门头、场馆、教练、学员训练、品牌 logo |
| 讲免费体验/测试 | 教练手持球、现场训练、报名氛围 |

B-roll 节奏：

```text
短视频 10-12 秒版本：约 1-2 秒一切。
30 秒版本：口播段可保持 3-8 秒，B-roll 段约 1.5-3 秒一切。
```

---

## 2.4 字幕位置共同点

字幕不是贴底，而是在**底部安全区上方**。

观察标准：

```text
字幕整体位于画面下方 68%-82% 区域。
不贴最底部。
不遮嘴。
不挡关键动作。
通常居中显示。
```

建议标准：

| 输出分辨率 | 字幕垂直位置 |
|---|---|
| 512×910 | y 约 650-760 区间，根据画面微调 |
| 1080×1920 | y 约 1380-1600 区间，根据画面微调 |

特别注意：

- 坐姿口播时，字幕可放在胸口以下、手部上方或下方安全区。
- 站姿口播时，字幕可放在腿部上方，但不能贴底。
- 球场训练画面，字幕不能遮挡羽毛球和动作关键点。

---

## 2.5 字幕形式共同点

四个参考版本大致有两类字幕风格。

### A. 强钩子字幕：大号蓝白描边

出现在 256、258 这种 10 秒左右短钩子视频里。

特征：

```text
大字号
白色主体
浅蓝/蓝色描边或阴影
字重偏粗
视觉冲击强
每句较短
适合重点句、钩子句、卖点句
```

示例画面文字：

```text
零基础学习羽毛球
第一节课
打了3年羽毛球
发力还是甩胳膊
正确的方式
免费做一次
```

### B. 正文解释字幕：白色干净字幕

出现在 408、416 这种 30 秒左右解释型/品牌型视频里。

特征：

```text
白色字体
轻微黑色阴影/描边
字号中等偏大
不花哨
句子不宜太长
通常每屏 8-16 个中文字符
```

示例画面文字：

```text
有的孩子两个月
有的孩子已经把
但真正专业的训练
这里没有光鲜亮丽的氛围
真正热爱羽毛球的教练员
```

# 3. 字幕标准规范

## 3.1 字幕必须重新生成

600/601 错误版中字幕太小、位置低、同步不稳，不能沿用。

必须：

1. 提取女生真实声音。
2. 根据逐字稿或 Whisper 生成 SRT。
3. 校对字幕和声音。
4. 重新烧录大字幕。
5. 如果用户提供完整逐字稿，以逐字稿为准。

---

## 3.2 字幕字体建议

优先级：

1. PingFang SC Semibold / 苹方简体中粗体
2. Source Han Sans SC / 思源黑体
3. Noto Sans CJK SC
4. Microsoft YaHei / 微软雅黑
5. Arial Unicode MS

不建议：

```text
太细的宋体
太花的艺术字
过窄字体
不支持中文的字体
```

---

## 3.3 字幕字号建议

| 分辨率 | 普通正文字幕 | 强钩子字幕 |
|---|---:|---:|
| 512×910 | 30-38 | 38-48 |
| 1080×1920 | 62-72 | 78-92 |

如果是手机端发布，宁愿稍大，不要太小。

---

## 3.4 字幕排版规则

| 项目 | 标准 |
|---|---|
| 每屏字数 | 8-16 个中文字符优先 |
| 行数 | 优先 1 行，最多 2 行 |
| 对齐 | 居中 |
| 颜色 | 白色主体 |
| 描边 | 黑色/深色描边，或蓝色强调描边 |
| 阴影 | 可以轻微阴影，保证球场背景可读 |
| 安全区 | 不贴底，不遮嘴，不挡动作 |

---

# 4. 镜头切换与节奏规范

## 4.1 10-12 秒短视频结构

适合 256、258 的风格。

```text
0-2 秒：强钩子，人物正脸或动作画面
2-5 秒：问题/痛点
5-8 秒：训练/教练/B-roll 证明
8-12 秒：免费体验/测试/行动召唤
```

特点：

- 快节奏
- 字幕大
- 每句都短
- 结尾直接收，不拖

---

## 4.2 30 秒解释型视频结构

适合 408、416 的风格。

```text
0-3 秒：门头/训练/口播开场
3-12 秒：人物口播解释核心观点
12-22 秒：B-roll 展示训练、学员、场馆、教练
22-30 秒：回到口播或品牌收束
```

特点：

- 不是一直快切
- 口播段允许稳定停留
- B-roll 负责证明文案
- 结尾干净

---

# 5. 特效与转场规范

四个参考视频整体特效克制，不是重特效风格。

允许：

```text
轻微推拉
轻微模糊转场
短暂发光/光效强调
logo 黑底收尾
自然硬切
根据音乐点切换
```

不允许：

```text
频繁花字乱飞
大面积遮挡人物脸
过多转场音效
强行卡点导致口型错位
字幕动画太复杂导致看不清
```

特效原则：

```text
招生视频第一目标是清楚可信，不是炫技。
```

---

# 6. 结尾处理强制规则

用户特别指出：AI 错误版结尾空了很多地方。

新版强制规则：

1. 如果用户提供完整逐字稿，**逐字稿最后一句说完后 0.2-0.5 秒内结束视频**。
2. 不需要额外空镜。
3. 不需要无意义门头尾巴。
4. 不需要长时间黑屏。
5. 不需要字幕结束后继续播放。
6. 如果要 logo 收尾，最多 1.0-1.5 秒。
7. 如果没有品牌收尾需求，最后一句讲完直接结束。

验收标准：

```text
听到最后一句话结束 → 字幕消失 → 0.2-0.5 秒内视频结束。
```

---

# 7. 600/601 修复要求

600.mp4 和 601.mp4 是 AI 错误剪辑，不作为风格参考。

必须修复：

| 问题 | 处理方式 |
|---|---|
| 口型和声音不匹配 | 回到原始音画绑定片段，不允许错配 |
| 字幕和声音不同步 | 根据逐字稿/音频重新生成 SRT |
| 字幕太小 | 按剪辑师版本放大并重烧录 |
| 字体不好 | 换成苹方/思源黑体/Noto Sans CJK |
| 男声残留 | 删除、静音或只保留画面 |
| 画面乱切 | 按文案内容匹配 B-roll |
| 结尾空白 | 逐字稿讲完立即结束 |
| 缺字幕 | 强制全程字幕 |

# 8. Claude Code 执行验收表

| 检查项 | 合格标准 |
|---|---|
| 口型同步 | 女生说话画面和原声一致 |
| 字幕同步 | 字幕出现时间和声音一致 |
| 字幕大小 | 手机端清晰可读 |
| 字幕字体 | 接近剪辑师版本，不用小细字 |
| B-roll | 与当前文案语义匹配 |
| 男声残留 | 不进入成片 |
| 结尾 | 逐字稿讲完即结束 |
| 输出文件 | 至少输出修复版和报告 |
| 剪辑表 | 记录每段来源、用途、处理方式 |
| 报告 | 写清修复点和待人工确认点 |

# 9. 最终导出标准

建议导出：

```text
分辨率：1080×1920
帧率：30fps
编码：H.264
音频：AAC
字幕：烧录字幕
响度：女生声音清楚，男声残留为 0
输出目录：~/Downloads/羽毛球招生视频修复版/
```

如果素材本身是 512×910，也可以先做 512×910 工作版，再导出 1080×1920 发布版。

---

# 10. 后续固定流程

每次新视频都按这个流程：

```text
1. 读取参考风格视频
2. 读取待修复视频
3. 提取逐字稿/用户提供逐字稿
4. 建立片段表
5. 锁定原始口播音画绑定
6. 删除男声残留
7. 根据文案插入 B-roll
8. 重新生成字幕
9. 按剪辑师字幕样式烧录
10. 逐字稿讲完立即结束
11. 导出成片、字幕、剪辑决策表、修复报告
```

# 11. 一句话给执行 Agent

**不要自由发挥。参考 256、258、408、416 的剪辑师版本做风格母版，600/601 只能作为待修复错误版。口播原声和嘴型必须绑定，字幕必须大、准、全，逐字稿讲完立刻结束。**
