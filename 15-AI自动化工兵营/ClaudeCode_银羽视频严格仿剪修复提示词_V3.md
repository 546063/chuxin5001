下面整段直接复制给 Claude Code 使用。

```text
你现在执行“银羽羽毛球招生视频严格仿剪修复 V3”。

这次任务不是自由剪辑，而是：
1. 读取 4 个剪辑师正确版本。
2. 总结它们的字幕、位置、字体、口型、镜头切换、B-roll、特效、结尾方式。
3. 用这 4 个正确版本作为唯一风格标准。
4. 修复 600.mp4 和 601.mp4 这两个 AI 错误剪辑版本。

==================================================
一、输入文件角色定义
==================================================

参考风格视频，也就是剪辑师正确版本：
- 256_raw.mp4
- 258_raw.mp4
- 408_raw.mp4
- 416_raw.mp4

待修复错误视频，也就是 AI 错误剪辑版本：
- 600.mp4
- 601.mp4

重要：
600.mp4 和 601.mp4 不能作为风格参考，只能作为待修复对象。
真正的风格参考只能来自：256_raw、258_raw、408_raw、416_raw。

如果当前目录找不到文件，请先搜索：
find ~ -iname "256_raw.mp4" -o -iname "258_raw.mp4" -o -iname "408_raw.mp4" -o -iname "416_raw.mp4" -o -iname "600.mp4" -o -iname "601.mp4"

输出目录：
~/Downloads/羽毛球招生视频修复版/

如果目录不存在，请自动创建。

==================================================
二、第一阶段：先分析 4 个正确参考视频
==================================================

请先用 ffprobe/ffmpeg 检查四个参考视频：
- 时长
- 分辨率
- 帧率
- 音轨
- 镜头切换点
- 字幕位置
- 字幕大小
- 字幕颜色
- 字幕是否有描边/阴影
- 是否有蓝白强调字幕
- 是否有白色正文字幕
- 是否有 logo 结尾
- 结尾是否拖尾

请生成：
reference_style_analysis.md

必须总结以下内容：

1. 字幕风格
   - 字体倾向
   - 字号大小
   - 字幕位置
   - 字幕颜色
   - 描边/阴影
   - 每屏字数
   - 是否一行/两行

2. 口播风格
   - 人物构图
   - 口型同步方式
   - 口播保留时长
   - 口播和 B-roll 的关系

3. 镜头切换
   - 大概几秒一切
   - 哪些地方用硬切
   - 哪些地方用轻转场
   - 是否允许频繁切换

4. B-roll 规律
   - 讲训练时用什么画面
   - 讲孩子时用什么画面
   - 讲场馆时用什么画面
   - 讲教练时用什么画面
   - 讲体验/报名时用什么画面

5. 特效规律
   - 是否有发光、模糊、推拉
   - 特效强度
   - 是否允许花哨动画

6. 结尾规律
   - 是否有 logo 页
   - logo 页多长
   - 如果无 logo，最后一句后多久结束

==================================================
三、第二阶段：修复 600 和 601
==================================================

必须修复的问题：

1. 口型和声音不同步
2. 字幕和声音不同步
3. 男声提醒、拍摄者声音、场外提示声残留
4. 字幕太小
5. 字体不好
6. 有些地方没有字幕
7. 画面切换不准
8. B-roll 插入不服务文案
9. 结尾空了很多地方
10. 字幕和声音内容不一致

==================================================
四、口型同步硬规则
==================================================

这是最高优先级，不能违反：

1. 女生口播画面必须使用该画面原本的同期声。
2. 不允许把其他时间段的声音套到女生嘴型上。
3. 不允许把 600 的声音套到 601 的口播画面上。
4. 不允许把 601 的声音套到 600 的口播画面上。
5. 只要画面里能看到女生嘴巴在说话，声音必须和嘴型匹配。
6. 如果不确定是否同步，宁愿保留原始音画，不要错配。
7. B-roll 只能插在不需要看嘴型的位置。

执行方法：
- 先按 2-5 秒切分 600 和 601。
- 建立片段表。
- 标记每段是否为女生口播、是否有男声、是否可保留、是否可做 B-roll。
- 主口播只能使用“原始音画绑定”的片段。

==================================================
五、字幕强制规则
==================================================

必须全程有字幕。

如果用户提供完整逐字稿：
- 字幕内容以用户逐字稿为准。
- 逐字稿最后一句讲完后，视频 0.2-0.5 秒内结束。
- 不要额外加空镜。
- 不要额外加长结尾。

如果没有逐字稿：
- 使用 Whisper / faster-whisper / 系统可用转写工具生成 SRT。
- 没有转写工具时，至少输出需要用户确认的字幕稿，不允许无字幕导出。

字幕样式必须接近 4 个剪辑师正确版本：

普通正文字幕：
- 白色
- 中粗字体
- 有轻微黑色描边或阴影
- 居中
- 放在底部安全区上方
- 不能贴底
- 不能遮嘴
- 不能遮关键动作

强钩子字幕：
- 大号
- 白色主体
- 蓝色/浅蓝描边或阴影
- 适合开头重点句、卖点句、免费体验句

字体优先级：
1. PingFang SC Semibold / 苹方简体中粗体
2. Source Han Sans SC / 思源黑体
3. Noto Sans CJK SC
4. Microsoft YaHei / 微软雅黑
5. Arial Unicode MS

字号建议：
如果导出 1080×1920：
- 普通字幕 62-72
- 强钩子字幕 78-92

如果保持 512×910：
- 普通字幕 30-38
- 强钩子字幕 38-48

每屏字数：
- 优先 8-16 个中文字符
- 优先一行
- 最多两行

==================================================
六、男声残留处理规则
==================================================

男声、拍摄者提醒、场外指导声全部不能进入成片。

处理优先级：
1. 能删就删。
2. 画面有用但有男声，就静音保留画面。
3. 必要时用环境音或低音量音乐补底。
4. 不要用降噪把女生声音处理得很机械。

禁止出现：
- “开始”
- “重来一遍”
- “看镜头”
- “再说一次”
- 拍摄者提醒声
- 男生闲聊声
- 明显场外提示声

==================================================
七、画面切换规则
==================================================

必须按照 256、258、408、416 的剪辑逻辑来。

人物讲重点：
- 保留人物正脸口播。
- 不要频繁切走。
- 不能破坏口型。

讲训练：
- 切儿童/成人训练画面。
- 切教练示范。
- 切球场实拍。

讲场馆/品牌：
- 切门头、前台、奖杯墙、场馆环境。

讲免费体验/报名：
- 切教练、球拍、训练氛围。
- 字幕加大强调。

禁止：
- 为了切而切。
- 文案和画面无关。
- 空镜太久。
- 字幕结束后画面还继续空放。
- 口播声音在播，但画面显示另一个不匹配嘴型的人说话。

==================================================
八、结尾强制规则
==================================================

用户特别要求：视频结尾不要空很多地方。

所以：
1. 如果有逐字稿，逐字稿最后一句讲完，0.2-0.5 秒内结束。
2. 没有必要添加额外内容。
3. 没有必要继续放门头、空镜、训练画面。
4. 如果必须加 logo 页，最多 1.0-1.5 秒。
5. 如果不是品牌片，不加 logo 也可以，直接结束。
6. 绝对不要出现 3 秒以上空尾。

==================================================
九、输出文件
==================================================

至少输出：

1. 600_fixed_editor_style_v3.mp4
2. 601_fixed_editor_style_v3.mp4
3. final_best_editor_style_v3.mp4
4. subtitle_600_editor_style_v3.srt
5. subtitle_601_editor_style_v3.srt
6. subtitle_final_editor_style_v3.srt
7. reference_style_analysis.md
8. edit_decision_list_v3.md
9. video_fix_report_v3.md
10. need_human_review_v3.md

输出目录：
~/Downloads/羽毛球招生视频修复版/

==================================================
十、剪辑决策表格式
==================================================

生成 edit_decision_list_v3.md，格式：

| 来源文件 | 原始开始 | 原始结束 | 用途 | 是否口播 | 是否原声保留 | 是否有男声 | 是否插入 B-roll | 字幕内容 | 处理方式 | 备注 |
|---|---:|---:|---|---|---|---|---|---|---|---|

用途可选：
- 主口播
- B-roll
- 删除
- 静音保留画面
- 转场
- 品牌页
- 结尾 CTA

处理方式可选：
- 原音画保留
- 裁剪保留
- 静音保留
- 删除男声
- 只取画面
- 只取声音
- 放弃不用

==================================================
十一、修复报告格式
==================================================

生成 video_fix_report_v3.md，必须包含：

1. 输入文件基础信息
2. 4 个参考视频风格总结
3. 600/601 发现的问题
4. 具体修复动作
5. 字幕样式参数
6. 字幕文件路径
7. 成片文件路径
8. 男声残留处理清单
9. 口型同步人工复核点
10. 结尾是否已按逐字稿截断

==================================================
十二、验收标准
==================================================

最终导出前逐项检查：

[ ] 是否读取并分析了 4 个剪辑师正确版本
[ ] 600/601 是否只作为待修复视频，没有作为风格参考
[ ] 女生口播是否保持原始音画绑定
[ ] 是否没有明显口型错配
[ ] 是否全程有字幕
[ ] 字幕是否和声音同步
[ ] 字幕是否足够大
[ ] 字体是否接近剪辑师版本
[ ] 是否有蓝白强钩子字幕或白色正文字幕
[ ] 字幕位置是否在底部安全区上方
[ ] 男声提醒是否已删除或静音
[ ] B-roll 是否和文案匹配
[ ] 是否没有无意义乱切
[ ] 逐字稿结束后是否 0.2-0.5 秒内结束
[ ] 是否没有长空尾
[ ] 是否输出到 ~/Downloads/羽毛球招生视频修复版/
[ ] 是否生成 reference_style_analysis.md
[ ] 是否生成 edit_decision_list_v3.md
[ ] 是否生成 video_fix_report_v3.md

现在开始执行。
```
