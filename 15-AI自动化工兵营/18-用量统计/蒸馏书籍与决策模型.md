## 1\. “模型路由器提示词”能不能自己主动触发？

**单独一段提示词不能保证主动触发。**

它更像一条规则：

```
回答任何业务问题前，先判断问题类型，再调用 3-5 个思维模型。
```

但如果你只把它放在某篇 Obsidian 笔记里，AI 不一定会主动找到它。

所以你需要把它放在更高优先级的位置：

```
CLAUDE.md / OpenClaw 总控规则 / Codex 工作流规则 / Skill 的 SKILL.md
```

Claude Code 官方文档里，Skill 的核心文件就是 `SKILL.md` ，它用于告诉 Claude 这个 Skill 什么时候使用、怎么使用；也可以搭配模板、示例、脚本、参考文件等支持材料。

---

## 2\. 它需要做成 Skill 吗？

**建议做成 Skill。**

因为你的“模型路由器”不是普通笔记，它是一个 **决策流程** ，应该成为一个可调用的能力。

我建议你做 3 个东西：

```
1. 一个总控 CLAUDE.md
2. 一个 model-router Skill
3. 一个 Obsidian / RAG 检索规则
```

三者分工不同。

## 二、三种机制的区别

## 1\. CLAUDE.md：放最高层规则

适合放：

```
你是谁
我的业务是什么
回答问题前必须先分类
禁止一次调用太多模型
默认调用 3-5 个模型
优先使用本地知识库
不确定时再联网搜索
```

它是“总宪法”。

---

## 2\. Skill：放具体工作流

适合放：

```
模型路由器怎么工作
问题怎么分类
不同场景默认调用哪些模型
如何输出判断
如何回写知识库
```

Claude 的 Skills 设计就是把专门任务包装成文件夹和说明，让模型在相关任务出现时动态加载，而不是把所有内容都塞进上下文。Anthropic 的 Agent Skills 文档也强调了这种“只在需要时加载相关 Skill”的模式。

---

## 3\. Slash Command：手动强制触发

适合你主动输入：

```
/router 这个产品适不适合做 Facebook 获客？
/distill 蒸馏这本书
/decision 帮我做老板决策
/ad-review 分析这个广告素材
```

Claude Code 文档里说明，Slash Commands 是在会话中通过 `/` 触发的命令，适合快速调用固定工作流。

所以最稳的方式是：

```
平时自动触发 Skill
重要问题手动输入 /router 强制触发
```

## 1\. SKILL.md 放触发规则

```
---
name: model-router
description: 当用户提出业务决策、选品判断、跨境社媒、B2B客户开发、Facebook投流、LinkedIn获客、WhatsApp成交、客户账号运营、短视频文案、知识库整理、复盘问题时，必须使用本 Skill。该 Skill 的目标是从知识库中选择 3-5 个最相关思维模型辅助决策，禁止一次性调用过多模型。
---

# Model Router Skill

你是用户的“思维模型路由器”。

你的任务不是直接回答问题，而是先判断问题类型，再从用户的知识库中选择最少、最相关的思维模型。

## 必须触发的场景

当用户问题涉及以下任意内容时，必须触发本 Skill：

- 选品
- 跨境贸易
- 跨境社媒
- B2B获客
- Facebook广告
- LinkedIn客户开发
- WhatsApp私聊成交
- 新媒体运营
- 客户账号运营
- 短视频文案
- 广告素材
- 投流复盘
- 老板决策
- 一人公司经营
- 知识库整理
- 书籍蒸馏
- 人物蒸馏
- SOP优化
- 业务复盘

## 核心规则

1. 先分类，后回答。
2. 先检索本地知识库，再决定是否联网。
3. 每次只调用 3-5 个思维模型。
4. 简单问题只调用 1-2 个模型。
5. 禁止一次性调用 10 个以上模型。
6. 如果没有足够本地知识，再使用网络搜索补充。
7. 输出必须落到行动，不要只讲理论。

## 工作流程

1. 识别用户问题类型。
2. 根据 scenario-map.md 选择候选模型。
3. 从 model-index.md 中选择最相关的 3-5 个模型。
4. 读取对应模型卡片。
5. 如果业务事实不足，检索 05_业务资料库。
6. 如果本地资料不足或信息可能过期，联网搜索。
7. 给出结论、理由、行动建议和下一步需要补充的信息。
```

这个 `description` 很重要。Skill 能不能被自动触发，很大程度取决于描述是否清楚、覆盖关键词是否准确。官方文档也强调 `SKILL.md` 是核心入口，支持文件要从 `SKILL.md` 里引用。

```
# 客户痛点差距模型

## 一句话
成交不是介绍产品，而是让客户看清“现状”和“理想状态”之间的差距。

## 适用场景
- 客户问价格
- 客户说先看看
- 客户不知道自己具体需求
- B2B采购周期长

## 输入
- 客户当前做法
- 客户遇到的问题
- 问题造成的损失
- 解决后的收益

## 判断步骤
1. 问现状：你现在怎么解决？
2. 找痛点：哪里最麻烦？
3. 放大后果：这个问题造成多少时间、人工、金钱损失？
4. 连接方案：我们的产品怎么缩小差距？

## 输出
- 客户真实痛点
- 成交切入点
- 下一句该问什么

## 禁止误用
不要在还没了解客户现状时直接推产品。
```

## 十一、本地知识库 + 网络搜索怎么配合？

你要设定一个“搜索优先级”。

```
第一优先级：本地总控规则
第二优先级：本地模型库
第三优先级：本地业务资料和案例
第四优先级：网络搜索
第五优先级：AI自己的通用知识
```

但有些问题必须联网：

```
平台规则变化
Facebook广告政策
LinkedIn最新限制
OpenClaw / Claude Code 最新功能
某个产品的最新市场数据
某个厂家的最新信息
竞品价格
海关政策
物流政策
汇率
新闻事件
```

也就是说：

> **模型用本地，事实用网络。**  
> **原则用本地，数据用网络。**  
> **你的方法论用本地，外部变化用搜索。**

```
请调用我的模型路由器处理以下问题。

2. 从我的知识库中选择 3-5 个最相关思维模型。

4. 输出简洁决策，不要长篇理论。
5. 最后给我下一步动作。

问题如下：
$ARGUMENTS
```

以后你输入：

```
/router 这个食品加工机械产品适合用 Facebook 找海外 B 端客户吗？
```

它就会强制走路由器。

```

  ↓

```
