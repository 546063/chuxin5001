## 四、💡 三个具体建议

### 建议 1：**L4 三尖刀按场景走，不要默认只押 Opus**

**理由**：三家有明显的能力非对称性，单押 Opus 是浪费：

- Opus 强在编码，但 上下文超过 200K 后价格翻倍到 $10/$37.50
- Gemini 3.1 Pro 原生 1M 上下文 + 只要 Opus 60% 价格
- GPT-5.4 在电脑操作上 75% 准确率，Opus 在这个维度没竞争力

**操作建议**：

- Anthropic API 充值 **$30 月度预算上限**（硬熔断）
- Google AI Studio 开账号，充 **$20 月度预算**
- OpenAI API 充值 **$15 月度预算**（主要跑浏览器操作任务）
- 总尖刀预算：**$65 ≈ ¥470**（但实际老板不会每家都打满，平均 ¥200-300）

### 建议 2：**Gemini 3.1 Pro 是被严重低估的性价比尖刀**，值得优先尝试

**理由**：

- $2.00/$12.00 per 1M tokens，比 Opus 便宜 60%，比 GPT-5.4 还便宜 20%
- 1M token 上下文免费（标准价），老板的长文档场景天然匹配
- 多模态能力是目前全球最强，**直接替代老板在 OpenClaw 里用的 Vision MCP**
- Google AI Studio **完全免费测试**，不花钱就能验证

**具体应用**：

- 鲸驰集团合同分析、投资意向书解读 → 丢 PDF 给 Gemini 3.1 Pro
- 跨境电商多语言市场调研 → Gemini 3.1 Pro 做多国语言对比
- 外贸选品海关数据分析 → 老板的 trade-research skill 可以升级用 Gemini

**操作建议**：**5 月先花 2 周免费试用 Gemini 3.1 Pro**，再决定是否纳入正式 L4 池。

### 建议 3：**用 Hermes 做智能路由，不要老板手动选模型**

**问题**：五段路由如果靠老板手动选模型，心智负担太大，会退化成"反正走 MiniMax 就行"的懒惰路径，方案失效。

**解决方案**：在 Hermes 里加**5 级任务分类器 + 场景标签**，自动路由。

**路由决策表**（Hermes 侧实现）：

|触发特征|自动路由到|
|---|---|
|输入 <200 tokens + 纯文本|**L1 GLM-Flash**|
|默认场景|**L2 MiniMax M2.7**|
|含关键词「分析/设计/方案/架构」|**L3 GLM-5.1**|
|含关键词「重构/跨文件/debug」+ 有代码|**L4a Opus 4.6**|
|输入 >100K tokens 或含「PDF/合同/长文档」|**L4b Gemini 3.1 Pro**|
|含关键词「浏览器/页面/点击/填表」|**L4c GPT-5.4**|
|含关键词「批量/多个/并行」+ 数量 >5|**L5 Kimi Agent Swarm**|
|L4/L5 月预算 >80%|**自动降级到 L3 GLM-5.1**|

**额外规则**：

- L4 尖刀每次调用前在 Feishu 弹确认窗
- 路由决策全程日志化，月底复盘 ROI
- 每 30 天自动生成路由报告：哪些场景路由错了 → 调整规则

## 六、最终决策矩阵

|维度|五段路由的表现|
|---|---|
|**完成度**|⭐⭐⭐⭐⭐（三尖刀覆盖所有场景天花板）|
|**效率**|⭐⭐⭐⭐⭐（自动路由 + 场景匹配最优模型）|
|**性价比**|⭐⭐⭐⭐（¥450/月，比纯 Max 方案省 70%）|
|**国产优先**|⭐⭐⭐⭐（85% 流量在国产，尖刀用全球最强）|
|**稳定性**|⭐⭐⭐⭐⭐（三尖刀互为兜底，单家限流不影响主流程）|
|**运维复杂度**|⭐⭐⭐（5 个模型栈 + Hermes 路由，但有自动化）|

**核心哲学**：日常 85% 流量用国产兜底，确保**成本可控** + **响应稳定**；15% 高价值流量按场景调全球最强尖刀，确保**完成度天花板**。**对一人公司老板，这是效率和性价比的最优解**。
