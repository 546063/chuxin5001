# 百度云课程 → Obsidian 知识库自动化方案

> **来源**：百度云课程知识库方案.pdf  
> **创建时间**：2026-04-24  
> **状态**：待实施  
> **优先级**：高

---

## 一句话背景

约有 1TB 付费课程（抖音投流、羽毛球等）存在百度云盘，不想下载本地，要让 Hermes 自动消化成 Obsidian 笔记，以后查询直接召回。

## 核心原则

1. **视频不落盘**：AList WebDAV 挂载 → ffmpeg 流式读取
2. **LLM 不能直接看视频**：必须 视频 → 音频转文字 + 关键帧截图 + VLM
3. **最终产物**：Markdown + 少量截图，1TB 视频压缩到 2-5GB 文字
4. **记忆分层**：MEMORY.md 存铁则 / session_search 存对话 / Obsidian + mnemo 存文档

---

## 技术栈

| 组件 | 作用 |
|------|------|
| AList（Docker, localhost:5244） | 百度云盘流式挂载 |
| ffmpeg | 流式抽音频 + 关键帧 |
| whisper.cpp（large-v3-turbo, Core ML 加速） | 音频转文字 |
| imagehash | 关键帧感知哈希去重 |
| Hermes skill（baidu-course-ingestor） | 编排入口 |
| Obsidian + Smart Connections | 知识库 + 向量检索 |
| mnemo-hermes（可选） | Ollama + pgvector 本地语义索引 |

---

## 数据流

```
百度云盘 (SVIP)
    ↓ AList WebDAV（流式，不落盘）
    ↓ baidu-course-ingestor skill
    ├─ ffmpeg → whisper-cli（.srt）
    ├─ ffmpeg → phash去重关键帧
    └─ 合并 Markdown → Obsidian Vault
    ↓ Obsidian + Smart Connections 向量化
    ↓ Hermes 查询：mnemo_recall + Obsidian 扫描 → 合并答案
```

---

## Hermes vs OpenClaw 分工

| 角色 | 职责 |
|------|------|
| **Hermes** | 总指挥：持久化记忆、编排多步流程、维护知识库索引 |
| **OpenClaw** | 即时工具：转写完成后发飞书/微信通知 |

---

## Memory 分层

| 存放位置 | 内容类型 | 容量 |
|----------|----------|------|
| `MEMORY.md` | 铁则、凭证、工作约束 | 2200 char |
| `state.db` (session_search) | 所有历史会话 | 无限 |
| Obsidian vault | 方案文档、课程笔记 | 无限 |
| mnemo-hermes | 语义索引层 | 无限 |

---

## 性能预估（1TB）

| 项目 | 数值 |
|------|------|
| 视频总时长 | 约 1500-2000 小时 |
| whisper large-v3-turbo（M芯片） | 约 10-15x 实时 |
| 转写总耗时 | 100-200 小时（两周内完成） |
| 最终产物 | 2-5GB Markdown+截图 |

---

## 执行优先级

1. **抖音投流课**（最急用，变现相关）
2. 其他业务课
3. 羽毛球课（兴趣）

---

## 关键风险

1. AList 首次挂载需浏览器 OAuth（必须人操作一次）
2. 第一个课程跑完必须人工审质量
3. whisper 对"千川"、"DOU+"等专有名词可能需补词表
