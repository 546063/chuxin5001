## 一句话背景

我有约 1TB 付费课程（抖音投流、羽毛球等）存在百度云盘，不想下载本地，要让 Hermes 自动消化成 Obsidian 笔记，以后查询直接召回。

## 核心原则

1. **视频不落盘**：AList WebDAV 挂载 → ffmpeg 流式读取
2. **LLM 不能直接看视频**：必须 视频 → 音频转文字 + 关键帧截图 + OCR/VLM
3. **最终产物**：Markdown + 少量截图，1TB 视频压缩到 2-5GB 文字
4. **记忆分层**：MEMORY.md 存铁则 / session_search 存对话 / Obsidian + mnemo 存文档

## 技术栈

| 组件 | 作用 | 路径 / 命令 |
|------|------|-------------|
| AList | 百度云盘流式挂载 | Docker, localhost:5244, WebDAV 协议 |
| ffmpeg | 流式抽音频 + 关键帧 | 已装 |
| whisper.cpp | 音频转文字（large-v3-turbo，Core ML 加速） | ~/Tools/whisper.cpp |
| imagehash | 关键帧感知哈希去重 | pip install imagehash |
| Hermes skill baidu-course-ingestor | 编排入口 | ~/.hermes/skills/baidu-course-ingestor/ |
| Obsidian + Smart Connections | 知识库 + 本地向量检索 | iCloud 同步 vault |
| mnemo-hermes（可选） | Hermes 语义记忆插件 | Ollama + pgvector 本地 |

## 数据流

```
百度云盘 (SVIP)
    ↓ AList WebDAV (流式，不落盘)
    ↓
baidu-course-ingestor skill
    ├─ ffmpeg 抽 16kHz 单声道 → whisper-cli (带时间戳 .srt)
    ├─ ffmpeg fps=1/30 抽帧 → phash 去重 → 保留关键帧
    └─ 合并 Markdown → Obsidian Vault
    ↓
Obsidian + Smart Connections 向量化
    ↓
Hermes 查询时：mnemo_recall + Obsidian 扫描 → 合并答案
```

## Hermes 与 OpenClaw 分工

- **Hermes 做总指挥**：持久化记忆、编排多步流程、维护知识库索引
- **OpenClaw 做即时工具**：转写完成后发飞书/微信通知、跨微信群搜索

> 不要让 OpenClaw 做转写编排（它没有 Hermes 的 MEMORY/skill 持久化）

## MEMORY 分层原则

| 存放位置 | 内容类型 | 容量 |
|----------|----------|------|
| MEMORY.md | 铁则、凭证片段、当前工作约束 | 2200 char（不调） |
| USER.md | 我的偏好、沟通风格 | 1375 char（不调） |
| state.db (session_search) | 所有历史会话 | 无限，FTS5 全文索引 |
| Obsidian vault | 方案文档、课程笔记、项目材料 | 无限 |
| mnemo-hermes (pgvector) | 语义索引层，覆盖 Obsidian | 无限 |

**关键动作**：config.yaml 里设 `nudge_interval: 0` + `flush_on_idle: false` + `sessions.retention_days: 0`，防止 Hermes 自动精简。

## 性能预估（1TB）

- 视频总时长：约 1500-2000 小时
- whisper.cpp large-v3-turbo 在 M 芯片：约 10-15x 实时
- 转写总耗时：100-200 小时（夜跑，两周内完成）
- AList SVIP 流式速度：5-10MB/s，不是瓶颈
- 最终 Markdown + 截图：2-5GB

## 执行优先级

1. 抖音投流课（最急用，变现相关）
2. 其他业务课
3. 羽毛球课（兴趣）

## 关键风险

1. **AList 首次挂载需要浏览器 OAuth**：AI 代替不了，必须人操作一次
2. **第一个课程跑完必须人工审质量**：否则 1TB 跑完才发现格式问题就惨了
3. **whisper 对专有名词识别**：抖音投流课术语多（如"千川"、"DOU+"），可能需要补 prompt 词表
4. **关键帧密度**：羽毛球动作类需要调密一些（FRAME_INTERVAL_SEC = 15），口播课可以稀疏（60）
