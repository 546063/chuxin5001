# AI执行工具地图

## 一、AI执行核心原则

1. iMac 是主控中枢。
2. Mac mini 是执行机、备份机、质检机。
3. Claude Code 负责深度整理和写入。
4. Codex 负责规划、提示词、验收。
5. OpenClaw 负责采集和自动化，不直接治理主库。
6. 所有新增内容必须接入作战地图。
7. 批量修改前必须先做 dry-run 或报告。

## 二、多 Agent 分工

### 主控 Agent

读取入口、判断任务、分配 Worker、控制写入范围、最终验收、生成报告。

### Worker A：属性与摘要

frontmatter、summary_cn、summary_en、retrieval_keywords、quality_level、decision_value。

### Worker B：双链与路由

接入作战地图、建立精准双链、修复断链、避免泛泛链接。

### Worker C：质检与报告

目录深度、空文档、C/D 级文档、红色 YAML、重复文档、治理报告。

## 三、iMac / Mac mini 分工

iMac 主库路径：

```text
/Users/mac/ai-workspaces/AI-Workspace-Obsidian
```

Mac mini 职责：远程连接 iMac、备份结果、质量抽检、辅助扫描，不抢写 iMac 当前批次文档。

## 四、标准流程

```text
读取 AI 入口
→ 判断任务类型
→ 读取对应作战地图
→ 扫描相关目录
→ 生成计划
→ 小范围执行
→ 质量检查
→ 写入报告
→ Git 提交
→ 备份到 Mac mini / NAS
```

## 五、检查项

- [ ] 一级目录保持稳定。
- [ ] 没有创建三级/四级文件夹。
- [ ] frontmatter 合法。
- [ ] 没有红色 YAML 源码泄漏。
- [ ] 没有空文档。
- [ ] 补齐 summary_cn。
- [ ] 补齐 retrieval_keywords。
- [ ] 接入至少一个作战地图。
- [ ] 建立精准双链。
- [ ] 生成治理报告。
- [ ] Git commit。
- [ ] 完成备份。

## 六、禁止事项

禁止 OpenClaw 直接整理主库；禁止两个机器同时写同一批文件；禁止无报告批量删除；禁止新增泛标签；禁止覆盖高质量旧文档而不备份。
