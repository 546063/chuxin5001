# 复盘与案例地图

## 一、复盘核心目标

复盘不是记录发生了什么，而是判断：

1. 什么有效？
2. 什么无效？
3. 为什么有效？
4. 为什么失败？
5. 下次怎么复制？
6. 哪些经验沉淀成 SOP？
7. 哪些素材可以保留？

## 二、调用顺序

```text
事件/内容/客户/广告
→ 原始情况
→ 结果数据
→ 成功或失败原因
→ 可复用经验
→ 下一步动作
→ 接入对应作战地图
```

## 四、素材保留与删除

保留：真实过程、专业能力、客户痛点、前后对比、可复用多平台、案例证据。

删除：重复、模糊、无业务价值、不能说明问题、不能接入任何作战地图。

## 五、AI调用规则

输出必须包含：复盘对象、成功/失败判断、可复用经验、是否值得入库、接入哪个作战地图、下一步动作。
