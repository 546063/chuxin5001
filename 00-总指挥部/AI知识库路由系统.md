# AI知识库路由系统

## 中文总结
本系统定义18个工作类型MOC的调用优先级、问题分类路由规则、routing_target字段使用规范。AI收到问题时，应先判断问题类型，再进入对应MOC，而不是全库乱搜。

## English Summary
This system defines call priorities for 18 working-type MOCs, question classification routing rules, and routing_target field usage specifications. AI should classify the question type first, then enter the corresponding MOC, not search the entire vault.

## 一句话结论
问题先分类，分类先看MOC，MOC先看优先级。

## 调用优先级说明

**S级：优先调用**
适合决策和执行。AI遇到相关问题时应首先查看S级MOC。

**A级：高价值参考**
适合辅助决策。AI可作为重要参考。

**B级：背景参考**
适合补充理解。AI仅当无S/A级可用时使用。

**C级：低优先级**
仅当无S/A/B可用时使用。

**D级：已清零**
Vault中不应出现D级文稿。

## related / relations / retrieval_keywords 使用规则

**related**：引用相关文稿，优先使用wikilink格式 `[[文稿名]]`

**relations**：引用结构关联，引用MOC、目录等导航性文稿

**retrieval_keywords**：领域关键词，用于AI检索匹配

## routing_target字段说明

```yaml
routing_target:
  - "[[对应MOC]]"
  - "[[项目路由]]"
```

**规则**：
1. 每篇文稿至少1个routing_target
2. routing_target优先指向工作类型MOC
3. 项目专项文稿指向项目路由
4. 内部链接用引号包住

## 检索协议与配置

AI 检索知识库时必须优先读取：

1. [[AI检索执行协议]]
2. [[AI检索路由配置]]

执行顺序：

1. 先根据 [[AI检索路由配置]] 判断问题属于哪个项目、哪个工作类型 MOC。
2. 再根据 [[AI检索执行协议]] 执行 metadata 过滤、关键词检索、图谱扩展和 Top K 重排序。
3. 不允许默认全库搜索。
4. 不允许只依赖单个泛关键词。
5. 不允许跳过 MOC 直接读大量普通文稿。

---

## 禁止全库乱搜规则

1. AI收到问题时，**先判断类型**，再走路由表
2. **不从碎片文稿开始**，而从MOC导航开始
3. **不默认全库扫描**，只在路由找不到时有限度扫描
4. 扫描范围：**先MOC→再核心文稿→再过程稿→最后碎片**
5. 如必须全库扫描，限制在相关目录内，不跨业务域

---

## 高风险目录只读规则

以下目录内容**只读，不改正文**：

- `17-个人财务与账单分析/` — 原始财务记录
- `05-客户开发与成交/` — 原始客户沟通
- `19-个人生活与兴趣项目/01-AI水族系统/` — 鱼缸原始数据

**可以写入**：模板、路由、SOP、摘要、分析结论

---

## 新文稿进入路由规则

1. 新文稿创建后，必须填写 `routing_target`
2. 优先在对应MOC中添加链接
3. 在 `retrieval_keywords` 中添加领域关键词
4. 目录总览MOC负责收录新增文稿的入口

---

## MOC维护规则

1. 每季度检查MOC中文稿数量变化
2. 新增文稿是否正确挂载到MOC
3. 失效文稿是否从MOC中移除
4. routing_target断链是否修复
