# 客户成交调用路由

## 核心目标

从咨询进入需求判断，再进入方案、报价、跟进和成交。

## 优先读取目录

- [[05-客户开发与成交]]
- [[06-客户沟通复盘]]
- [[09-销售过程复盘]]
- [[08-交付与售后复盘]]
- [[25-成交案例战报库]]

## 沟通顺序

```text
客户来源
→ 需求
→ 预算
→ 决策权
→ 时间
→ 方案匹配
→ 信任证据
→ 报价
→ 跟进
→ 复盘
```

## 报价前必须确认

- 客户具体需求。
- 使用场景。
- 预算范围。
- 决策时间。
- 交付范围。
- 售后要求。
- 是否需要备选方案。

## 跟进原则

不要只问“考虑得怎么样”。优先补充案例、风险提醒、选型建议、预算方案、对比标准和下一步动作。

## 输出标准

必须给出：客户阶段判断、当前沟通目标、推荐回复、下一步追问、成交风险。
