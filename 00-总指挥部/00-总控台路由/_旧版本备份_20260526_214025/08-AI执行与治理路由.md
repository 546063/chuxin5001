# AI执行与治理路由

## 核心原则

iMac 是主控中枢，Mac mini 是执行机/备份机/质检机。OpenClaw 采集，不直接治理主库。

## 多 Agent 分工

### 主控 Agent

负责规划、分配、验收、写报告。

### Worker A：属性与摘要

负责 frontmatter、summary_cn、summary_en、retrieval_keywords、quality_level。

### Worker B：双链与路由

负责接入路由、精准双链、断链修复。

### Worker C：质检与备份

负责目录深度、红色 YAML、空文档、Git、备份。

## 标准执行流程

```text
读取路由
→ 判断任务
→ 扫描相关目录
→ 生成计划
→ 小范围执行
→ 质量检查
→ 写治理报告
→ Git commit
→ 备份到 Mac mini / NAS
```

## 禁止事项

- 不允许全库无目标重构。
- 不允许无报告批量删除。
- 不允许两个机器同时写同一批文件。
- 不允许创建三级/四级目录。
- 不允许新增泛标签。
