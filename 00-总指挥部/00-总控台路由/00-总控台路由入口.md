# 总控台路由入口

## 1. 本文件用途

这是 iMac Obsidian 主库的第一路由入口。所有 Claude Code、Codex、OpenClaw 或其他 AI 工具在整理、检索、写文案、做投流分析、复盘客户案例前，先读本文件。

核心原则：

- 文件夹负责存放资料。
- 路由页负责调用资料。
- Properties / frontmatter 负责筛选资料。
- 双向链接负责建立关系。
- 作战地图负责把资料变成业务动作。

## 3. 路由文件建议放置位置

方案一总控台路由型建议放在：

```text
00-总指挥部/00-总控台路由/
```

原因：

- 总控台属于全库入口，应该放在 `00-总指挥部`。
- 不新增一级目录。
- 只新增二级目录，符合目录深度规则。

## 5. 禁止事项

- 不要全库乱扫。
- 不要新增一级目录。
- 不要创建三级/四级文件夹。
- 不要使用泛标签，例如 `#获客`、`#Facebook`、`#内容`。
- 不要把所有文章互相乱链。
- 不要把旧资料当成最新策略。
- 不要让 OpenClaw 直接治理主库。
- 不要两个机器同时写同一批文件。

## 6. AI 标准处理流程

```text
用户问题
→ 读取本文件
→ 判断业务阶段
→ 进入对应调用路由
→ 只读取相关 S/A 级文档
→ 输出可执行建议
→ 如产生新内容，接入对应路由和双链
```
