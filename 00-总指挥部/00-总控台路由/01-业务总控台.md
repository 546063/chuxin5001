# 业务总控台

## 1. 全库定位

本库不是资料仓库，而是业务作战系统。

总业务链路：

```text
市场/产品判断
→ 获客定位
→ 内容生产
→ 投流放大
→ 私信/WhatsApp/微信承接
→ 客户成交
→ 交付售后
→ 复盘案例
→ 方法论沉淀
```

## 3. 新文档接入标准

任何新文档必须回答：

1. 服务哪条业务主线？
2. 应该进入哪个调用路由？
3. 解决什么业务问题？
4. 是否值得 AI 优先调用？
5. 是否要加入案例库或复盘库？
6. 是否有明确 `routing_target`？
7. 是否补充 `summary_cn` 和 `retrieval_keywords`？

## 4. 质量等级调用规则

- S：用户亲自验证，可直接调用。
- A：来源可靠，适合决策参考。
- B：可参考，但需要结合场景判断。
- C：低优先级，只做历史参考。
- D：垃圾、重复、过时、无业务价值，原则上不进入主路由。
