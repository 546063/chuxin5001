# 00-任务分类器路由_V2

> **定位**：本文件是**第一层任务分类器**，仅负责判断用户输入属于哪一类任务。
> **不是知识库正文索引**，不包含各场景的具体执行步骤或知识包。
> **流程**：用户输入 → 任务类型判断 → 推荐下一层场景路由 → 后续由场景路由决定知识包。

---

## 路由流程

```
用户输入（自然语言 / 飞书消息 / 队列任务）
  │
  ▼
关键词匹配 + 意图判断 → 8 类之一
  │
  ▼
输出：{ task_type, primary_route, backup_route, mode, read_budget }
  │
  ▼
交给对应场景路由加载知识包（本文件不读取知识包正文）
```

## 全局禁止规则

1. **禁止全库扫描**：本文件及所有下游路由不得发起对整个 Obsidian Vault 的递归扫描。
2. **禁止默认全文读取**：所有任务类型默认 `全文读取 = 否`，仅按需读取 Top K 片段。
3. **禁止课程/人物/书籍蒸馏默认挂入文案任务**（类别 7 ≠ 类别 1）：蒸馏结果需经过改写才能进入获客文案流程，两步分离。
4. **禁止越权写入**：`write_draft` 模式下生成的内容必须经人工确认后才可 `execute_write` 落盘。
5. **禁止跨路由自动串联**：不得未经人工确认自动从一类任务跳转到另一类任务（如 蒸馏 → 文案 → 投流 串联需每步确认）。

## 分类冲突处理规则

当用户输入同时命中多类触发关键词时：

| 优先级 | 规则 | 示例 |
|---|---|---|
| 1 | **付费投放 > 自然内容**：出现广告/投流关键词优先归入 4 | "帮我写个 Google Ads 文案同时优化 SEO" → 4（投放优先），SEO 部分建议辅助路由 |
| 2 | **成交环节 > 获客环节**：出现客户跟进/报价/谈判关键词优先归入 5 | "客户说太贵了帮我写回复" → 5，即使含"写"字也不是获客文案 |
| 3 | **行业知识 > 通用技能**：出现具体食品机械/认证/工艺关键词优先归入 6 | "FDA 对烘焙设备的要求" → 6 |
| 4 | **治理任务独立**：涉及 Claude Code/路由/配置的请求不与其他业务任务冲突 | "整理一下投流知识库的索引" → 8 |
| 5 | **蒸馏独立**：明确提到书籍/人物/课程 + 蒸馏/总结/提炼 → 7 | "把这本书蒸馏成决策框架然后帮我改写成广告文案" → 先 7 后 1（分两步） |

**冲突输出格式**：当分类存在歧义时，返回：
```
{
  primary_type: "主分类",
  confidence: 0.6,
  ambiguous_with: ["备选分类1", "备选分类2"],
  reason: "简要说明歧义原因",
  action: "建议先做 X 再做 Y"  // 或 "请用户确认"
}
```
