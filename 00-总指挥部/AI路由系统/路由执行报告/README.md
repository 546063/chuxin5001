# V2 路由执行报告

本目录用于保存 Claude Code 按 V2 路由系统执行任务后的报告。

## 保存内容

- 路由实战测试报告
- 任务分类验收报告
- 知识包调用记录
- Top K 读取记录
- 写入前 dry-run 报告
- 写入后验收报告
- 路由系统优化记录

## 命名规则

建议使用：

V2路由执行报告-任务类型-YYYYMMDD-HHMM.md

示例：

V2路由执行报告-获客文案-20260527-1135.md
V2路由执行报告-视频剪辑-20260527-1140.md
V2路由执行报告-知识库治理-20260527-1150.md

## 执行规则

每次执行 V2 路由任务后，报告必须包含：

1. 任务类型
2. 执行模式
3. 已读取入口
4. 主场景路由
5. 辅助场景路由
6. 主知识包
7. 辅助知识包
8. Top K 预算
9. 实际读取文件
10. 是否写入文件
11. 是否全库扫描
12. 是否符合 V2
13. 下一步建议

## 禁止事项

- 禁止把报告散落到各业务目录
- 禁止无报告直接批量写入
- 禁止不记录 Top K 读取情况
- 禁止不记录是否使用 execute_write
- 禁止不记录是否触发旧路由 fallback
