# 02-短视频脚本场景路由_V2

> **定位**：承接任务分类器判定为「2. 短视频脚本」的任务，加载知识包、控制读取预算、输出脚本草稿。
> **本文件不替代知识包正文**，仅规定「读什么、读多少、怎么写、怎么验收」。

---

## 任务识别规则

| 维度 | 规则 |
|---|---|
| **确认来自分类器** | `task_type = "短视频脚本"` 或用户输入命中分类器第 2 类触发词 |
| **必须包含** | 目标平台（TikTok / Reels / YouTube Shorts）或内容方向 |
| **排除** | 已有素材只需剪辑拼接（→ 03-视频剪辑场景路由）、纯投流素材策略（→ 07-知识包索引总表）、泛泛问"怎么拍视频"无平台指向 |
| **冲突处理** | 同时出现剪辑关键词 → 先脚本后剪辑，分两步执行并分别确认 |

## 默认上下文预算

| 任务级别 | 知识包上限 | Top K 上限 | 最大总行数 |
|---|---|---|---|
| 简单（单平台单脚本） | 1 个 | 3 篇 | ≤ 450 行 |
| 普通（多脚本变体/系列） | 2 个 | 5 篇 | ≤ 750 行 |
| 复杂（完整系列脚本 + 多语言） | 3 个 | 8 篇 | ≤ 1200 行 |

**硬规则**：
- 禁止默认全库 grep
- 禁止默认全文读取 8 篇以上
- C/D 级文稿不得作为主依据
- 增强知识包（安先生/薛辉/书籍人物）不得默认自动触发，必须用户明确要求

---

## Claude Code 执行方式

```
1. 识别任务级别（简单/普通/复杂）
2. 根据预算加载主知识包 Top K 文稿（仅读取摘要 + 关键片段）
3. 如需辅助知识包，按触发条件加载，不超额
4. 生成脚本草稿 → 存入 write_draft 区域
5. 输出验收报告 → 标注命中验收标准哪些项
6. 等待人工确认 → execute_write 落盘
```
