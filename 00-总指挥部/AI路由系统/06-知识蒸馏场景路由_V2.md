# 06-知识蒸馏场景路由_V2

> **定位**：承接任务分类器判定为「7. 书籍/人物/课程蒸馏」的任务，加载蒸馏知识包、控制读取预算、输出蒸馏结果。
> **本文件不替代知识包正文**，仅规定「读什么、读多少、怎么写、怎么验收」。
> **重要**：蒸馏结果需经过改写才能进入获客文案流程，两步分离。安先生、薛辉、书籍人物蒸馏默认作为「增强知识包」，不能每次文案任务都自动触发。

---

## 任务识别规则

| 维度 | 规则 |
|---|---|
| **确认来自分类器** | `task_type = "书籍/人物/课程蒸馏"` 或用户输入命中分类器第 7 类触发词 |
| **必须包含** | 目标对象（书名/人物名/课程名）或蒸馏类型（框架提炼/决策逻辑/精华总结） |
| **排除** | 把蒸馏结果直接当获客文案用（需先蒸馏再改写，两步分离）、纯行业知识查询（→ 04-食品机械获客场景路由） |
| **冲突处理** | 出现"蒸馏 + 改写为文案" → 先完成蒸馏（步骤 7），再进入获客文案场景路由（步骤 1），分两步并分别确认 |

## 默认上下文预算

| 任务级别 | 知识包上限 | Top K 上限 | 最大总行数 |
|---|---|---|---|
| 简单（单本书/单人框架提炼） | 1 个 | 2 篇 | ≤ 400 行 |
| 普通（多来源交叉提炼） | 2 个 | 5 篇 | ≤ 1000 行 |
| 复杂（完整课程体系蒸馏 + 业务映射） | 3 个 | 8 篇 | ≤ 1600 行 |

**硬规则**：
- 禁止默认全库 grep
- 禁止默认全文读取 8 篇以上
- C/D 级文稿不得作为主依据
- 增强知识包（安先生/薛辉/书籍人物）不得默认自动触发，必须用户明确要求或蒸馏目标直接指向该人物/课程

## Claude Code 执行方式

```
1. 识别任务级别（简单/普通/复杂）
2. 根据预算加载主知识包 Top K 文稿（仅读取摘要 + 关键框架片段）
3. 如需辅助知识包，按触发条件加载，不超额
4. 生成蒸馏结果 → 存入 write_draft 区域
5. 输出验收报告 → 标注命中验收标准哪些项
6. 等待人工确认 → execute_write 归档
7. 如需改写为文案/脚本 → 进入对应场景路由，重新确认
```

## 复盘入库规则

| 阶段 | 规则 |
|---|---|
| 生成后 | 草稿标注 `draft/` 前缀，不进入正式目录 |
| 人工确认后 | 移入 `18-个人学习与成长` 对应子目录，更新 MOC 反向链接 |
| 复盘记录 | 记录：使用了哪些知识包、哪些框架在实际决策中被引用 |
| 低质量标记 | 蒸馏结果超过 6 个月未被引用的，标记 `staleness:review` |
| 两步分离 | 蒸馏结果要改写为文案/脚本时，必须进入对应场景路由，不得直接挂入 |
