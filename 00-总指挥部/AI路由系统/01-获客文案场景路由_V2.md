# 01-获客文案场景路由_V2

> **定位**：承接任务分类器判定为「1. 获客文案」的任务，加载知识包、控制读取预算、输出文案草稿。
> **本文件不替代知识包正文**，仅规定「读什么、读多少、怎么写、怎么验收」。

---

## 任务识别规则

| 维度 | 规则 |
|---|---|
| **确认来自分类器** | `task_type = "获客文案"` 或用户输入命中分类器第 1 类触发词 |
| **必须包含** | 目标平台或载体（官网/博客/着陆页/SEO 文章/引流文案/lead magnet 等） |
| **排除** | 纯短视频脚本（→ 02-短视频脚本场景路由）、纯投流广告设置（→ 07-知识包索引总表）、纯视频剪辑（→ 03-视频剪辑场景路由） |
| **冲突处理** | 同时出现广告关键词 → 投流优先，归入 07-知识包索引总表选 Facebook投流知识包；同时出现客户跟进 → 成交优先，归入 05-客户开发成交场景路由 |

## 主知识包

| 字段 | 值 |
|---|---|
| **knowledge_pack** | `高询盘文案知识包` |
| **默认读取目录** | `02-跨境社媒获客` |
| **默认 Top K** | 3 |
| **是否允许全文读取** | 否（仅按需 read_draft） |
| **是否允许 Claude Code 写入** | write_draft（生成草稿需人工确认后才 execute_write） |

## 默认上下文预算

| 任务级别 | 知识包上限 | Top K 上限 | 最大总行数 |
|---|---|---|---|
| 简单（单平台单文案） | 1 个 | 3 篇 | ≤ 600 行 |
| 普通（多平台或多文案变体） | 2 个 | 5 篇 | ≤ 1000 行 |
| 复杂（全套文案 + 多语言 + SEO 优化） | 3 个 | 8 篇 | ≤ 1800 行 |

**硬规则**：
- 禁止默认全库 grep
- 禁止默认全文读取 8 篇以上
- C/D 级文稿不得作为主依据

---

## Claude Code 执行方式

```
1. 识别任务级别（简单/普通/复杂）
2. 根据预算加载主知识包 Top K 文稿（仅读取摘要 + 关键片段）
3. 如需辅助知识包，按触发条件加载，不超额
4. 生成文案草稿 → 存入 write_draft 区域
5. 输出验收报告 → 标注命中验收标准哪些项
6. 等待人工确认 → execute_write 落盘
```

## 复盘入库规则

| 阶段 | 规则 |
|---|---|
| 生成后 | 草稿标注 `draft/` 前缀，不进入正式目录 |
| 人工确认后 | 移入对应业务目录，更新 MOC 反向链接 |
| 复盘记录 | 记录：使用了哪些知识包、Top K 是否足够、哪些文稿命中率高 |
| 低质量标记 | 连续 2 次未达验收标准的知识包文稿，标记 `quality:low` |
