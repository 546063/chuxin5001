# 04-食品机械获客场景路由_V2

> **定位**：承接任务分类器判定为「6. 食品机械行业内容」的任务，加载行业知识包、控制读取预算、输出行业内容或获客材料。
> **本文件不替代知识包正文**，仅规定「读什么、读多少、怎么写、怎么验收」。

---

## 任务识别规则

| 维度 | 规则 |
|---|---|
| **确认来自分类器** | `task_type = "食品机械行业内容"` 或用户输入命中分类器第 6 类触发词 |
| **必须包含** | 具体食品机械品类、工艺、认证、市场或竞品指向 |
| **排除** | 写通用营销文案（→ 01-获客文案场景路由）、做广告投放（→ 07-知识包索引总表）、纯客户跟进（→ 05-客户开发成交场景路由） |
| **冲突处理** | 出现具体机型/认证/工艺关键词 → 行业知识优先；同时出现文案需求 → 先行业内容再改写为文案，两步分离 |

| 字段 | 值 |

## 辅助知识包

| 场景 | 辅助知识包 | 触发条件 | 读取预算 |

| 文案/脚本改写 | `短视频脚本知识包` | 任务明确要求将行业内容改为短视频脚本 | +2 文稿（总 Top 5） |
| 文案/脚本改写 | `高询盘文案知识包` | 任务明确要求将行业内容改为获客文案 | +2 文稿（总 Top 5） |

## 默认上下文预算

| 任务级别 | 知识包上限 | Top K 上限 | 最大总行数 |
|---|---|---|---|
| 简单（单一机型/认证查询） | 1 个 | 3 篇 | ≤ 900 行 |
| 普通（竞品分析/多机型对比） | 2 个 | 5 篇 | ≤ 1500 行 |
| 复杂（行业报告/全套内容矩阵） | 3 个 | 8 篇 | ≤ 2400 行 |

**硬规则**：
- 禁止默认全库 grep
- 禁止默认全文读取 8 篇以上
- C/D 级文稿不得作为主依据
- 涉及法规、认证标准、产品技术参数时，必须标注数据来源和日期

---

## Claude Code 执行方式

```
1. 识别任务级别（简单/普通/复杂）
2. 根据预算加载主知识包 Top K 文稿（仅读取摘要 + 关键数据片段）
3. 如需辅助知识包，按触发条件加载，不超额
4. 生成行业内容草稿 → 存入 write_draft 区域
5. 输出验收报告 → 标注命中验收标准哪些项
6. 等待人工确认 → execute_write 落盘
```

## 复盘入库规则

| 阶段 | 规则 |
|---|---|
| 生成后 | 草稿标注 `draft/` 前缀，不进入正式目录 |

| 低质量标记 | 连续 2 次未达验收标准的内容，标记 `quality:low`；过期参数标记 `expired` |
