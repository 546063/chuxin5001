# 08-ClaudeCode执行图_V2

> **定位**：规定 Claude Code 接到任意任务后的标准执行流程。
> **目标**：让 Claude Code 按固定节点顺序执行，而不是每次自由发挥、全库搜索、乱读乱写。
> **本文件是所有场景路由的上层约束**，场景路由只负责知识包与内容模板，本文件规定「怎么查、读多少、怎么写、谁执行、怎么验收」。

---

## 一、入口节点

```
用户任务输入
  │
  ▼
【禁止】直接全库 grep / 全库扫描 / 随机读取正文
  │
  ▼
读取 00-总指挥部/AI路由系统/00-任务分类器路由_V2.md
  │
  ▼
判断任务类型 → 输出 { task_type, primary_route, backup_route, mode }
  │
  ▼
进入「条件路由节点」
```

**入口规则**：
1. 每次任务开始必须先读分类器文件，不得跳过。
2. 判断任务类型时只比对分类器的 8 类关键词和触发词，不读正文。
3. 如果用户输入已明确指定场景路由，直接跳入对应路由。
4. 如果无法匹配，输出「未识别任务类型，请用户确认」，不继续执行。

## 二、条件路由节点

根据任务类型进入对应场景路由文件：

| 任务类型 | 场景路由文件 | 说明 |
|---|---|---|
| 1. 获客文案 | `01-获客文案场景路由_V2.md` | 官网/博客/着陆页/SEO/Lead Magnet 文案 |
| 2. 短视频脚本 | `02-短视频脚本场景路由_V2.md` | TikTok/Reels/Shorts 分镜脚本 |
| 3. 视频剪辑 | `03-视频剪辑场景路由_V2.md` | 素材剪辑、字幕、BGM、转场 |
| 4. 食品机械获客 | `04-食品机械获客场景路由_V2.md` | 食品加工机械行业获客 |
| 5. 客户开发成交 | `05-客户开发成交场景路由_V2.md` | 客户跟进、谈判、报价、成交 |
| 6. 知识蒸馏 | `06-知识蒸馏场景路由_V2.md` | 人物/书籍/课程蒸馏 |
| 7. 投流广告 | `07-知识包索引总表_V2.md` | 不建独立场景路由，进入索引总表选择 Facebook 投流知识包 |
| 8. 知识库治理 / Claude Code 执行 | `08-ClaudeCode执行图_V2.md` | 本文件，默认 `scan_only` |

**路由规则**：
- 每类任务只进入一个主场景路由，不并行加载多个场景路由。
- 场景路由内部负责加载知识包、规定执行模式、生成输出模板。
- 投流广告不走独立场景路由，统一由 07-知识包索引总表 提供知识包。
- 知识库治理 / Claude Code 执行类任务回到本文件继续执行。

## 三、知识包选择节点

### 选择上限

| 任务复杂度 | 主知识包 | 辅助知识包 | 正文读取上限 |
|---|---|---|---|
| 简单任务（Top 3） | 1 个 | 最多 2 个 | 最多 3 篇正文 |
| 普通任务（Top 5） | 1 个 | 最多 2 个 | 最多 5 篇正文 |
| 复杂任务（Top 8） | 1 个 | 最多 2 个 | 最多 8 篇正文 |

### 选择流程

```
场景路由已确定
  │
  ▼
读取场景路由中规定的知识包列表
  │
  ▼
按任务复杂度选择 1 个主知识包 + 最多 2 个辅助知识包
  │
  ▼
先读知识包摘要 / MOC / 索引，不直接读正文
  │
  ▼
根据摘要判断哪些正文与当前任务相关
  │
  ▼
仅读取相关正文（不超过上限）
```

**硬性禁止**：
1. 禁止全库全文扫描。
2. 禁止把所有知识包都召回。
3. 禁止默认读取人物蒸馏、书籍蒸馏全文塞进无关任务。
4. 禁止跳过摘要直接读正文。
5. 读取正文数量超过上限时，必须先说明理由并征得用户同意。

## 四、执行模式节点

### 四种执行模式

| 模式 | 权限 | 适用场景 |
|---|---|---|
| `read_only` | 只读资料，回答问题，不写库 | 查询、确认、验证、学习 |
| `generate_only` | 生成文案/脚本，输出到对话或临时文件，不写库 | 快速生成、用户先审阅 |
| `write_draft` | 生成草稿写入 Obsidian `草稿/` 或 `待确认/` 目录，等待用户确认 | 知识库内容创作、治理建议 |
| `execute_write` | 用户明确要求后才直接写入主库 | 用户说「直接写入」「确认发布」 |

### 模式选择规则

- 场景路由文件内规定默认模式，本文件为上层约束。
- **知识库治理任务默认先 `scan_only` / `report_only`**，不得直接批量写入。
- 首次执行新类型任务时，优先 `write_draft`，确认后再 `execute_write`。
- 用户未明确授权写入时，默认 `read_only` 或 `generate_only`。
- 每次切换模式时必须向用户说明模式变化。

### 知识库治理执行流程

```
治理任务进入本文件
  │
  ▼
【默认模式】scan_only / report_only
  │
  ▼
扫描目标目录，列出问题清单（坏链、孤儿笔记、标签不一致等）
  │
  ▼
输出治理报告到 16-知识库治理/ 或路由执行报告目录
  │
  ▼
用户审阅报告 → 确认修复范围
  │
  ▼
进入 write_draft 模式，生成修复建议草稿
  │
  ▼
用户确认 → 进入 execute_write 模式
  │
  ▼
执行写入 → 输出变更报告
```

## 五、Claude Code 工具使用规则

### 查询阶段

| 操作 | 允许 | 禁止 |
|---|---|---|
| 关键词搜索 | `rg` 精准关键词 | `find` 全库乱扫 |
| 读取文件 | 先读 MOC/路由/知识包摘要 | 直接全库读正文 |
| 读取正文 | 说明预计读取哪些文件 | 静默读取 8 篇以上 |
| 批量修改 | 修改前 dry-run 报告 | 未 dry-run 直接写入 |

### 读取预算控制

1. 每次任务开始前，声明预计读取的文件列表和数量。
2. 读取正文前先说明「该文件与当前任务的关系」。
3. 读取数量接近上限时，暂停并确认是否需要继续读取。
4. 不读与当前任务类型无关的知识包。

### 写入阶段

1. 批量修改前必须输出 dry-run 报告，列出目标文件、变更内容、影响范围。
2. 用户确认 dry-run 报告后，才执行实际写入。
3. 写入后必须生成执行报告，包含：
   - 修改了哪些文件
   - 每个文件改了什么
   - 是否有备份
   - 是否有需要用户确认的事项

### 报告输出路径

| 报告类型 | 输出目录 |
|---|---|
| 路由执行报告 | `00-总指挥部/AI路由系统/路由执行报告/` |
| 知识库治理报告 | `16-知识库治理/` |
| 知识蒸馏报告 | `06-知识蒸馏与课程研发/` 对应子目录 |
| 文案/脚本草稿 | Obsidian `草稿/` 或场景路由规定目录 |

## 六、iMac / Mac mini 分工

### 角色定义

| 角色 | 职责 | 权限 |
|---|---|---|
| **iMac** | 唯一主控、任务调度、主库写入、路由决策 | 所有读写权限 |
| **Mac mini** | 辅助扫描、备份、视频素材处理、质检 | 只读主库、写入辅助目录 |

### 分工规则

1. **iMac 是唯一主库写入方**，所有 Obsidian 主库的路由文件、知识包文件写入必须由 iMac 完成。
2. **Mac mini 不做主库写入**，禁止与 iMac 同时修改同一批文件。
3. 视频大文件处理（剪辑、渲染、转码）可交给 Mac mini。
4. Mac mini 完成辅助任务后，输出结果到共享目录，由 iMac 审阅后决定是否并入主库。
5. Mac mini 可执行的操作：
   - 扫描备份状态
   - 视频素材整理与质检
   - 大文件上传/下载
   - 运行本地脚本处理视频
6. Mac mini 禁止执行的操作：
   - 修改路由文件
   - 修改知识包索引
   - 直接写入 Obsidian 主库
   - 删除主库文件

## 七、验收规则

### 每次执行后必须输出验收报告

```markdown
## 执行验收报告

| 项目 | 内容 |
|---|---|
| 识别的任务类型 | {task_type} |
| 使用的场景路由 | {route_file} |
| 使用的知识包 | {primary_kb} + {辅助知识包列表} |
| 读取的文稿数量 | {n} 篇 |
| 是否超出 Top K | {是/否，如是说明原因} |
| 是否有全库扫描 | {是/否} |
| 是否有写入 | {是/否} |
| 新增/修改文件清单 | {文件列表，如无写「无」} |
| 下一步建议 | {建议} |
```

### 硬性禁止（违反任一视为执行失败）

1. **禁止默认全库 grep** —— 必须用 `rg` 精准关键词或先读索引/MOC。
2. **禁止一次性读取 8 篇以上正文** —— 超过必须先申请。
3. **禁止把所有知识包都召回** —— 按复杂度选择上限。
4. **禁止把人物蒸馏、书籍蒸馏默认塞进文案任务** —— 无关知识包不读取。
5. **禁止未 dry-run 就批量修改** —— 修改前必须出报告。
6. **禁止 Mac mini 抢写 iMac 主库** —— 主库写入仅 iMac 可执行。
