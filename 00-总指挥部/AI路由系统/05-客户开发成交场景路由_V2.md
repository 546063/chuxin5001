# 05-客户开发成交场景路由_V2

> **定位**：承接任务分类器判定为「5. 客户开发成交」的任务，加载知识包、控制读取预算、输出沟通/开发草稿。
> **本文件不替代知识包正文**，仅规定「读什么、读多少、怎么写、怎么验收」。

---

## 任务识别规则

| 维度 | 规则 |
|---|---|
| **确认来自分类器** | `task_type = "客户开发成交"` 或用户输入命中分类器第 5 类触发词 |
| **必须包含** | 客户类型（工厂/经销商/采购商/中央厨房）、沟通渠道（邮件/WhatsApp/LinkedIn/电话）或具体场景 |
| **排除** | 广告投放问题（→ 07-知识包索引总表）、泛泛问行业知识（→ 06-知识蒸馏场景路由）、写获客文案（→ 01-获客文案场景路由） |
| **冲突处理** | 出现报价/谈判关键词 → 成交环节优先；同时出现文案需求 → 先获客文案再跟进，两步分离 |

## 主知识包

| 字段 | 值 |
|---|---|
| **knowledge_pack** | `客户异议成交知识包` |
| **默认读取目录** | `05-客户开发与成交` |
| **默认 Top K** | 3 |
| **是否允许全文读取** | 否（仅按需 read_draft） |
| **是否允许 Claude Code 写入** | write_draft（生成草稿需人工确认后才 execute_write） |

## 默认上下文预算

| 任务级别 | 知识包上限 | Top K 上限 | 最大总行数 |
|---|---|---|---|
| 简单（单一询盘回复/开发信） | 1 个 | 3 篇 | ≤ 450 行 |
| 普通（多轮跟进策略/报价模板） | 2 个 | 5 篇 | ≤ 750 行 |
| 复杂（全套客户开发流程 + 多语言） | 3 个 | 8 篇 | ≤ 1200 行 |

**硬规则**：
- 禁止默认全库 grep
- 禁止默认全文读取 8 篇以上
- C/D 级文稿不得作为主依据
- 涉及具体报价、合同条款、付款条件、交期承诺时，必须标注 `[待确认]` 占位

---

## Claude Code 执行方式

```
1. 识别任务级别（简单/普通/复杂）
2. 根据预算加载主知识包 Top K 文稿（仅读取摘要 + 关键话术片段）
3. 如需辅助知识包，按触发条件加载，不超额
4. 生成沟通草稿 → 存入 write_draft 区域
5. 输出验收报告 → 标注命中验收标准哪些项
6. 等待人工确认 → execute_write 落盘或直接发送
```

## 复盘入库规则

| 阶段 | 规则 |
|---|---|
| 生成后 | 草稿标注 `draft/` 前缀，不进入正式目录 |
| 人工确认后 | 移入 `06-客户沟通复盘` 对应客户目录，更新 MOC 反向链接 |
| 复盘记录 | 记录：使用了哪些知识包、客户实际反馈、哪些话术转化率高 |
| 低质量标记 | 连续 2 次未回复或被客户拒绝的话术模板，标记 `quality:low` |
