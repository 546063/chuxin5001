## 三、每批人工复核方法

### P0: 大额支出 (40 条) — 逐条确认

1. 打开 `manual_review_P0_large_expense.xlsx`
2. 逐条阅读 `description` 和 `counterparty_name`
3. 确认用途、是否实际消费、是否业务相关
4. 填写 `manual_final_category`、`manual_is_business_related`、`manual_comment`、`manual_review_result`

### P1: 退款关联 (53 条) — 关联原消费

1. 打开 `manual_review_P1_refund.xlsx`
2. 在 `master_ledger.xlsx` 中搜索对应原始消费记录
3. 填写 `manual_comment` 注明对应原消费
4. `manual_review_result` 填 `refund_matched`

### P2: 银行缺年份 (1,051 条) — 补日期或批量分类

1. 打开 `manual_review_P2_bank_missing_year.xlsx`
2. **0402 不是日期** — 工商银行地区/机构代码
3. 根据原 PDF 上下文补日期，或按交易描述直接分类
4. `manual_review_result` 填 `confirmed` / `recategorized` / `still_uncertain`

### P3: 小额未分类 (2,112 条) — 按商户批量归类

1. 打开 `manual_review_P3_uncategorized_small.xlsx`
2. **按 `counterparty_name` 或 `merchant_name` 排序**
3. 相同商户的批量归类（如所有 "深圳市财付通支付" 归为某分类）
4. 建议按 "Top 20 商户" 逐批处理

## 四、回写数据库

1. 使用 `manual_review_update_template.xlsx` 模板
2. 填入最终分类和审核结果
3. 运行 `scripts/apply_manual_review.py` 回写
4. 重新运行 `classify_transactions.py` → `export_reports.py` → `validate_results.py`

**`manual_review_result` 可选值**: confirmed / recategorized / non_transaction / internal_transfer / repayment / refund_matched / ignore / still_uncertain

## 五、重要规则

### 0402 不是日期

工商银行 PDF 中的 `0402` 是地区/银行机构代码，不是 04月02日。不用于推断 year/quarter/transaction_time。

### 不重新解析原始账单

只基于现有 `personal_bills.sqlite` 和 Excel 导出做复核。
